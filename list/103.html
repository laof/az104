You have an Azure subscription named Subscription1 that contains an Azure Log Analytics workspace named Workspace1.

You need to view the error events from a table named Event.
Which query should you run in Workspace1?

  • A..select * from Event where EventType == "error"
  • CorrectB..Event | search "error"
  • C..Event | where EventType is "error"
  • D..Get-Event Event | where {$_.EventType == "error"}
Suggested Answer:

Correct Answer:

B.. Event | search "error"
Explanation: In Azure Log Analytics, the correct query language is Kusto Query Language (KQL). The query Event | search "error" is valid KQL syntax and searches for the term "error" across all columns in the Event table. This is the most straightforward way to find error-related events.
-----------------

Other Options:

A.. select * from Event where EventType == "error"
This is SQL syntax, not KQL, and is not valid in Log Analytics.
C.. Event | where EventType is "error"
This is close, but the correct KQL syntax uses == for equality, not is.
D.. Get-Event Event | where {$_.EventType == "error"}
This is PowerShell syntax, not KQL, and cannot be used in Log Analytics queries.
