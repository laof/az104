You have an Azure App Services web app named App1.

You plan to deploy App1 by using Web Deploy.
You need to ensure that the developers of App1 can use their Azure AD credentials to deploy content to App1. The solution must use the principle of least privilege.
What should you do?

  • A..Assign the Owner role to the developers
  • B..Configure app-level credentials for FTPS
  • CorrectC..Assign the Website Contributor role to the developers
  • D..Configure user-level credentials for FTPS
Suggested Answer:

Correct Answer:

C.. Assign the Website Contributor role to the developers
Explanation: The Website Contributor role allows users to manage web apps, including deploying content using tools like Web Deploy. It supports using Azure AD credentials and aligns with the principle of least privilege by granting only the necessary permissions to deploy and manage web apps, without broader access to the subscription or other resources.
-----------------

Other Options:

A.. Assign the Owner role to the developers
This grants full access to all resources, which violates the principle of least privilege.
B.. Configure app-level credentials for FTPS
This does not use Azure AD credentials and is less secure and harder to manage.
D.. Configure user-level credentials for FTPS
Similar to option B, this does not leverage Azure AD and does not meet the requirement to use Azure AD credentials.
