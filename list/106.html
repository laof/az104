HOTSPOT -

You have an Azure subscription that is linked to an Azure AD tenant. The tenant contains the custom role-based access control (RBAC) roles shown in the following table.
From the Azure portal, you need to create two custom roles named Role3 and Role4. Role3 will be an Azure subscription role. Role4 will be an Azure AD role.
Which roles can you clone to create the new roles? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.

Suggested Answer:

Correct Answers:

Role3: Role1 and built-in Azure subscription roles only
Role4: Built-in Azure AD roles only
Explanation: When creating custom roles in Azure:
- Role3 is an Azure subscription role, so it can be cloned from other Azure subscription roles such as Role1 or built-in subscription roles.
- Role4 is an Azure AD role. However, custom Azure AD roles (like Role2) cannot be cloned. Only built-in Azure AD roles can be used as templates for creating new Azure AD custom roles.
This ensures that role creation adheres to the correct scope and supported cloning capabilities.
-----------------

Other Options:

Role3:
- Role1 only
Too limited; built-in subscription roles are also valid sources.
- Built-in Azure subscription roles only
Excludes custom roles like Role1, which are valid.
- Role1, Role2, built-in Azure subscription roles, and built-in Azure AD roles
Incorrect because Role2 and Azure AD roles cannot be used to create a subscription-level role.
Role4:
- Role2 only
Invalid because custom Azure AD roles cannot be cloned.
- Role2 and built-in Azure AD roles only
Includes Role2, which is not a valid source for cloning.
- Role1, Role2, built-in Azure AD, and built-in Azure subscription Roles
Includes Role1 and Role2, which are not valid sources for Azure AD role creation.
