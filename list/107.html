DRAG DROP

-
You have an Azure subscription named Sub1 that contains two users named User1 and User2.
You need to assign role-based access control (RBAC) roles to User1 and User2. The users must be able to perform the following tasks in Sub1:
• User1 must view the data in any storage account.
• User2 must assign users the Contributor role for storage accounts.
The solution must use the principle of least privilege.
Which RBAC role should you assign to each user? To answer, drag the appropriate roles to the correct users. Each role may be used once, more than once, or not at all. You may need to drag the split bar between panes or scroll to view content.
NOTE: Each correct selection is worth one point.

Suggested Answer:

Correct Answers:

User1: Reader and Data Access
User2: Owner
Explanation:
- User1view data in any storage account. The Reader and Data Access role allows read access to both the storage account resource and the data within it (e.g., blobs, files), which satisfies this requirement using the principle of least privilege.
- User2assign users the Contributor role for storage accounts. This requires role assignment permissions, which are only available through the Owner or User Access Administrator roles. Since Owner includes full access including role assignments, it is the correct choice here.
-----------------

Other Options:

- Contributor: Grants full management access to resources but does not allow role assignments, so it’s insufficient for User2.
- Storage Account Contributor: Allows management of storage accounts but not access to the data or role assignments, so it’s not suitable for either user in this scenario.
