Question : 70 Topic: 2

HOTSPOT -

You have an Azure AD tenant named contoso.com.
You have two external partner organizations named fabrikam.com and litwareinc.com. Fabrikam.com is configured as a connected organization.
You create an access package as shown in the Access package exhibit. (Click the Access package tab.)
You configure the external user lifecycle settings as shown in the Lifecycle exhibit. (Click the Lifecycle tab.)
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:

Question 2-70

1. Litwareinc.com users can be assigned to package1. - No
2. After 365 days, fabrikam.com users will be removed from Group1. - No
3. After 395 days, fabrikam.com users will be removed from the contoso.com tenant. - Yes
-----------------
Explanation of correct option:
3. After 395 days, fabrikam.com users will be removed from the contoso.com tenant.
This is correct because the lifecycle settings specify that external users are blocked after 365 days and removed 30 days later. Therefore, they are removed after 395 days.
Explanation of other options:
1. Litwareinc.com users can be assigned to package1.
This is incorrect because only fabrikam.com is configured as a connected organization. Users from litwareinc.com are not eligible for assignment to the access package.
2. After 365 days, fabrikam.com users will be removed from Group1.
This is incorrect because the lifecycle settings do not specify removal from groups. They only define blocking sign-in and eventual removal from the directory, not from specific groups.
