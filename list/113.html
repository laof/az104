Question : 73 Topic: 2

You have an Azure subscription named Subscription1 that contains a virtual network named VNet1. VNet1 is in a resource group named RG1.

Subscription1 has a user named User1. User1 has the following roles:
• Reader
• Security Admin
• Security Reader
You need to ensure that User1 can assign the Reader role for VNet1 to other users.
What should you do?

  • A..Remove User1 from the Security Reader role for Subscript on 1. Assign User1 the Contributor role for RG1.
  • CorrectB..Assign User1 the Owner role for VNet1.
  • C..Remove User1 from the Security Reader and Reader roles for Subscription1. Assign User1 the Contributor role for Subscription 1.
  • D..Assign User1 the Contributor role for VNet1.
Suggested Answer:

Correct Answer:

B. Assign User1 the Owner role for VNet1.
Explanation: The Owner role includes full access to manage resources and assign roles, which is required for User1 to assign the Reader role to others for VNet1.
-----------------

Explanation of Other Options:

A. Remove User1 from the Security Reader role for Subscription1. Assign User1 the Contributor role for RG1. — The Contributor role allows managing resources but does not allow assigning roles, so this does not meet the requirement.
C. Remove User1 from the Security Reader and Reader roles for Subscription1. Assign User1 the Contributor role for Subscription1. — Contributor still lacks permission to assign roles, so this is insufficient.
D. Assign User1 the Contributor role for VNet1. — This allows managing the resource but not assigning roles, so it does not fulfill the requirement.
-----------------

Additional:

In Azure, only the Owner and User Access Administrator roles have the necessary permissions to assign roles to other users. These roles can manage access and create role assignments using Azure RBAC.
