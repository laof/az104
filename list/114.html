Question : 74 Topic: 2

Your on-premises network contains a VPN gateway.

You have an Azure subscription that contains the resources shown in the following table.
You need to ensure that all the traffic from VM1 to storage1 travels across the Microsoft backbone network.
What should you configure?

  • A..Azure Application Gateway
  • CorrectB..private endpoints
  • C..a network security group (NSG)
  • D..Azure Virtual WAN
Suggested Answer:

Correct Answer:

B. private endpoints
Explanation: To ensure traffic from VM1 to storage1 travels across the Microsoft backbone network, configure a private endpoint for the storage account. This maps a private IP from the VNet to the storage account, keeping traffic within Microsoft’s network even with forced tunneling enabled.
-----------------

Explanation of Other Options:

A. Azure Application Gateway — Used for web traffic load balancing, not for routing VM-to-storage traffic.
C. Network Security Group (NSG) — Controls traffic flow but does not affect routing paths or ensure backbone usage.
D. Azure Virtual WAN — Designed for large-scale branch connectivity, not required for VM-to-storage routing within the same region.
-----------------

Additional:

To ensure that all traffic from your VMs to your Storage Account stays within the Microsoft backbone network, you can use either Virtual Network (VNet) Service Endpoints or Azure Private Link. Service Endpoints allow secure access to the storage account from the VNet, while Private Link assigns a private IP from the VNet to the storage account, ensuring all traffic remains on the Azure backbone. Both options provide secure and optimized routing.
