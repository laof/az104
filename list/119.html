Question : 79 Topic: 2

You have an Azure subscription named Subscription1 that contains a virtual network named VNet1. VNet1 is in a resource group named RG1.

Subscription1 has a user named User1. User1 has the following roles:
• Reader
• Security Admin
• Security Reader
You need to ensure that User1 can assign the Reader role for VNet1 to other users.
What should you do?

  • A..Remove User1 from the Security Reader role for Subscription1. Assign User1 the Contributor role for RG1.
  • CorrectB..Assign User1 the User Access Administrator role for VNet1.
  • C..Remove User1 from the Security Reader and Reader roles for Subscription1.
  • D..Assign User1 the Contributor role for VNet1.
Suggested Answer:

Correct Answer:

B. Assign User1 the User Access Administrator role for VNet1.
Explanation: The User Access Administrator role allows managing user access to Azure resources, including assigning roles. Assigning this role to User1 on VNet1 enables them to assign the Reader role to others for that resource.
-----------------

Explanation of Other Options:

A. Remove User1 from the Security Reader role for Subscription1. Assign User1 the Contributor role for RG1. — Incorrect. Contributor allows managing resources but not assigning roles.
C. Remove User1 from the Security Reader and Reader roles for Subscription. — Incorrect. This does not grant any additional permissions and does not help with role assignment.
D. Assign User... — Incomplete. The option is cut off and cannot be evaluated.
