Question : 12 Topic: 1

Your company has a Microsoft Azure subscription.

The company has datacenters in Los Angeles and New York.
You are configuring the two datacenters as geo-clustered sites for site resiliency.
You need to recommend an Azure storage redundancy option.
You have the following data storage requirements:
✑ Data must be stored on multiple nodes.
✑ Data must be stored on nodes in separate geographic locations.
✑ Data can be read from the secondary location as well as from the primary location.
Which of the following Azure stored redundancy options should you recommend?

  • A..Geo-redundant storage
  • CorrectB..Read-only geo-redundant storage
  • C..Zone-redundant storage
  • D..Locally redundant storage
Suggested Answer:
B. Read-only geo-redundant storage
Explanation: Read-only geo-redundant storage (RA-GRS) stores data in multiple geographic locations and allows read access from the secondary region, meeting all requirements for geo-redundancy and read access.
-----------------
Option A. Geo-redundant storage: Provides geo-redundancy but does not allow read access from the secondary location.
Option C. Zone-redundant storage: Replicates data across availability zones in a single region, not across geographic locations.
Option D. Locally redundant storage: Replicates data within a single data center, offering no geographic redundancy.
