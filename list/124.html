Question : 84 Topic: 2

You have an Azure subscription named Subscription1 that contains virtual network named VNet1. VNet1 is in a resource group named RG1.

A user named User1 has the following roles for Subscription1:
• Reader
• Security Admin
• Security Reader
You need to ensure that User1 can assign the Reader role for VNet1 to other users.
What should you do?

  • A..Assign User1 the Contributor role for VNet1.
  • B..Assign User1 the Network Contributor role for VNet1.
  • CorrectC..Assign User1 the User Access Administrator role for VNet1.
  • D..Remove User1 from the Security Reader and Reader roles for Subscription1. Assign User1 the Contributor role for Subscription1.
Suggested Answer:

Correct Answer: C

Assign User2 the User Access Administrator role for VNet1. — This is correct because only users with the User Access Administrator role (or equivalent permissions) can assign roles like Reader to other users. Since User1 does not have this role, assigning it to another user (User2) is a valid solution.

Explanation of Other Options:

A. Assign User1 the Contributor role for VNet1. — Incorrect because Contributor allows managing resources but not assigning roles. Without User Access Administrator, User1 cannot assign the Reader role.
B. Assign User1 the Network Contributor role for VNet1. — Incorrect because this role is limited to managing network resources and does not include permission to assign roles.
D. Remove User1 from the Security Reader roles for Subscription1. Assign User2 the Contributor role for Subscription2. — Incorrect and irrelevant. These changes do not affect User1’s ability to assign roles on VNet1.
