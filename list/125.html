Question : 85 Topic: 2

You have an Azure subscription named Subscription1 that contains virtual network named VNet1. VNet1 is in a resource group named RG1.

User named User1 has the following roles for Subscription1:
• Reader
• Security Admin
• Security Reader
You need to ensure that User1 can assign the Reader role for VNet1 to other users.
What should you do?

  • A..Remove User1 from the Security Reader and Reader roles for Subscription1. Assign User1 the Contributor role for Subscription1.
  • B..Remove User1 from the Security Reader role for Subscription1. Assign User1 the Contributor role for RG1.
  • C..Assign User1 the Network Contributor role for VNet1.
  • CorrectD..Assign User1 the User Access Administrator role for VNet1.
Suggested Answer:

Correct Answer: D

Assign User1 the User Access Administrator role for VNet1. — This is correct because only users with the User Access Administrator role (or equivalent permissions) can assign roles like Reader to other users. This role grants the ability to manage user access to Azure resources.

Explanation of Other Options:

A. Remove User1 from the Security Reader and Reader roles for Subscription1. Assign User1 the Contributor role for Subscription1. — Incorrect because Contributor allows managing resources but not assigning roles. Without User Access Administrator, User1 still cannot assign the Reader role.
B. Remove User1 from the Security Reader role for Subscription1. Assign User1 the Contributor role for RG1. — Incorrect for the same reason: Contributor does not grant permission to assign roles.
C. Assign User1 the Network Contributor role for VNet1. — Incorrect because Network Contributor is limited to managing network resources and does not include role assignment capabilities.
