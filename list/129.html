Question : 89 Topic: 2

You have an Azure subscription named Subscription1 that contains virtual network named VNet1. VNet1 is in a resource group named RG1.

A user named User1 has the following roles for Subscription1:
• Reader
• Security Admin
• Security Reader
You need to ensure that User1 can assign the Reader role for VNet1 to other users.
What should you do?

  • A..Remove User1 from the Security Reader and Reader roles for Subscription1. Assign User1 the Contributor role for Subscription1.
  • B..Assign User1 the Contributor role for VNet1.
  • CorrectC..Assign User1 the Owner role for VNet1.
  • D..Assign User1 the Network Contributor role for RG1.
Suggested Answer:

Correct Answer: C

Assign User1 the Owner role for VNet1. — This is correct because the Owner role includes all permissions of the Contributor role plus the ability to assign roles to other users. This satisfies the requirement for User1 to assign the Reader role to others.

Explanation of Other Options:

A. Remove User1 from the Security Reader and Reader roles for Subscription1. Assign User1 the Contributor role for Subscription1. — Incorrect because Contributor does not include permission to assign roles.
B. Assign User1 the Contributor role for VNet1. — Incorrect because Contributor allows resource management but not role assignments.
D. Assign User1 the Network Contributor role for RG1. — Incorrect because Network Contributor is limited to managing network resources and does not include permission to assign roles.
