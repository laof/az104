Question : 91 Topic: 2

HOTSPOT

-
You have an Azure AD tenant.
You need to create a Microsoft 365 group that contains only members of a marketing department in France.
How should you complete the dynamic membership rule? To answer, select the appropriate options in the answer area.
NOTE: Each correct answer is worth one point.

Suggested Answer:

Correct Answer: user.department | and | -eq

user.department — This is correct because the rule targets users based on their department, which is "Marketing".
and — This is correct because both conditions (department and country) must be true for a user to be included in the group.
-eq — This is correct because it checks for exact equality, which is appropriate for matching the country value "France".

Explanation of Other Options:

device.managementType and device.organizationalUnit — Incorrect because the rule is targeting user attributes, not device attributes.
user.usageLocation — Incorrect because usage location is not the same as physical country and may not reflect the user's actual department location.
or — Incorrect because it would include users who are either in Marketing or in France, not necessarily both.
typeof — Incorrect because it's not a valid operator in Azure AD dynamic membership rules.
-in and -match — Incorrect because -eq is the appropriate operator for exact string comparison in this context.
