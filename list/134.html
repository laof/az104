You have an Azure Active Directory (Azure AD) tenant named contoso.com.
You have a CSV file that contains the names and email addresses of 500 external users.
You need to create a guest user account in contoso.com for each of the 500 external users.
Solution: You create a PowerShell script that runs the New-MgUser cmdlet for each user.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:

Answer:

No
-----------------

Explanation of Correct Option:

B. No
The New-MgUser cmdlet is primarily used to create internal users and requires manual configuration of all guest user properties. It does not automatically handle guest invitations or send emails. The correct and recommended approach for inviting external users is to use the New-MgInvitation cmdlet, which simplifies the process by setting UserType to Guest, sending an invitation email, and requiring minimal input. This makes it the feasible and scalable solution for onboarding 500 external users from a CSV file.
-----------------

Explanation of Incorrect Option:

A. Yes
This is incorrect because although New-MgUser can technically create guest users, it is not designed for that purpose and lacks the automation and simplicity provided by New-MgInvitation, making it impractical for bulk guest onboarding.
