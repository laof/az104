HOTSPOT -

You have a Microsoft Entra tenant that contains the groups shown in the following table.
The tenant contains the users shown in the following table.
Which users and groups can you delete? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.

Suggested Answer:

Correct Answer (Updated Based on Microsoft Entra Behavior):

Users: User1, User2, User3, and User4
Groups: Group2 and Group4 only
In Microsoft Entra, users can be deleted regardless of license status. This includes users with direct licenses and those who inherit licenses from groups. Therefore, all listed users — User1, User2, User3, and User4 — can be deleted.
For groups, Group2 can be deleted because its members have direct licenses and do not rely on the group for licensing. Group4 can be deleted because it has no license assigned.
-----------------

Explanation of Other Options:

User4 only is incorrect because all users can be deleted regardless of license status.
User1 and User4 only is incorrect for the same reason — User2 and User3 are also deletable.
User2 and User4 only is a conservative answer based on license-aware cleanup, but technically all users are deletable.
Group2 only is incorrect because Group4 can also be deleted.
Group2 and Group3 only is incorrect because Group3 has a license and its members may rely on it.
Group1, Group2, Groups3, and Groups4 is incorrect because Group1 and Group3 may still be in use or have license implications.
