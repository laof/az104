Question : 106 Topic: 2

You have an Azure subscription that contains the Microsoft Entra identities shown in the following table.

You need to enable self-service password reset (SSPR).
For which identities can you enable SSPR in the Azure portal?

  • CorrectA..User1 only
  • B..Group1 only
  • C..User1 and Group1 only
  • D..Group1 and Group2 only
  • E..User1, Group1, and Group2
Suggested Answer:

Correct Answer:

A. User1 only
Self-Service Password Reset (SSPR) is a user-specific feature in Microsoft Entra (Azure AD). It cannot be directly enabled for groups (including both security groups and Microsoft 365 groups). Instead, groups are used to scope which users SSPR applies to — but the feature itself is always applied to individual user accounts[1](https://learn.microsoft.com/en-us/entra/identity/authentication/tutorial-enable-sspr).
-----------------

Explanation of Other Options:

B. Group1 only is incorrect because SSPR cannot be enabled for a group itself.
C. User1 and Group1 only is incorrect because Group1 is not a valid target for enabling SSPR.
D. Group1 and Group2 only is incorrect because neither group type can have SSPR directly enabled.
E. User1, Group1, and Group2 is incorrect for the same reason — only User1 is eligible for direct SSPR enablement.
[1](https://learn.microsoft.com/en-us/entra/identity/authentication/tutorial-enable-sspr): https://learn.microsoft.com/en-us/entra/identity/authentication/tutorial-enable-sspr
