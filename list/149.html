Question : 109 Topic: 2

HOTSPOT -

You have an Azure subscription that contains the resources shown in the following table.
You plan to use an Azure key vault to provide a secret to app1.
What should you create for app1 to access the key vault, and from which key vault can the secret be used? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.

Suggested Answer:

Correct Answer:

Create a: Managed identity
Use the secret from: Vault1, Vault2, or Vault3
-----------------

Explanation of Correct Options:

Managed identity:
Managed identities are the recommended way for Azure resources like container apps to securely access other Azure services, including Key Vault, without managing credentials.
Vault1, Vault2, or Vault3:
App1 can access any Key Vault in the same region (East US) as long as it has the correct permissions. The resource group does not restrict access; what matters is the region and access policy or role assignment.
-----------------

Explanation of Other Options:

Private endpoint:
This is used to access Key Vault over a private IP within a virtual network, but it does not grant identity or access permissions.
Service principal:
Service principals are used for app registrations and are more complex to manage than managed identities for Azure resources.
User account:
User accounts are not used for app-to-service authentication in production scenarios.
Vault1 only:
This unnecessarily limits the app to a single vault, even though others are accessible.
Vault1 and Vault3 only:
There is no reason to exclude Vault2 if all are in the same region and accessible.
Only vaults in RG2:
Resource group does not limit access; permissions and region do.
