Question : 110 Topic: 2

You have a Microsoft Entra tenant named contoso.com.

You collaborate with an external partner named fabrikam.com.
You plan to invite users in fabrikam.com to the contoso.com tenant.
You need to ensure that invitations can be sent only to fabrikam.com users.
What should you do in the Microsoft Entra admin center?

  • A..From Cross-tenant access settings, configure the Tenant restrictions settings.
  • B..From Cross-tenant access settings, configure the Microsoft cloud settings.
  • C..From External collaboration settings, configure the Guest user access restrictions settings.
  • CorrectD..From External collaboration settings, configure the Collaboration restrictions settings.
Suggested Answer:

Correct Answer:

D. From External collaboration settings, configure the Collaboration restrictions settings.
-----------------

Explanation of Correct Option:

D. From External collaboration settings, configure the Collaboration restrictions settings.:
This is the correct location in the Microsoft Entra admin center to configure domain allow/block lists for guest invitations. Microsoft documentation confirms that domain-based invitation restrictions are managed under Collaboration restrictions, where you can specify an allowlist (e.g., fabrikam.com) to limit who can be invited.
-----------------

Explanation of Other Options:

A. From Cross-tenant access settings, configure the Tenant restrictions settings.:
This setting controls access to external tenants, not who can be invited into your tenant.
B. From Cross-tenant access settings, configure the Microsoft cloud settings.:
This is used for managing collaboration across different Microsoft cloud environments, not domain-specific invitation control.
C. From External collaboration settings, configure the Guest user access restrictions settings.:
This setting controls what guest users can do after being invited, not which domains are allowed or blocked for invitations.
