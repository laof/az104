Question : 111 Topic: 2

You have an Azure subscription that contains a storage account named storage1. The storage1 account contains blob data.

You need to assign a role to a user named User1 to ensure that the user can access the blob data in storage1. The role assignment must support conditions.
Which two roles can you assign to User1? Each correct answer presents a complete solution.
NOTE: Each correct selection is worth one point.

  • A..Owner
  • B..Storage Account Contributor
  • C..Storage Account Backup Contributor
  • CorrectD..Storage Blob Data Contributor
  • CorrectE..Storage Blob Data Owner
  • F..Storage Blob Delegator
Suggested Answer:

Correct Answer:

D. Storage Blob Data Contributor
E. Storage Blob Data Owner
-----------------

Explanation of Correct Options:

D. Storage Blob Data Contributor:
This role allows read, write, and delete access to blob data and supports Azure role-based access control (RBAC) with conditions. It is suitable for granting access to blob data specifically.
E. Storage Blob Data Owner:
This role provides full access to blob data, including setting access control, and supports RBAC conditions. It is ideal when the user needs full control over blob data.
-----------------

Explanation of Other Options:

A. Owner:
This role grants full management access to all Azure resources, including the ability to assign roles, but it is not scoped specifically to blob data and does not support data actions or RBAC conditions for blob access.
B. Storage Account Contributor:
This role allows management of storage accounts but not access to the blob data itself. It does not support data plane access or RBAC conditions for blob data.
C. Storage Account Backup Contributor:
This role is intended for backup scenarios and does not provide general blob data access or support for conditional access.
F. Storage Blob Delegator:
This role is used for Azure AD-based authentication with Shared Access Signatures (SAS) and is not intended for direct blob data access or conditional role assignments.
