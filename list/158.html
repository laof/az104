HOTSPOT -

You have an Azure Storage account named storage1.
You have an Azure App Service app named App1 and an app named App2 that runs in an Azure container instance. Each app uses a managed identity.
You need to ensure that App1 and App2 can read blobs from storage1. The solution must meet the following requirements:
✑ Minimize the number of secrets used.
✑ Ensure that App2 can only read from storage1 for the next 30 days.
What should you configure in storage1 for each app? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

What should you configure in storage1 for each app?

App1: Access control (IAM)
App2: Shared access signatures (SAS)
-----------------
App1: Access keys – Incorrect because using access keys involves secrets, which violates the requirement to minimize the number of secrets.
App1: Advanced security – Incorrect as it is not a method for granting access to blobs.
App1: Shared access signatures (SAS) – Incorrect because SAS involves generating and managing secrets, which is not ideal for long-term access with managed identities.
App2: Access keys – Incorrect because it uses secrets and does not allow time-limited access control.
App2: Advanced security – Incorrect as it does not directly control access permissions.
App2: Access control (IAM) – Incorrect because IAM does not support time-limited access like the 30-day requirement; SAS is more appropriate for temporary access.
