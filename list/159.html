HOTSPOT -

You need to create an Azure Storage account that meets the following requirements:
✑ Minimizes costs
✑ Supports hot, cool, and archive blob tiers
✑ Provides fault tolerance if a disaster affects the Azure region where the account resides
How should you complete the command? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

How should you complete the command?

--kind StorageV2
--sku Standard_GRS
-----------------
--kind FileStorage – Incorrect because FileStorage accounts are optimized for file shares and do not support blob tiers like hot, cool, and archive.
--kind Storage – Incorrect because this is a legacy type and does not support all features, including tiering and archive.
--sku Standard_LRS – Incorrect because LRS (Locally Redundant Storage) does not provide regional fault tolerance.
--sku Standard_RAGRS – Incorrect because although it provides read-access geo-redundancy, it is more expensive than Standard_GRS and the requirement is to minimize costs.
--sku Premium_LRS – Incorrect because Premium storage is more expensive and typically used for high-performance workloads, not cost minimization.
