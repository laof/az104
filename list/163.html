You have Azure subscription that includes following Azure file shares:
You have the following on-premises servers:
You create a Storage Sync Service named Sync1 and an Azure File Sync group named Group1. Group1 uses share1 as a cloud endpoint.
You register Server1 and Server2 in Sync1. You add D:\Folder1 on Server1 as a server endpoint of Group1.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Correct Answers

share2 can be added as a cloud endpoint for Group 1: No
E:\Folder2 on Server 1 can be added as a server endpoint for Group 1: No
D:\Data on Server 2 can be added as a server endpoint for Group 1: Yes
-----------------
share2 can be added as a cloud endpoint for Group 1: No – A sync group can only have one cloud endpoint, and share1 is already used. You cannot add share2 to the same group.
E:\Folder2 on Server 1 can be added as a server endpoint for Group 1: No – A server can only have one server endpoint per sync group. Since D:\Folder1 is already added from Server1, E:\Folder2 cannot be added to the same group.
D:\Data on Server 2 can be added as a server endpoint for Group 1: Yes – A registered server can support multiple server endpoints across different sync groups. Since Server2 is registered and not yet used in Group1, it can add D:\Data as a server endpoint in this group.

Additional:

Azure File Sync Server Endpoint Rules
✔ A registered server can support multiple server endpoints
✔ A server can have one server endpoint per sync group
✔ A server can participate in multiple sync groups
-----------------
What this means:
✅ A server can sync multiple folders, but each folder must be part of a different sync group.
✅ Within a single sync group, a server can only contribute one folder as a server endpoint.
✅ So yes, a registered server can have multiple folders as server endpoints, but each must belong to a different sync group.
