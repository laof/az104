HOTSPOT -

You plan to create an Azure Storage account in the Azure region of East US 2.
You need to create a storage account that meets the following requirements:
✑ Replicates synchronously.
✑ Remains available if a single data center in the region fails.
How should you configure the storage account? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Question 3-18

Answer: 4. Zone-redundant storage (ZRS)
Answer: 3. StorageV2 (general purpose v2)
-----------------
Explanation of correct options:
Zone-redundant storage (ZRS) — ZRS replicates data synchronously across three availability zones in a region, ensuring high availability even if one data center fails.
StorageV2 (general purpose v2) — This is the recommended and most flexible account type that supports ZRS and all modern features.
Explanation of other options:
Geo-redundant storage (GRS) — Replicates asynchronously to a secondary region, not suitable for synchronous replication.
Locally-redundant storage (LRS) — Replicates synchronously but only within a single data center, so it does not meet the availability requirement.
Read-access geo-redundant storage (RA GRS) — Like GRS, it is asynchronous and not suitable for the requirement.
Blob storage — Limited to blob workloads and does not support all redundancy options.
StorageV1 (general purpose v1) — Legacy account type with fewer features and no support for ZRS.
