Question : 22 Topic: 3

You have an Azure subscription named Subscription1.

You have 5 TB of data that you need to transfer to Subscription1.
You plan to use an Azure Import/Export job.
What can you use as the destination of the imported data?

  • A..a virtual machine
  • B..an Azure Cosmos DB database
  • CorrectC..Azure File Storage
  • D..the Azure File Sync Storage Sync Service
Suggested Answer:

Correct Answer:

C..Azure File Storage
Explanation: Azure Import/Export supports importing data into Azure Blob Storage and Azure File Storage, making it suitable for transferring large datasets like 5 TB using physical drives.
-----------------
A..a virtual machine
This is incorrect because Azure Import/Export cannot import data directly into a virtual machine; data must first be imported into storage and then manually moved to the VM.
B..an Azure Cosmos DB database
Azure Import/Export does not support direct import into Azure Cosmos DB; data must be transformed and ingested using tools like Azure Data Factory.
D..the Azure File Sync Storage Sync Service
This service syncs on-premises file servers with Azure File Storage but is not a direct destination for Import/Export jobs.
