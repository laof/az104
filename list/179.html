HOTSPOT -

You have an Azure Storage account named storage1 that uses Azure Blob storage and Azure File storage.
You need to use AzCopy to copy data to the blob storage and file storage in storage1.
Which authentication method should you use for each type of storage? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Question 3-25 Answer:

Blob storage: 5. Azure Active Directory (Azure AD), access keys, and shared access signatures (SAS)
File storage: 3. Access keys and shared access signatures (SAS)
-----------------
Explanation of correct options:
Blob storage - D: Blob storage supports all three authentication methods: Azure AD for role-based access control, access keys for full access, and SAS for delegated access. This makes option D the most comprehensive and correct.
File storage - C: Azure Files supports authentication using access keys and SAS. Azure AD is only supported in specific configurations, so C is the most generally correct.
Explanation of other options:
Blob storage - A: Azure AD only is incorrect because while supported, it is not the only method.
Blob storage - B: SAS only is too limited; access keys and Azure AD are also valid.
Blob storage - C: Access keys and SAS are valid, but omits Azure AD, which is also supported.
File storage - A: Azure AD only is incorrect because it’s not always supported or configured.
File storage - B: SAS only is too limited; access keys are also valid.
File storage - D: Including Azure AD makes it too broad unless domain services are configured, which isn’t always the case.

Azure AD is only supported in specific configurations - Explanation

Azure Files supports Azure AD-based authentication only when integrated with specific identity services like Active Directory Domain Services (AD DS), Microsoft Entra Domain Services, or Microsoft Entra Kerberos for hybrid identities [1](https://learn.microsoft.com/en-us/azure/storage/files/storage-files-active-directory-overview).
This means that to use Azure AD for authentication with Azure Files, your environment must be configured to support Kerberos-based authentication over SMB, and your users or applications must be domain-joined or hybrid-joined.
Without these configurations, Azure Files will not accept Azure AD credentials directly, and you must rely on access keys or shared access signatures (SAS) instead.
-----------------
Why this matters: Many organizations do not have Azure AD DS or hybrid identity set up, so Azure AD authentication is not universally available for Azure Files. This is why the more broadly applicable answer is to use access keys and SAS unless the specific identity infrastructure is in place.
