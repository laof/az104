HOTSPOT -

You have an Azure subscription that contains the file shares shown in the following table.
You have the on-premises file shares shown in the following table.
You create an Azure file sync group named Sync1 and perform the following actions:
✑ Add share1 as the cloud endpoint for Sync1.
✑ Add data1 as a server endpoint for Sync1.
✑ Register Server1 and Server2 to Sync1.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Question 3-30 Answer:

You can add share3 as an additional cloud endpoint for Sync1: No
You can add data2 as an additional server endpoint for Sync1: Yes
You can add data3 as an additional server endpoint for Sync1: No
-----------------
Explanation of correct options:
share3 - No: Azure File Sync does not support multiple cloud endpoints in different Azure regions within the same sync group. Since share1 is in West US and share3 is in East US, share3 cannot be added to Sync1.
data2 - Yes: Server2 is already registered to Sync1, and Azure File Sync supports multiple server endpoints in the same sync group as long as they are on different paths or servers.
data3 - No: Server3 is not registered to Sync1, and only registered servers can be added as server endpoints. Therefore, data3 cannot be added unless Server3 is registered first.
