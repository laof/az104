You have an Azure subscription named Subscription1.
In Subscription1, you create an Azure file share named share1.
You create a shared access signature (SAS) named SAS1 as shown in the following exhibit:
To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

If on September 2, 2018, you run Microsoft Azure Storage Explorer on a computer that has an IP address of 193.77.134.1, and you use SAS1 to connect to the storage account, you [answer choice].

will have no access

If on September 10, 2018, you run the net use command on a computer that has an IP address of 193.77.134.50, and you use SAS1 as the password to connect to share1, you [answer choice].

will have no access

-----------------

Explanations of right options:

will have no access: For the first scenario, the IP address 193.77.134.1 is NOT within the allowed IP range (193.77.134.10–193.77.134.50), so access is denied regardless of other settings.
will have no access: For the second scenario, even though 193.77.134.50 is within the allowed IP range, the SAS token only allows HTTPS protocol, but the net use command does NOT support HTTPS (it uses SMB over TCP), so access is denied.

-----------------

Explanation of other options:

will be prompted for credentials: The SAS restriction is at the network/service layer, so no authentication prompt will be shown—access is simply denied.
will have read, write, and delete access: IP or protocol restrictions prevent access before permissions are evaluated, so these permissions are not granted.
will have read, write, and list access: The SAS does not grant List permission, and access is denied due to IP/protocol restrictions.
will have read-only access: Even though the SAS includes read permission, access is denied due to IP/protocol restrictions.
