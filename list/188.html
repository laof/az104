Question : 34 Topic: 3

You have two Azure virtual machines named VM1 and VM2. You have two Recovery Services vaults named RSV1 and RSV2.

VM2 is backed up to RSV1.
You need to back up VM2 to RSV2.
What should you do first?

  • CorrectA..From the RSV1 blade, click Backup items and stop the VM2 backup
  • B..From the RSV2 blade, click Backup. From the Backup blade, select the backup for the virtual machine, and then click Backup
  • C..From the VM2 blade, click Disaster recovery, click Replication settings, and then select RSV2 as the Recovery Services vault
  • D..From the RSV1 blade, click Backup Jobs and export the VM2 job
Suggested Answer:

The first thing you should do is:

A..From the RSV1 blade, click Backup items and stop the VM2 backup

-----------------

Explanations of right option:

A..From the RSV1 blade, click Backup items and stop the VM2 backup: An Azure VM can only be backed up to one Recovery Services vault at a time. Before you can back up VM2 to RSV2, you must first stop (remove) the backup of VM2 from RSV1.

-----------------

Explanation of other options:

B..From the RSV2 blade, click Backup. From the Backup blade, select the backup for the virtual machine, and then click Backup: You cannot configure a new backup for a VM in a new vault (RSV2) while it is still being backed up in another vault (RSV1).
C..From the VM2 blade, click Disaster recovery, click Replication settings, and then select RSV2 as the Recovery Services vault: This option is for disaster recovery (replication), not backup.
D..From the RSV1 blade, click Backup Jobs and export the VM2 job: Exporting backup jobs does not move or stop the backup for VM2; you must stop the backup before configuring a new one in RSV2.

If you want to change the recovery service vault you need to dissociate the previous RSV and delete the backup data. To delete backup data, you need to stop the backup first.
So:

1. Stop the backup in RSV1 (D)
2. Remove the backup data.
3. Disassociate the VM in RSV1.
4. Associate the VM in RSV2.
