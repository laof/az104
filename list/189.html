Question : 35 Topic: 3

You have a general-purpose v1 Azure Storage account named storage1 that uses locally-redundant storage (LRS).

You need to ensure that the data in the storage account is protected if a zone fails. The solution must minimize costs and administrative effort.
What should you do first?

  • A..Create a new storage account.
  • B..Configure object replication rules.
  • CorrectC..Upgrade the account to general-purpose v2.
  • D..Modify the Replication setting of storage1.
Suggested Answer:

The first thing you should do is:

C..Upgrade the account to general-purpose v2.

-----------------

Explanations of right option:

C..Upgrade the account to general-purpose v2.: To protect data against zone failures, you need to use zone-redundant storage (ZRS) or geo-redundant storage (GRS), which are only supported on general-purpose v2 accounts. Upgrading to v2 enables you to select these replication options without having to create a new storage account, minimizing cost and administrative effort.

-----------------

Explanation of other options:

A..Create a new storage account: This is not necessary because you can upgrade your existing account to v2.
B..Configure object replication rules: Object replication is for synchronizing blobs between accounts, not for zone failure protection.
D..Modify the Replication setting of storage1: You cannot change the replication type to ZRS or GRS on a v1 account; you must upgrade to v2 first.

Addtional:

Standard general-purpose v1 = LRS/GRS/RA-GRS
Blob Storage = LRS/GRS/RA-GRS
Standard general-purpose v2 = LRS/ZRS/GRS/RA-GRS/GZRS/RA-GZRS
Premium block blobs = LRS/ZRS
Premium page blobs = LRS/ZRS
Premium file shares = LRS/ZRS
