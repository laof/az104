HOTSPOT -

You have an Azure subscription that contains an Azure Storage account named storageaccount1.
You export storageaccount1 as an Azure Resource Manager template. The template contains the following sections.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point
Hot Area:

Suggested Answer:
Explanations of right options:
1. Yes – The networkAcls section has "defaultAction": "Allow", meaning all IP addresses, including public ones, are allowed unless specifically restricted.
2. Yes – storageaccount1 is of kind "StorageV2", which supports blob tiers including hot, cool, and archive, so individual blobs can be set to the archive tier.
3. No – Azure File Shares do not support using Azure AD global admin credentials for direct SMB access; Azure AD DS integration is required for AD-based authentication.
-----------------
Explanation of other options:
1. No – Would be incorrect because public IP access is not restricted.
2. No – Would be incorrect because archive is supported for StorageV2 blobs.
3. Yes – Would be incorrect because Azure AD credentials alone can't directly authenticate for SMB access to Azure File Shares.

Additional:

Azure AD Roles like Global Administrator don't provide access to resources.
