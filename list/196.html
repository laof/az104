You create lifecycle management rules in storage1 as shown in the following table.
You perform the actions shown in the following table.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Explanations of right options:

No
for Dep1File1.docx: Rule1 applies to blobs with prefix container1/Dep1. Dep1File1.docx was last modified on October 2. By October 10, it has not been modified for 8 days; Rule1 says if not modified for 2 days, move to archive, and if not modified for 9 days, delete. On October 10, it's not deleted yet, but it is in the archive tier, which means it cannot be read unless rehydrated.
Yes
for File2.docx: Rule2 applies to all blobs. File2.docx was last modified on October 5, so by October 10, it's only 5 days old. Rule2 moves blobs to cool storage after 3 days and archive after 9 days, so File2.docx is in cool storage and can still be read.
Yes
for File3.docx: Rule2 applies. File3.docx was last modified on October 2, so by October 10, it's 8 days old. It would have been moved to cool storage after 3 days, but not yet to archive (that happens at 9 days). It can still be read.

Yes for Dep1File1.docx – Incorrect, because it is in archive tier and can't be read directly.
