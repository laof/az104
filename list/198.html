Question : 44 Topic: 3

HOTSPOT -

You have an Azure Storage account named storage1 that stores images.
You need to create a new storage account and replicate the images in storage1 to the new account by using object replication.
How should you configure the new account? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Account type: StorageV2 only
Object type to create in the new account: Container
Explanations of right options:
Account type: StorageV2 only – Object replication for blobs is only supported with StorageV2 accounts, which provide support for advanced features like object replication.
Object type: Container – To replicate images (blobs), you must create a container in the new account, as blob data is stored inside containers.
-----------------
Explanation of other options:
StorageV2 or FileStorage only, StorageV2 or BlobStorage only, StorageV2, BlobStorage, or FileStorage – Incorrect, because only StorageV2 supports object replication; FileStorage and BlobStorage accounts lack required features.
File share, Table, Queue – Incorrect, because object replication applies only to blobs, which are stored in containers, not in file shares, tables, or queues.

Additional:

Object replication is supported for general-purpose v2 storage accounts and premium block blob accounts. Both the source and destination accounts must be either general-purpose v2 or premium block blob accounts. Object replication supports block blobs only; append blobs and page blobs aren't supported.
