Question : 45 Topic: 3

You have an on-premises server that contains a folder named D:\Folder1.

You need to copy the contents of D:\Folder1 to the public container in an Azure Storage account named contosodata.
Which command should you run?

  • A..https://contosodata.blob.core.windows.net/public
  • B..azcopy sync D:\folder1 https://contosodata.blob.core.windows.net/public --snapshot
  • CorrectC..azcopy copy D:\folder1 https://contosodata.blob.core.windows.net/public --recursive
  • D..az storage blob copy start-batch D:\Folder1 https://contosodata.blob.core.windows.net/public
Suggested Answer:
C..azcopy copy D:\folder1 https://contosodata.blob.core.windows.net/public --recursive
Explanations of right options:
C..azcopy copy D:\folder1 https://contosodata.blob.core.windows.net/public --recursive: This is the correct command to copy all contents of a local folder to an Azure blob container. The azcopy copy command with the --recursive flag uploads the folder and all its contents to the destination container.
-----------------
Explanation of other options:
A..https://contosodata.blob.core.windows.net/public – Incorrect, because this is just a URL, not a command.
B..azcopy sync D:\folder1 https://contosodata.blob.core.windows.net/public --snapshot – Incorrect, because azcopy sync is generally used for synchronizing changes, not for initial full folder copy, and --snapshot is not relevant here.
D..az storage blob copy start-batch D:\Folder1 https://contosodata.blob.core.windows.net/public – Incorrect, because az storage blob copy start-batch is not the recommended or simplest way to upload an entire local folder to a blob container; azcopy is optimized for this scenario.
