Question 3-47: AzCopy Authentication for Copying from Blob to File Share

storage1: A shared access signature (SAS) token
storage2: A shared access signature (SAS) token
Explanations of right options:
storage1: A shared access signature (SAS) token: AzCopy supports authenticating to Blob Storage with a SAS token. When you provide the SAS token in the Blob Storage URL, AzCopy uses it for access, allowing simultaneous authentication to both the source and destination in a single operation.
storage2: A shared access signature (SAS) token: AzCopy requires a SAS token for Azure File Storage because OAuth is not supported for file shares. By including a SAS token in the file share URL, AzCopy grants secure and granular access to the destination.
-----------------
storage1: OAuth: Incorrect. While Blob Storage supports OAuth, you cannot use OAuth for both Blob and File endpoints in a single AzCopy command since File Storage does not support OAuth with AzCopy.
storage2: OAuth: Incorrect. Azure File Storage does not support OAuth authentication with AzCopy. Attempting to use OAuth will result in authentication failure.
storage1: Anonymous: Incorrect. Anonymous access is disabled in this scenario, so a SAS token or other secure method is required.
storage2: Anonymous: Incorrect. Anonymous access is not allowed for file shares here.
storage1: A storage account access key: Not recommended. While access keys work, SAS tokens provide more security and control for delegated access.
storage2: A storage account access key: Not recommended. Like above, using a SAS token is more secure and preferred.
