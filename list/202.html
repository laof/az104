Question : 48 Topic: 3

You create an Azure Storage account.

You plan to add 10 blob containers to the storage account.
For one of the containers, you need to use a different key to encrypt data at rest.
What should you do before you create the container?

  • A..Generate a shared access signature (SAS).
  • B..Modify the minimum TLS version.
  • C..Rotate the access keys.
  • CorrectD..Create an encryption scope.
Suggested Answer:
D. Create an encryption scope.
Explanations of right options:
D. Create an encryption scope.: To use a different key for encrypting data at rest for a specific blob container, you must first create an encryption scope in Azure Storage. An encryption scope allows you to specify a customer-managed key or a different key than the account-level key, and you assign this scope to the blob container upon creation.
-----------------
A. Generate a shared access signature (SAS).: Incorrect. A SAS is used to delegate access to storage resources, not for encryption or key management.
B. Modify the minimum TLS version.: Incorrect. Modifying TLS version affects transport security, not encryption at rest.
C. Rotate the access keys.: Incorrect. Rotating access keys is for securing access, not for assigning different encryption keys to containers.
