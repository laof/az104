HOTSPOT

-
You have an Azure subscription.
You plan to deploy a storage account named storage1 by using the following Azure Resource Manager (ARM) template.
For each of the following statements, select Yes if the statement is hue. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:

Question 3-50: ARM Template for Azure Storage Account - Statements Analysis

Statement 1: Changes made to the data in storage1 can be rolled back after seven days. No
Statement 2: Only users located in the East US Azure region can connect to storage1. No
Statement 3: Three copies of storage1 will be maintained in the East US Azure region. Yes
Explanations of right options:
Statement 1: No: The restorePolicy is set to 6 days, not seven. This means point-in-time restore can only be performed within 6 days, not 7.
Statement 2: No: The network rules allow public access (allowPublicAccess: true and networkAcls.bypass: "AzureServices", defaultAction: "Allow"), so users from any region can connect, not just East US.
Statement 3: Yes: The storage account uses Standard_LRS (Locally Redundant Storage), which maintains three copies of the data within a single region (East US).
-----------------
Statement 1: Yes: Incorrect. The retention for point-in-time restore is only 6 days according to the ARM template.
Statement 2: Yes: Incorrect. There are no restrictions preventing users from outside the East US region from connecting.
Statement 3: No: Incorrect. LRS maintains three copies within the region, so this statement is true, not false.

Additional:

The retention period for point-in-time restore must be at least one day less than the retention period specified for soft delete. For example, if the soft delete retention period is set to seven days, then the point-in-time restore retention period may be between 1 and 6 days.
