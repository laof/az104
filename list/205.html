You have an on-premises server that contains a folder named D:\Folder1.

You need to copy the contents of D:\Folder1 to the public container in an Azure Storage account named contosodata.
Which command should you run?

  • A..az storage blob copy start D:\Folder1 https://contosodata.blob.core.windows.net/public
  • B..azcopy sync D:\folder1 https://contosodata.blob.core.windows.net/public --snapshot
  • CorrectC..azcopy copy D:\folder1 https://contosodata.blob.core.windows.net/public --recursive
  • D..az storage blob copy start-batch D:\Folder1 https://contosodata.blob.core.windows.net/public
Suggested Answer:
C. azcopy copy D:\folder1 https://contosodata.blob.core.windows.net/public --recursive
Explanations of right options:
C. azcopy copy D:\folder1 https://contosodata.blob.core.windows.net/public --recursive: azcopy copy is the correct command for copying files from a local folder to an Azure Blob container. The --recursive flag ensures that all files and subfolders within D:\folder1 are copied to the blob container. This is the standard and recommended method for bulk uploads to Azure Blob Storage.
-----------------
A. az storage blob copy start D:\Folder1 https://contosodata.blob.core.windows.net/public: Incorrect. The az storage blob copy start command is intended for copying blobs between storage accounts, not for uploading local files to Azure Blob Storage.
B. azcopy sync D:\folder1 https://contosodata.blob.core.windows.net/public --snapshot: Incorrect. While azcopy sync is used for synchronizing the source and destination, the --snapshot parameter is not needed here, and azcopy copy is more appropriate for a straightforward copy operation.
D. az storage blob copy start-batch D:\Folder1 https://contosodata.blob.core.windows.net/public: Incorrect. There is no start-batch command in az storage blob copy; this is not a valid Azure CLI command.
