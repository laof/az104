Question : 52 Topic: 3

HOTSPOT

-
You have an Azure subscription that contains a storage account named storage1. The storage1 account contains a container named container1.
You need to create a lifecycle management rule for storage1 that will automatically move the blobs in container1 to the lowest-cost tier after 90 days.
How should you complete the rule? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.

Suggested Answer:
tierToArchive
prefixMatch
Explanations of right options:
tierToArchive: To move blobs to the lowest-cost tier (Archive) after 90 days, you must use tierToArchive in the actions section. This action ensures that after 90 days, the blobs are moved to the Archive tier, which is the lowest-cost storage tier in Azure.
prefixMatch: To target blobs specifically in container1, use prefixMatch in the filters section and set its value to container1/. This way, the rule applies only to blobs in that container.
-----------------
enableAutoTierToHotFromCool: Incorrect. This setting enables automatic tiering from Cool to Hot, not to the lowest-cost tier.
tierToCool: Incorrect. While Cool is a lower-cost tier than Hot, Archive is the lowest-cost tier, so tierToArchive is the correct action.
blobIndexMatch: Incorrect. This filter matches blobs by index tags, not by container or path.
blobTypes: Incorrect. This filter matches by blob type (block, append, page), not by container.
