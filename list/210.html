HOTSPOT

-
You have an Azure AD user named User1 and a read-access geo-redundant storage (RA-GRS) account named contoso2023.
You need to meet the following requirements:
• User1 must be able to write blob data to contoso2023.
• The contoso2023 account must fail over to its secondary endpoint.
Which two settings should you configure? To answer, select the appropriate settings in the answer area.
NOTE: Each correct selection is worth one point.

Suggested Answer:
>
Required settings:
1. Access Control (IAM) — to grant User1 write permissions to the storage account.
2. Geo-replication — to initiate failover to the secondary endpoint.
Explanation:
- To allow User1 to write blob data, you must configure permissions using Access Control (IAM).
- To allow the storage account to failover to its secondary endpoint, you use Geo-replication.
