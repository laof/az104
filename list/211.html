Question : 57 Topic: 3

You have an Azure subscription that contains a storage account named storage1.

You plan to create a blob container named container1.
You need to use customer-managed key encryption for container1.
Which key should you use?

  • A..an EC key that uses the P-384 curve only
  • B..an EC key that uses the P-521 curve only
  • C..an EC key that uses the P-384 curve or P-521 curve only
  • D..an RSA key with a key size of 4096 only
  • CorrectE..an RSA key type with a key size of 2048, 3072, or 4096 only
Suggested Answer:
Correct Answer: E. an RSA key type with a key size of 2048, 3072, or 4096 only
Explanation: For Azure Storage customer-managed key encryption, only RSA keys with key sizes of 2048, 3072, or 4096 bits are supported. Elliptic Curve (EC) keys are not supported.
A. an EC key that uses the P-384 curve only: Incorrect. Azure Storage does not support EC keys for customer-managed encryption.
B. an EC key that uses the P-521 curve only: Incorrect. EC keys are not supported for this purpose.
C. an EC key that uses the P-384 curve or P-521 curve only: Incorrect. EC keys, regardless of curve, are not supported for storage encryption.
D. an RSA key with a key size of 4096 only: Incorrect. While 4096 is supported, Azure Storage also supports 2048 and 3072 key sizes, so this is too restrictive.
E. an RSA key type with a key size of 2048, 3072, or 4096 only: Correct. These are the supported key sizes for RSA keys in customer-managed key encryption for Azure Storage.
