HOTSPOT

-
You have an Azure subscription that contains a storage account named storage1. The storage1 account contains a container named container1.
You create a blob lifecycle rule named rule1.
You need to configure rule1 to automatically move blobs that were NOT updated for 45 days from contained to the Cool access tier.
How should you complete the rule? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.

Suggested Answer:
1: daysAfterModificationGreaterThan
2: Blockblob
-----------------

Explanation of right options:

daysAfterModificationGreaterThan is used to specify the number of days after the blob was last modified, which matches the requirement to move blobs that were NOT updated for 45 days.
Blockblob is the correct blob type because standard blob storage most commonly stores data as block blobs and lifecycle management supports block blobs.

Explanation of other options:

daysAfterCreationGreaterThan would move blobs based on when they were created, not when they were last updated, which does not satisfy the requirement.
daysAfterLastAccessTimeGreaterThan would move blobs based on their last access time, not last update.
AppendBlob and Pageblob are other blob types but are not mentioned in the scenario, and block blobs are the default and most common for lifecycle management rules.
