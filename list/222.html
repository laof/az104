Question : 68 Topic: 3

You have an Azure subscription that contains a storage account named storage1.

You plan to use conditions when assigning role-based access control (RBAC) roles to storage1.
Which storage1 services support conditions when assigning roles?

  • A..containers only
  • B..file shares only
  • C..tables only
  • D..queues only
  • CorrectE..containers and queues only
  • F..files shares and tables only
Suggested Answer:
E..containers and queues only
-----------------

Explanation of right option:

Containers and queues only support Azure RBAC role assignment conditions. As of now, conditions are supported for Azure Blob Storage (containers) and Azure Queue Storage.

Explanation of other options:

A.. containers only: Containers support conditions, but queues do as well, so this is incomplete.
B.. file shares only: File shares do not support RBAC conditions.
C.. tables only: Tables do not support RBAC conditions.
D.. queues only: Queues do support RBAC conditions, but containers do as well, so this is incomplete.
F.. files shares and tables only: Neither file shares nor tables support RBAC conditions.

Addtional:

Currently, conditions can be added to built-in or custom RBAC role assignments that have blob storage or queue storage data actions. This means that only Azure Blob Storage (containers) and Azure Queue Storage support conditions when assigning roles. This aligns exactly with option E ("containers and queues only") from the previous question.
The following build-in roles included:
- Storage Blob Data contributor
- Storage Blob Data Owner
- Storage Blob Data Reader
- Storage Queue Data Contributor
- Storage Queue Data Owner
- Storage Queue Data Reader
- Storage Queue Data Message Processor
