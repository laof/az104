HOTSPOT -

You have an Azure subscription that has offices in the East US and West US Azure regions.
You plan to create the storage account shown in the following exhibit.
Use the drop-down menus to select the answer choice that completes each statement based on the information presented in the graphic.
NOTE: Each correct selection is worth one point.

Suggested Answer:
To minimize the network costs of accessing adatum22, modify the Default routing tier setting.
After adatum22 is created, you can modify the Enable support for customer-managed keys setting.

Explanation of right options:

Default routing tier affects the network routing and can help minimize network egress costs by choosing between Microsoft network routing and Internet routing.
Enable support for customer-managed keys can be modified after the storage account is created, allowing you to enable or disable encryption with your own keys.
-----------------
Endpoint type: Endpoint type determines the type of endpoint (standard or premium) but does not directly minimize network costs.
Location: Location cannot be changed after creation; it's not related to network cost minimization after creation.
Network connectivity: Network connectivity (public/private endpoint) is chosen at creation for access control, not specifically for minimizing costs.
Performance: Performance (standard/premium) impacts performance, not network costs.
Enable infrastructure encryption: Enable infrastructure encryption is a security setting and can only be set at creation.
Encryption type: Encryption type (Microsoft-managed or customer-managed keys) cannot be changed after creation.
Premium account type: Premium account type cannot be changed after creation.
