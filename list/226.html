HOTSPOT

-
You have an Azure subscription.
You plan to deploy a new storage account.
You need to configure encryption for the account. The solution must meet the following requirements:
• Use a customer-managed key stored in a key vault.
• Use the maximum supported bit length.
Which type of key and which bit length should you use? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.

Suggested Answer:
Key: RSA
Bit length: 4096

Explanation of right options:

RSA is the supported algorithm for Azure Storage account encryption with customer-managed keys stored in Azure Key Vault.
4096 is the maximum supported bit length for RSA keys in Azure Key Vault for this scenario.
-----------------
AES: AES keys are not supported as customer-managed keys in Azure Key Vault for storage account encryption.
3DES: 3DES is not supported for customer-managed keys in Azure Key Vault for storage account encryption.
2048, 3072, 8192: While 2048 and 3072 are valid bit lengths for RSA, they are not the maximum supported. 8192 is not supported for this scenario.
