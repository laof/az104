Question : 76 Topic: 3

You have an Azure subscription named Subscription1.

You have 5 TB of data that you need to transfer to Subscription1.
You plan to use an Azure Import/Export job.
What can you use as the destination of the imported data?

  • A..Azure Data Factory
  • B..the Azure File Sync Storage Sync Service
  • CorrectC..Azure File Storage
  • D..Azure SQL Database
Suggested Answer:
C..Azure File Storage is correct.

Explanation of right options:

Azure File Storage is supported as a destination for Azure Import/Export jobs. You can import data directly into Azure file shares within a storage account, making it suitable for large-scale offline data transfers.
-----------------
A.. Azure Data Factory: Not supported as a direct destination for Azure Import/Export jobs. Data Factory is used for data movement and transformation, not for direct Import/Export disk jobs.
B.. the Azure File Sync Storage Sync Service: This service enables syncing file servers with Azure File shares, but it is not a direct destination for Import/Export jobs.
D.. Azure SQL Database: Not supported as a destination for Azure Import/Export jobs. The service only supports importing data to Azure Blob Storage and Azure File Storage.
