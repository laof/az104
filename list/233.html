Question : 79 Topic: 3

You have an Azure subscription that contains a virtual machine named VM1 and an Azure key vault named KV1.

You need to configure encryption for VM1. The solution must meet the following requirements:
• Store and use the encryption key in KV1.
• Maintain encryption if VM1 is downloaded from Azure.
• Encrypt both the operating system disk and the data disks.
Which encryption method should you use?

  • A..customer-managed keys
  • B..Confidential disk encryption
  • CorrectC..Azure Disk Encryption
  • D..encryption at host
Suggested Answer:
C.. Azure Disk Encryption

Explanation of right options:

Azure Disk Encryption meets all requirements: it uses keys stored in Key Vault (KV1), encrypts both OS and data disks, and maintains encryption if the VM is downloaded from Azure because the encryption is applied at the disk level, not just at the host.
-----------------
A.. customer-managed keys: This is a key management option, not a full encryption method. It is often paired with other features but does not itself guarantee OS/data disk encryption or persistence after VM download.
B.. Confidential disk encryption: This is a new feature for confidential VMs, but does not guarantee disk encryption persists if VM is downloaded or for all VM types.
D.. encryption at host: This encrypts data only while it is processed by the Azure host and does not persist if the VM is downloaded. It also does not use Key Vault for key management.

Additional:

Azure Disk Encryption (ADE) encrypts the OS and data disks of Azure virtual machines (VMs) inside your VMs by using the DM-Crypt feature of Linux or the BitLocker feature of Windows. ADE is integrated with Azure Key Vault to help you control and manage the disk encryption keys and secrets, with the option to encrypt with a key encryption key (KEK). This means it can store and use the encryption key in KV1, as per your requirement.
