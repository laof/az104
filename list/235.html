Question : 81 Topic: 3

You have an Azure subscription that contains a storage account named storage1. The storage1 account contains a container named container1.

You need to configure access to container1. The solution must meet the following requirements:
• Only allow read access.
• Allow both HTTP and HTTPS protocols.
• Apply access permissions to all the content in the container.
What should you use?

  • A..an access policy
  • CorrectB..a shared access signature (SAS)
  • C..Azure Content Delivery Network (CDN)
  • D..access keys
Suggested Answer:
B.. a shared access signature (SAS)

Explanation of right options:

A shared access signature (SAS) is more appropriate because it allows you to provide time-limited, read-only access to all content in the specified container, and you can configure it to allow both HTTP and HTTPS protocols. SAS is designed for granting granular, temporary permissions to storage resources without exposing account keys.
-----------------
A.. an access policy: An access policy by itself does not directly provide access; instead, it is used to define a set of constraints that can be associated with SAS tokens for easier management and revocation. To actually grant access, you must use a SAS, optionally with a stored access policy. Therefore, SAS is the mechanism that enforces access, while the access policy enhances SAS management.
