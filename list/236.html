Question : 82 Topic: 3

You need to create an Azure Storage account named storage1. The solution must meet the following requirements:

• Support Azure Data Lake Storage.
• Minimize costs for infrequently accessed data.
• Automatically replicate data to a secondary Azure region.
Which three options should you configure for storage1? Each correct answer presents part of the solution.
NOTE: Each correct answer is worth one point.

  • A..zone-redundant storage (ZRS)
  • CorrectB..the Cool access tire
  • CorrectC..geo-redundant storage (GRS)
  • D..the Hot access tier
  • CorrectE..hierarchical namespace
Suggested Answer:
B.. the Cool access tier
C.. geo-redundant storage (GRS)
E.. hierarchical namespace

Explanation of right options:

the Cool access tier minimizes costs for infrequently accessed data by offering lower storage prices for data that is not accessed frequently.
geo-redundant storage (GRS) automatically replicates data to a secondary Azure region, providing geographic redundancy.
hierarchical namespace must be enabled to support Azure Data Lake Storage, as it provides filesystem-like capabilities needed for Data Lake analytics scenarios.
-----------------
A.. zone-redundant storage (ZRS): ZRS provides redundancy across availability zones in a single region, but does not replicate data to a secondary Azure region.
D.. the Hot access tier: The Hot tier is intended for frequently accessed data and does not minimize costs for infrequently accessed data.
