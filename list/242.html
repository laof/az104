Question : 88 Topic: 3

HOTSPOT -

You have an Azure subscription that contains a storage account named storage1. The storage1 account contains blobs in a container named container1.
You plan to share access to storage1.
You need to generate a shared access signature (SAS). The solution must meet the following requirements:
• Ensure that the SAS can only be used to enumerate and download blobs stored in container1.
• Use the principle of least privilege.
Which three settings should you enable? To answer, select the appropriate settings in the answer area.

Suggested Answer:
Allowed services: Blob
Allowed resource types: Object
Allowed permissions: Read, List

Explanation of right options:

Blob is the correct allowed service because you only need to share access to blobs, not files, queues, or tables.
Object is the correct allowed resource type because to enumerate (list) and download blobs, access to blob objects is required. Container type could allow listing containers, which is more privilege than needed.
Read and List are the correct permissions because Read allows downloading blobs and List allows enumerating blobs. No additional permissions are needed to meet the requirements and follow the principle of least privilege.
-----------------
File, Queue, Table (Allowed services): Not needed because access is only required for blobs.
Service, Container (Allowed resource types): Not required. The service type is for account-level operations, and container type is for managing containers, which is unnecessary for just enumerating and downloading blobs.
Write, Delete, Add, Create, Update, Process, Immutable storage, Permanent delete (Allowed permissions): These provide more permissions than necessary, violating the principle of least privilege.
Blob versioning permissions and Allowed blob index permissions: Not required for the task of listing and downloading blobs.
