Question : 90 Topic: 3

HOTSPOT -

You have an Azure Storage account named contoso2024 that contains the resources shown in the following table.
You have users that have permissions for contoso2024 as shown in the following table.
The contoso2024 account is configured as shown in the following exhibit.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:
User1 can read File1. — No
User2 can read File2. — No
User3 can read File1 and File2. — No

Explanation of right options:

User1 can read File1. — No
User1 has the Reader role on the storage account, which allows viewing account configuration and metadata but does not grant permission to read actual data inside blobs or file shares. Even if Allow Blob public access is Enabled at the storage account level, the blob container itself (container1) must also have its public access level set to allow anonymous read access. By default, blob containers are private. There is no evidence that container1 is set to public access, so User1 still cannot read files by public (anonymous) access.
User2 can read File2. — No
User2 has the Storage Account Contributor role, which allows managing the storage account's settings and resources but does not allow reading data within blobs or file shares. Data access requires Storage Blob Data Reader or having the access key.
User3 can read File1 and File2. — No
Although User3 has an access key for the storage account, the setting Allow storage account key access is Disabled. This means that even with the key, User3 cannot use it to access any blob or file data in the account.
