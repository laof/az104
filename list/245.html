Question : 91 Topic: 3

HOTSPOT

-
You have an Azure subscription linked to a hybrid Microsoft Entra tenant. The tenant contains the users shown in the following table.
You create the Azure Files shares shown in the following table.
You configure identity-based access for contoso2024 as shown in the following exhibit.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:
User1 can access the content in share1. — No
User2 can access the content in share2. — Yes
User2 can access the content in share3. — No

Explanation of right options:

User2 can access the content in share2. — Yes
User2 has on-premises sync enabled and is a hybrid identity, which is required for identity-based access to Azure Files when using Active Directory integration. share2 is in the contoso2024 storage account, which is configured to allow identity-based access for authenticated users with the Storage File Data SMB Share Contributor role, so User2 can access its content.
-----------------
User1 can access the content in share1. — No
User1 does not have on-premises sync enabled, so User1 is not a hybrid identity. Only hybrid identities that are synced from on-premises AD to Azure AD can authenticate to Azure Files when Active Directory is the selected authentication method. Therefore, User1 cannot access share1.
User2 can access the content in share3. — No
share3 is in the contoso2025 storage account, not contoso2024. The identity-based access configuration and permissions described only apply to contoso2024, so User2 cannot access share3 based on the given configuration.
