Note: The question is included in a number of questions that depicts the identical set-up. However, every question has a distinctive result. Establish if the solution satisfies the requirements.

Your company's Azure subscription includes two Azure networks named VirtualNetworkA and VirtualNetworkB.
VirtualNetworkA includes a VPN gateway that is configured to make use of static routing. Also, a site-to-site VPN connection exists between your company's on- premises network and VirtualNetworkA.
You have configured a point-to-site VPN connection to VirtualNetworkA from a workstation running Windows 10. After configuring virtual network peering between
VirtualNetworkA and VirtualNetworkB, you confirm that you are able to access VirtualNetworkB from the company's on-premises network. However, you find that you cannot establish a connection to VirtualNetworkB from the Windows 10 workstation.
You have to make sure that a connection to VirtualNetworkB can be established from the Windows 10 workstation.
Solution: You choose the Allow gateway transit setting on VirtualNetworkB.
Does the solution meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B..No
-----------------
Explanation of Correct Option:
B..No — This is the correct answer. Setting Allow gateway transit on VirtualNetworkB alone does not enable it to use the VPN gateway in VirtualNetworkA. For gateway transit to work, VirtualNetworkA must have Allow gateway transit enabled, and VirtualNetworkB must have Use remote gateway enabled. The Allow gateway transit setting on VirtualNetworkB is not valid and has no effect.
-----------------
Explanation of Other Option:
A..Yes — Incorrect. This option misunderstands the configuration. Allow gateway transit is a setting that must be enabled on the network that owns the gateway (VirtualNetworkA), not on the network that wants to use it. To allow VirtualNetworkB to use the gateway in VirtualNetworkA, VirtualNetworkA must allow gateway transit, and VirtualNetworkB must use the remote gateway.
