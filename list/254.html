You deploy an Azure Kubernetes Service (AKS) cluster named AKS1.
You need to deploy a YAML file to AKS1.
Solution: From Azure CLI, you run the kubectl client.
Does this meet the goal?

Suggested Answer:

Answer and Explanation

A..Yes
Explanation of right option:
Using the kubectl client from Azure CLI is the correct approach to deploy a YAML file to an AKS cluster. After configuring kubectl to connect to AKS1 (typically with 'az aks get-credentials'), you can run 'kubectl apply -f <yourfile.yaml>' to deploy resources defined in the YAML file to AKS1.
