You have the App Service plans shown in the following table.
You plan to create the Azure web apps shown in the following table.
You need to identify which App Service plans can be used for the web apps.
What should you identify? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

App Service Plan Selection for Web Apps

WebApp1: ASP1 and ASP3 only
WebApp2: ASP1 only
Explanation of right options: WebApp1 (.NET Core 3.0, West US) can be hosted on both ASP1 (Windows, West US) and ASP3 (Linux, West US) because .NET Core 3.0 is cross-platform and supported on both Windows and Linux App Service plans. WebApp2 (ASP.NET 4.7, West US) can only be hosted on ASP1 because ASP.NET 4.7 is only supported on Windows App Service plans, and ASP1 is the only Windows plan in West US.
-----------------
WebApp1: ASP1 only
Incorrect because WebApp1 can also run on ASP3 (Linux) as .NET Core 3.0 is supported on Linux; limiting to ASP1 excludes a valid option.
WebApp1: ASP3 only
Incorrect because WebApp1 can be hosted on both ASP1 (Windows) and ASP3 (Linux); selecting only ASP3 is too restrictive.
WebApp1: ASP1 and ASP2 only
Incorrect because ASP2 is in Central US, and the location must match the app's West US requirement; ASP2 is invalid for West US web apps.
WebApp1: ASP1, ASP2, and ASP3
Incorrect because ASP2, although Windows, is in Central US, not West US; only ASP1 and ASP3 meet both OS and location requirements.
WebApp2: ASP3 only
Incorrect because ASP.NET 4.7 is only supported on Windows, and ASP3 is Linux.
WebApp2: ASP1 and ASP2 only
Incorrect because ASP2 is in Central US, not West US, so it cannot host the West US web app.
WebApp2: ASP1 and ASP3 only
Incorrect because ASP3 is Linux and does not support ASP.NET 4.7; only ASP1 is valid.
WebApp2: ASP1, ASP2, and ASP3
Incorrect because only ASP1 meets the OS (Windows) and location (West US) requirements for ASP.NET 4.7 web apps.

Additional:

Web App can only be created and identified in App Service plan in same region and resource group.
