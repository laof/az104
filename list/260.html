Question 4-8 Part 2 Answers (Azure VMSS Scaling)

If Scale1 is utilized at 85 percent for six minutes after it is deployed, Scale1 will be running 6 virtual machines.
If Scale1 is first utilized at 25 percent for six minutes after it is deployed, and then utilized at 50 percent for six minutes, Scale1 will be running 2 virtual machines.
Explanation of right options:
6 virtual machines: The initial instance count is 4. If utilization is 85% (above the 80% CPU threshold) for over 5 minutes, the scale set increases by 2 VMs: 4 + 2 = 6.
2 virtual machines: The initial instance count is 4. If utilization is 25% (below the 30% CPU threshold) for over 5 minutes, the scale set decreases by 4 VMs. However, since the minimum is 2, Scale1 will run 2 VMs. Utilization at 50% does not trigger any scaling, so it remains at 2 VMs.
-----------------
2 virtual machines (for 85% scenario): Incorrect because scaling out increases from 4, not decreases.
4 virtual machines (for 85% scenario): Incorrect because the high utilization triggers scale out.
10 virtual machines (for 85% scenario): Incorrect because only an increase of 2 VMs is triggered, not to 10.
20 virtual machines (for 85% scenario): Incorrect because scaling does not reach maximum unless multiple scale-out events are triggered.
4 virtual machines (for 25% then 50% scenario): Incorrect because utilization below 30% triggers scale in, reducing below initial count, but minimum VMs is 2.
6 virtual machines (for 25% then 50% scenario): Incorrect because there is no scale-out event triggered at 25% or 50%.
8 virtual machines (for 25% then 50% scenario): Incorrect because the scale set never increases from 4 in this scenario.
10 virtual machines (for 25% then 50% scenario): Incorrect because scaling never increases that high; only scale in occurs.
