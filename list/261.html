Question : 9 Topic: 4

You plan to automate the deployment of a virtual machine scale set that uses the Windows Server 2016 Datacenter image.

You need to ensure that when the scale set virtual machines are provisioned, they have web server components installed.
Which two actions should you perform? Each correct answer presents part of the solution.
NOTE: Each correct selection is worth one point.

  • CorrectA..Upload a configuration script
  • B..Create an automation account
  • C..Create an Azure policy
  • CorrectD..Modify the extensionProfile section of the Azure Resource Manager template
  • E..Create a new virtual machine scale set in the Azure portal
Suggested Answer:

Question 4-9 Answers

A.. Upload a configuration script
D.. Modify the extensionProfile section of the Azure Resource Manager template
Explanation of right options:
A.. Upload a configuration script: You need a script (such as a PowerShell script) to install the web server components (e.g., IIS) on the VMs when they are provisioned.
D.. Modify the extensionProfile section of the Azure Resource Manager template: You use the extensionProfile in the ARM template to specify and run the script automatically during VM provisioning, ensuring all VMs are configured with the required web server components.
-----------------
B.. Create an automation account: Incorrect because automation accounts are for runbooks and automation tasks, not for VMSS provisioning scripts.
C.. Create an Azure policy: Incorrect because Azure Policy is used for compliance and governance, not for installing software on VMs during provisioning.
E.. Create a new virtual machine scale set in the Azure portal: Incorrect because simply creating a VMSS does not ensure the web server components are installed; you must specify the installation process via script and extension.

Additional:

Virtual Machine Scale Sets can be used with the Azure Desired State Configuration (DSC) extension handler. Virtual machine scale sets provide a way to deploy and manage large numbers of virtual machines, and can elastically scale in and out in response to load. DSC is used to configure the VMs as they come online so they are running the production software.
The Custom Script Extension downloads and executes scripts on Azure VMs. This extension is useful for post deployment configuration, software installation, or any other configuration/management task. Scripts can be downloaded from Azure storage or GitHub, or provided to the Azure portal at extension run-time.
The Custom Script extension integrates with Azure Resource Manager templates, and can also be used with the Azure CLI, Azure PowerShell, Azure portal, or the REST API.
