HOTSPOT -

You have an Azure Kubernetes Service (AKS) cluster named AKS1 and a computer named Computer1 that runs Windows 10. Computer1 that has the Azure CLI installed.
You need to install the kubectl client on Computer1.
Which command should you run? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
az aks
Explanation of right options:
az: The az command is the Azure CLI, which is used to manage Azure resources, including AKS clusters.
aks: The aks subcommand is used within the Azure CLI to manage Azure Kubernetes Service clusters, including installing the kubectl client via commands like az aks install-cli.
-----------------
docker: Incorrect; Docker is a container runtime and not used for installing kubectl via Azure CLI.
msiexec.exe: Incorrect; msiexec.exe is used for installing Windows Installer packages, not for Azure CLI operations.
Install-Module: Incorrect; Install-Module is a PowerShell command for installing PowerShell modules, not kubectl via Azure CLI.
/package: Incorrect; there is no such subcommand for installing kubectl with Azure CLI.
-name: Incorrect; this is not a valid Azure CLI subcommand for installing kubectl.
pull: Incorrect; pull is used for pulling images (e.g., with Docker), not for installing kubectl using Azure CLI.
