DRAG DROP -

You onboard 10 Azure virtual machines to Azure Automation State Configuration.
You need to use Azure Automation State Configuration to manage the ongoing consistency of the virtual machine configurations.
Which three actions should you perform in sequence? To answer, move the appropriate actions from the list of actions to the answer area and arrange them in the correct order.
NOTE: More than one order of answer choices is correct. You will receive credit for any of the correct orders you select.
Select and Place:

Suggested Answer:

Question 4-11 Answers

Upload a configuration to Azure Automation State Configuration
Compile a configuration into a node configuration
Check the compliance status of the node
Explanation of right options:
Upload a configuration to Azure Automation State Configuration: First, you must upload the configuration (such as a PowerShell DSC script) to Azure Automation State Configuration so it is available for assignment.
Compile a configuration into a node configuration: After uploading, compile it into a node configuration so it can be applied to the Azure VMs.
Check the compliance status of the node: Once the configuration is assigned and applied, you check the compliance status to ensure the VMs are configured as required.
-----------------
Assign tags to the virtual machines: Incorrect; tagging is for organization and management, not for state configuration sequence.
Create a management group: Incorrect; management groups are used for organizing subscriptions, not for deploying or monitoring VM configuration consistency.

Complete process for example:

Step 1: Create and upload a configuration to Azure Automation
Step 2: Compile a configuration into a node configuration
Step 3: Register a VM to be managed by State Configuration
Step 4: Specify configuration mode settings
Step 5: Assign a node configuration to a managed node
Step 6: Check the compliance status of a managed node
