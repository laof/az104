Question : 13 Topic: 4

You create an App Service plan named Plan1 and an Azure web app named webapp1.

You discover that the option to create a staging slot is unavailable.
You need to create a staging slot for Plan1.
What should you do first?

  • CorrectA..From Plan1, scale up the App Service plan
  • B..From webapp1, modify the Application settings
  • C..From webapp1, add a custom domain
  • D..From Plan1, scale out the App Service plan
Suggested Answer:
A..From Plan1, scale up the App Service plan
Explanation of right option: The ability to create a staging slot is only available for App Service plans in the Standard tier or higher (Standard, Premium, Isolated). If you are on the Free or Basic tier, this option is unavailable, so you must scale up Plan1 to at least the Standard tier to enable deployment slots.
-----------------
B..From webapp1, modify the Application settings
Explanation: Modifying application settings does not enable deployment slots; these settings are related to app configuration and do not impact the availability of staging slots.
C..From webapp1, add a custom domain
Explanation: Adding a custom domain is unrelated to deployment slots; it's for assigning a custom URL to your web app.
D..From Plan1, scale out the App Service plan
Explanation: Scaling out increases the number of instances for load balancing and high availability, but does not change the tier or unlock deployment slots.
