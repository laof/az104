Question : 14 Topic: 4

You plan to move a distributed on-premises app named App1 to an Azure subscription.

After the planned move, App1 will be hosted on several Azure virtual machines.
You need to ensure that App1 always runs on at least eight virtual machines during planned Azure maintenance.
What should you create?

  • CorrectA..one virtual machine scale set that has 10 virtual machines instances
  • B..one Availability Set that has three fault domains and one update domain
  • C..one Availability Set that has 10 update domains and one fault domain
  • D..one virtual machine scale set that has 12 virtual machines instances
Suggested Answer:
A..one virtual machine scale set that has 10 virtual machines instances
Explanation of right option: By default, 10 VMs in a scale set with 5 fault domains will be distributed as 2 VMs per fault domain. If one fault domain becomes unavailable, at least 8 VMs remain available. This configuration meets the requirement to ensure App1 always runs on at least eight virtual machines during planned Azure maintenance.
-----------------
B..one Availability Set that has three fault domains and one update domain
Explanation: Only one update domain means all VMs could be updated at once, which does not guarantee any VMs remain available during maintenance.
C..one Availability Set that has 10 update domains and one fault domain
Explanation: While 10 update domains help distribute updates, a single fault domain limits resilience against hardware failures and cannot guarantee eight VMs stay available.
D..one virtual machine scale set that has 12 virtual machines instances
Explanation: This option also meets the requirement and provides a larger buffer, but it is not strictly necessary if 10 instances and fault domains already ensure at least 8 VMs are available.
