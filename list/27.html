Question : 27 Topic: 1

Your company has virtual machines (VMs) hosted in Microsoft Azure. The VMs are located in a single Azure virtual network named VNet1.

The company has users that work remotely. The remote workers require access to the VMs on VNet1.
You need to provide access for the remote workers.
What should you do?

  • A..Configure a Site-to-Site (S2S) VPN.
  • B..Configure a VNet-toVNet VPN.
  • CorrectC..Configure a Point-to-Site (P2S) VPN.
  • D..Configure DirectAccess on a Windows Server 2012 server VM.
  • E..Configure a Multi-Site VPN
Suggested Answer:
C. Configure a Point-to-Site (P2S) VPN.
-----------------
Explanation of Correct Option:
C. Configure a Point-to-Site (P2S) VPN. — This is the correct answer. A Point-to-Site VPN is designed for individual clients (like remote workers) to securely connect to an Azure virtual network from anywhere over the internet. It does not require a VPN device or gateway on the client side, making it ideal for remote access scenarios.
-----------------
Explanation of Other Options:
A. Configure a Site-to-Site (S2S) VPN. — Incorrect. S2S VPN is used to connect entire networks (e.g., on-premises to Azure), not individual remote users.
B. Configure a VNet-to-VNet VPN. — Incorrect. This connects two Azure virtual networks, not remote users to a virtual network.
D. Configure DirectAccess on a Windows Server 2012 server VM. — Incorrect. DirectAccess is a legacy solution and not recommended for new deployments. It also requires complex setup and is not integrated with Azure networking.
E. Configure a Multi-Site VPN. — Incorrect. Multi-Site VPN is an extension of S2S VPN that connects multiple on-premises sites to Azure, not individual users.
