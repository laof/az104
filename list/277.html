You deploy an Azure Kubernetes Service (AKS) cluster named Cluster1 that uses the IP addresses shown in the following table.

You need to provide internet users with access to the applications that run in Cluster1.
Which IP address should you include in the DNS record for Cluster1?

  • CorrectA..131.107.2.1
  • B..10.0.10.11
  • C..172.17.7.1
  • D..192.168.10.2
Suggested Answer:
A..131.107.2.1
Explanation of right option:
131.107.2.1 is the Load balancer front end IP address, which is a public IP that provides access to applications running in the AKS cluster for internet users. This is the correct IP to include in the DNS record for Cluster1 so external users can access the cluster.
-----------------
B..10.0.10.11: This is a private IP assigned to a Kubernetes cluster node and is not accessible from the internet.
C..172.17.7.1: This is a Docker bridge address, used for internal network communication on the host, not for external access.
D..192.168.10.2: This is a private IP for the Kubernetes DNS service, used for internal DNS resolution within the cluster, not for internet access.
