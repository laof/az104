HOTSPOT -

You plan to deploy an Azure container instance by using the following Azure Resource Manager template.
Use the drop-down menus to select the answer choice that completes each statement based on the information presented in the template.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Internet users can connect to the container from any device.
If Internet Information Services (IIS) in the container fail, the container will restart automatically.
Explanation of right options:
The template configures a public IP address and exposes port 80, allowing access from any device on the internet.
The restartPolicy is set to OnFailure, so the container will automatically restart if IIS fails.
-----------------
Internet users cannot connect to the container: Incorrect because a public IP is assigned and port 80 is open, enabling internet access.
Internet users can only connect to the container from devices that run Windows: Incorrect because the operating system of the client does not restrict access; any device can connect via HTTP.
If Internet Information Services (IIS) in the container fail, the container will only restart manually: Incorrect because the policy is set to automatically restart on failure.
If Internet Information Services (IIS) in the container fail, the container must be redeployed: Incorrect because redeployment is not required; the container will restart itself.
