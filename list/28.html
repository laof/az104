Your company has a Microsoft SQL Server Always On availability group configured on their Azure virtual machines (VMs).
You need to configure an Azure internal load balancer as a listener for the availability group.
Solution: You create an HTTP health probe on port 1433.
Does the solution meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B. No
-----------------
Explanation of Correct Option:
B. No — This is the correct answer. Port 1433 is used for SQL Server database connections, not for health probes. For an Azure internal load balancer to work with an Always On availability group, the health probe must use a custom TCP probe on a port that is configured in the availability group listener settings, not HTTP on port 1433. Using HTTP on 1433 is incorrect and will not properly detect the health of the SQL Server instances.
-----------------
Explanation of Other Option:
A. Yes — Incorrect. An HTTP probe on port 1433 is not valid for SQL Server Always On availability groups. The probe must be TCP-based and configured on a port that the availability group listener is set to respond to, ensuring proper failover and routing.
