Question : 29 Topic: 4

You have an app named App1 that runs on an Azure web app named webapp1.

The developers at your company upload an update of App1 to a Git repository named Git1.
Webapp1 has the deployment slots shown in the following table.
You need to ensure that the App1 update is tested before the update is made available to users.
Which two actions should you perform? Each correct answer presents part of the solution.
NOTE: Each correct selection is worth one point.

  • CorrectA..Swap the slots
  • B..Deploy the App1 update to webapp1-prod, and then test the update
  • C..Stop webapp1-prod
  • CorrectD..Deploy the App1 update to webapp1-test, and then test the update
  • E..Stop webapp1-test
Suggested Answer:
A..Swap the slots
D..Deploy the App1 update to webapp1-test, and then test the update
Explanation of right options:
First, you deploy the update to the staging slot (webapp1-test) and test it to ensure correctness. After successful testing, you swap the slots to move the tested update into production without downtime.
-----------------
B..Deploy the App1 update to webapp1-prod, and then test the update: Incorrect because deploying updates directly to production does not allow for pre-release testing.
C..Stop webapp1-prod: Incorrect because stopping the production slot is not necessary for update testing or deployment.
E..Stop webapp1-test: Incorrect because you need the test slot running to validate the update before swapping.
