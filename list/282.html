Question : 30 Topic: 4

You have an Azure subscription named Subscription1 that has the following providers registered:

✑ Authorization
✑ Automation
✑ Resources
✑ Compute
✑ KeyVault
✑ Network
✑ Storage
✑ Billing
✑ Web
Subscription1 contains an Azure virtual machine named VM1 that has the following configurations:
✑ Private IP address: 10.0.0.4 (dynamic)
✑ Network security group (NSG): NSG1
✑ Public IP address: None
✑ Availability set: AVSet
✑ Subnet: 10.0.0.0/24
✑ Managed disks: No
✑ Location: East US
You need to record all the successful and failed connection attempts to VM1.
Which three actions should you perform? Each correct answer presents part of the solution.
NOTE: Each correct selection is worth one point.

  • CorrectA..Enable Azure Network Watcher in the East US Azure region.
  • B..Add an Azure Network Watcher connection monitor.
  • C..Register the MicrosoftLogAnalytics provider.
  • CorrectD..Create an Azure Storage account.
  • E..Register the Microsoft.Insights resource provider.
  • CorrectF..Enable Azure Network Watcher flow logs.
Suggested Answer:
A..Enable Azure Network Watcher in the East US Azure region.
D..Create an Azure Storage account.
F..Enable Azure Network Watcher flow logs.
Explanation of right options:
Azure Network Watcher must be enabled in the relevant region to monitor network resources.
Enabling flow logs on the NSG records connection attempts to VM1.
Flow logs must be stored, so you need an Azure Storage account to collect and analyze the log data.
-----------------
B..Add an Azure Network Watcher connection monitor: Connection monitor is used for tracking connectivity between resources, not for logging all connection attempts.
C..Register the MicrosoftLogAnalytics provider: Log Analytics is not required for enabling or storing NSG flow logs, although it can be useful for advanced analytics.
E..Register the Microsoft.Insights resource provider: This provider is related to Azure Monitor and metrics, not required for NSG flow logs or basic connection logging.
