Question : 31 Topic: 4

You need to deploy an Azure virtual machine scale set that contains five instances as quickly as possible.

What should you do?

  • A..Deploy five virtual machines. Modify the Availability Zones settings for each virtual machine.
  • B..Deploy five virtual machines. Modify the Size setting for each virtual machine.
  • C..Deploy one virtual machine scale set that is set to VM (virtual machines) orchestration mode.
  • CorrectD..Deploy one virtual machine scale set that is set to ScaleSetVM orchestration mode.
Suggested Answer:
D..Deploy one virtual machine scale set that is set to ScaleSetVM orchestration mode.
Explanation of right option:ScaleSetVM orchestration mode is the default and recommended approach for quickly deploying and managing multiple identical VMs as a group. It enables you to deploy five instances at once and provides built-in scaling and management capabilities for all instances together.
-----------------
A..Deploy five virtual machines. Modify the Availability Zones settings for each virtual machine.: This approach requires deploying and configuring each VM individually, which is slower and does not provide the benefits of scale sets.
B..Deploy five virtual machines. Modify the Size setting for each virtual machine.: Deploying VMs individually and then changing their sizes is inefficient and time-consuming, and does not leverage the orchestration and scaling features of scale sets.
C..Deploy one virtual machine scale set that is set to VM (virtual machines) orchestration mode.: VM orchestration mode is designed for stateful workloads and does not provide the fast, automated scaling and management features of ScaleSetVM mode. It is less suitable for quickly deploying multiple identical instances.
Additional:
VM (virtual machines) orchestration mode allows you to manage each instance of a virtual machine as a separate entity. This is mainly used for situations where you want to customize the instances individually.
ScaleSetVM orchestration mode (the default mode) treats the instances in the scale set as a set, making it easier to manage them as a group, which is ideal for deploying multiple instances quickly.
ScaleSetVM orchestration mode: Virtual machine instances added to the scale set are based on the scale set configuration model. The virtual machine instance lifecycle - creation, update, deletion - is managed by the scale set. It is the current default VMSS behavior. (Scale set VMs are created in a single shot).
VM (virtual machines) orchestration mode: Virtual machines created outside of the scale set can be explicitly added to the scale set. The orchestration mode VM will only create an empty VMSS without any instances, and you will have to manually add new VMs into it by specifying the VMSS ID during the creation of the VM. (Separately VMs are created and added to scale set later)
