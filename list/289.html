Question 4-37 Answer

App1 can be moved to RG2: No
App1 can be moved to RG3: Yes
App1 can be moved to RG4: Yes
-----------------
App1 can be moved to RG2: RG2 has a Read Only lock, which prevents moving resources into the resource group.
App1 can be moved to RG3: Although moving a resource between regions may require additional steps for some types, the question is about lock limitations. RG3 has a Delete lock, which only prevents deletion, not moves, so App1 can be moved to RG3.
App1 can be moved to RG4: RG4 has no lock, so there is no lock limitation to prevent moving App1 to RG4.
Saying retained: While moving between regions may not be possible for all resource types and may require migration, this question focuses only on lock limitations, not technical migration feasibility.
Additional:
Azure does allow certain resources to be moved between regions, but not all resource types support cross-region moves. The ability to move resources between regions is limited by the resource type. For example, Azure Web Apps/App Services (such as App1 in your earlier question) cannot be moved between regions; instead, you must create a new resource in the target region and migrate your data or configuration.
