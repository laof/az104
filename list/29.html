Your company has a Microsoft SQL Server Always On availability group configured on their Azure virtual machines (VMs).
You need to configure an Azure internal load balancer as a listener for the availability group.
Solution: You set Session persistence to Client IP.
Does the solution meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B. No
-----------------
Explanation of Correct Option:
B. No — This is the correct answer. While setting Session persistence to Client IP is necessary, it is not sufficient by itself. Another critical requirement for SQL Server Always On availability groups using an Azure internal load balancer is enabling the Floating IP setting. Without Floating IP, the load balancer cannot properly route traffic during failover, even if session persistence is correctly configured.
-----------------
Explanation of Other Option:
A. Yes — Incorrect. This option overlooks the need for Floating IP to be enabled. Session persistence alone does not ensure proper failover behavior for Always On availability groups in Azure.
