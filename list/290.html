HOTSPOT -

You have an Azure subscription named Subscription1 that contains the following resource group:
✑ Name: RG1
✑ Region: West US
✑ Tag: `tag1`: `value1`
You assign an Azure policy named Policy1 to Subscription1 by using the following configurations:
✑ Exclusions: None
✑ Policy definition: Append a tag and its value to resources
✑ Assignment name: Policy1
✑ Parameters:
✑ Tag name: tag2
Tag value: value2 -
After Policy1 is assigned, you create a storage account that has the following configuration:
✑ Name: storage1
✑ Location: West US
✑ Resource group: RG1
✑ Tags: `tag3`: `value3`
You need to identify which tags are assigned to each resource.
What should you identify? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Question 4-38 Part 2 Answer

Tags assigned to RG1: "tag1": "value1" only
Tags assigned to storage1: "tag2": "value2" and "tag3": "value3" only
Explanation:
Tags assigned to RG1: "tag1": "value1" only — The policy only appends tags to new resources after its assignment, not to existing resources like RG1, so only the tag is present.
Tags assigned to storage1: "tag2": "value2" and "tag3": "value3" only — When storage1 is created, it gets the specified tag "tag3": "value3" and the policy appends "tag2": "value2".
Tags assigned to RG1: "tag2": "value2" only, "tag1": "value1" and "tag2": "value2" — These are incorrect because the policy does not retroactively add tags to existing resources at the time of assignment.
Tags assigned to storage1: "tag3": "value3" only, "tag1": "value1" and "tag3": "value3" only, "tag1": "value1", "tag2": "value2" and "tag3": "value3" — These are incorrect because only "tag2": "value2" is appended by policy and "tag1": "value1" is not assigned to storage1.
Additional:
Tags applied to the resource group are not inherited by the resources in that resource group. This means if you assign a tag to a resource group, its resources (such as VMs, storage accounts, etc.) do not automatically receive that tag. Each resource must be tagged individually if you want them to have the same tags as the resource group.
