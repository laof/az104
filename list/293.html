You have an Azure Kubernetes Service (AKS) cluster named AKS1.

You need to configure cluster autoscaler for AKS1.
Which two tools should you use? Each correct answer presents a complete solution.
NOTE: Each correct selection is worth one point.

  • A..the kubectl command
  • CorrectB..the az aks command
  • C..the Set-AzVm cmdlet
  • CorrectD..the Azure portal
  • E..the Set-AzAks cmdlet
Suggested Answer:
B. the az aks command
D. the Azure portal
Explanation:
A. the kubectl command: Kubectl configures pod-level scaling (Horizontal Pod Autoscaler), not the AKS cluster autoscaler.
B. the az aks command: The az aks command configures the AKS cluster autoscaler at the node pool level, allowing the cluster to automatically add or remove nodes.
C. the Set-AzVm cmdlet: Set-AzVm is used for managing Azure Virtual Machines, not AKS or its autoscaler.
D. the Azure portal: The Azure portal provides a graphical interface to enable and configure the AKS cluster autoscaler for node pools.
E. the Set-AzAks cmdlet: There is no Set-AzAks cmdlet to configure the AKS cluster autoscaler;It is used to create or update an AKS cluster. use Azure CLI or portal instead.
