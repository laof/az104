Question : 42 Topic: 4

You create the following resources in an Azure subscription:

✑ An Azure Container Registry instance named Registry1
✑ An Azure Kubernetes Service (AKS) cluster named Cluster1
You create a container image named App1 on your administrative workstation.
You need to deploy App1 to Cluster1.
What should you do first?

  • CorrectA..Run the docker push command.
  • B..Create an App Service plan.
  • C..Run the az acr build command.
  • D..Run the az aks create command.
Suggested Answer:
A. Run the docker push command
Explanation:
A. Run the docker push command: The first step is to upload (push) the locally built container image (App1) from your workstation to the Azure Container Registry (Registry1). This makes the image available for deployment to the AKS cluster.
B. Create an App Service plan: App Service plans are used for Azure App Service, not for AKS or container registry deployments.
C. Run the az acr build command: az acr build is used to build container images in Azure cloud from source code, but in this scenario, the image has already been created on your workstation.
D. Run the az aks create command: The AKS cluster (Cluster1) is already created, so this step is unnecessary.
Additional:
az acr build builds and pushes the image in one command.
No need to run docker push separately.
