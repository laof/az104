Your company has a Microsoft SQL Server Always On availability group configured on their Azure virtual machines (VMs).
You need to configure an Azure internal load balancer as a listener for the availability group.
Solution: You enable Floating IP.
Does the solution meet the goal?

  • CorrectA..Yes
  • B..No
Suggested Answer:
A. Yes
-----------------
Explanation of Correct Option:
A. Yes — This is the correct answer. Enabling Floating IP is a required configuration when setting up an Azure internal load balancer for a SQL Server Always On availability group. It allows the load balancer to direct traffic to the correct backend VM during failover by supporting the reuse of the same IP address across multiple VMs.
-----------------
Explanation of Other Option:
B. No — Incorrect. Without Floating IP enabled, the internal load balancer cannot properly route traffic to the active SQL Server instance during failover, which is essential for high availability in Always On configurations.
