Question : 52 Topic: 4

You plan to deploy several Azure virtual machines that will run Windows Server 2019 in a virtual machine scale set by using an Azure Resource Manager template.

You need to ensure that NGINX is available on all the virtual machines after they are deployed.
What should you use?

  • A..Deployment Center in Azure App Service
  • CorrectB..A Desired State Configuration (DSC) extension
  • C..the New-AzConfigurationAssignment cmdlet
  • D..a Microsoft Intune device configuration profile
Suggested Answer:
B. A Desired State Configuration (DSC) extension
-----------------
Explanation:
To ensure that NGINX is installed and available on all virtual machines in a scale set immediately after deployment using an Azure Resource Manager template, you should use the Desired State Configuration (DSC) extension. The DSC extension allows you to define and enforce the configuration of software (like NGINX) on Azure VMs during deployment.
-----------------
Explanation of other options:
A. Deployment Center in Azure App Service: This is used for deploying code to Azure App Service, not for configuring VM software.
C. the New-AzConfigurationAssignment cmdlet: This is used with Azure Policy and Azure Blueprints, not for directly configuring VM extensions.
D. a Microsoft Intune device configuration profile: Intune is for endpoint management, typically for client devices, not for configuring server VM workloads in Azure.
An example
Azure Virtual Machine extension run post-deployment configuration and uatomation on Azure virtual machines; it will run the specified script each time the extension is applied or updated to the VM, not only once at the first VM startup. below is a example:
az vm extension set \
  --resource-group MyResourceGroup \
  --vm-name MyVM \
  --name CustomScript \
  --publisher Microsoft.Azure.Extensions \
  --settings '{"commandToExecute":"apt-get update && apt-get install -y nginx"}'
