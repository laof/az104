You have an Azure web app named App1. App1 has the deployment slots shown in the following table:

In webapp1-test, you test several changes to App1.
You back up App1.
You swap webapp1-test for webapp1-prod and discover that App1 is experiencing performance issues.
You need to revert to the previous version of App1 as quickly as possible.
What should you do?

  • A..Redeploy App1
  • CorrectB..Swap the slots
  • C..Clone App1
  • D..Restore the backup of App1
Suggested Answer:
B..Swap the slots
Explanation of right option:
Swap the slots: Swapping deployment slots allows you to quickly revert to the previous version of the app with minimal downtime by swapping the production and staging slots back, which is the fastest rollback method.
-----------------
Explanation of other options:
Redeploy App1: This would take more time and may require access to the build artifacts.
Clone App1: Cloning creates a new app, not a rollback.
Restore the backup of App1: Restoring a backup is slower and more disruptive than simply swapping the slots.
