HOTSPOT -

You have an Azure subscription named Subscription1. Subscription1 contains two Azure virtual machines VM1 and VM2. VM1 and VM2 run Windows Server
2016.
VM1 is backed up daily by Azure Backup without using the Azure Backup agent.
VM1 is affected by ransomware that encrypts data.
You need to restore the latest backup of VM1.
To which location can you restore the backup? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Question 4-60 Answer

You can perform a file recovery of VM1 to: Any Windows computer that has Internet connectivity
You can restore VM1 to: VM1 or a new Azure virtual machine only
Explanation of right options:
Any Windows computer that has Internet connectivity: Azure Backup's file recovery allows you to mount the backup recovery point as a drive on any Windows machine with Internet access, making files accessible for recovery without restoring the full VM.
VM1 or a new Azure virtual machine only: Azure Backup allows you to restore the backed-up VM as a replacement for the original VM (VM1) or as a new VM, but not to other existing VMs.
-----------------
Explanation of other options:
VM1 only: This would limit restores to only the original VM, but Azure Backup supports restoring to a new VM as well.
VM1 and VM2 only: You cannot restore the backup of VM1 to another existing VM, such as VM2, directly.
A new Azure virtual machine only: This would prevent restoring to the original VM, which is possible.
Addititonal:you cannot restore files to a previous or future operating system version. The restore must be to the same OS version or a compatible client OS, as shown in the compatibility table.
The compatibility table specifies which client operating systems are compatible with each server OS for file restoration
Server OS -> Compatible Client OS
Windows Server 2022 -> Window11 and window 10
Windows Server 2019 -> Windows 10
Windows Server 2016 -> Windows 10
Windows Server 2012 R2 -> Windows 8.1
Windows Server 2012 -> Windows 8
Windows Server 2008 R2 -> Windows 7
