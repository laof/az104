HOTSPOT -

You have an Azure subscription.
You plan to use Azure Resource Manager templates to deploy 50 Azure virtual machines that will be part of the same availability set.
You need to ensure that as many virtual machines as possible are available if the fabric fails or during servicing.
How should you configure the template? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
platformFaultDomainCount: 3
platformUpdateDomainCount: 20
platformFaultDomainCount: 3 - This is the maximum supported value in most Azure regions and ensures VMs are spread across three separate racks or physical hardware, maximizing protection against rack or hardware failures.
platformUpdateDomainCount: 20 - This is the maximum supported value, allowing your 50 VMs to be distributed across 20 update domains. This ensures that as many VMs as possible remain available during planned Azure maintenance or servicing, as only one update domain is maintained at a time.
-----------------
platformFaultDomainCount: 0, 1, 2, 4 - Lower numbers would concentrate VMs on fewer racks, increasing the risk of downtime if a fault domain fails. Value 4 is not supported in most regions.
platformUpdateDomainCount: 10, 25, 30, 40, 50 - Values below 20 would place more VMs within each update domain, increasing the impact of maintenance. Values above 20 are not supported for availability sets.
Additional:
These regions have 3 Fault Domains:
* East US
* East US 2
* West US
* Central US
* North Central US
* South Central US
* Canada Central
* North Europe
* West Europe
The rest have 2 Fault Domains.
