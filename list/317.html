Suggested Answer:
At 9:00 AM, the scale set starts and CPU utilization is 90 percent for 15 minutes. How many virtual machine instances will be running at 9:15 AM? 3
At 10:00 AM, the scale set has five virtual machine instances running and CPU utilization falls to less than 15 percent for 60 minutes. How many virtual machine instances will be running at 11:00 AM? 1
3 - The initial instance count is 2. CPU usage stays above the 75% scale-out threshold for 15 minutes (longer than the 10-minute duration set), so the scale set increases by 1 VM, resulting in 3 VMs running at 9:15 AM.
1 - The minimum number of VMs is set to 1. With CPU utilization below the 25% scale-in threshold for 60 minutes (well over the 10-minute duration required), the scale set decreases by 1 VM at a time until it reaches the minimum of 1 VM at 11:00 AM.
-----------------
2, 4, 5 (for the first part) - No scaling occurs at 2 because the CPU threshold is exceeded long enough to trigger a scale out. It only increases by 1 VM per scale action, so not 4 or 5.
2, 3, 4 (for the second part) - The scale set will continue scaling in, but cannot go below the minimum of 1 VM. Since the decrease is by 1 VM per scale action, it will not remain at higher counts if the low CPU persists.
Additional:
Default Scale in and Out Default Durations are 10 minutes with 5 minute cool down.
