Question : 67 Topic: 4

You plan to deploy several Azure virtual machines that will run Windows Server 2019 in a virtual machine scale set by using an Azure Resource Manager template.

You need to ensure that NGINX is available on all the virtual machines after they are deployed.
What should you use?

  • A..the New-AzConfigurationAssignment cmdlet
  • CorrectB..a Desired State Configuration (DSC) extension
  • C..Azure Active Directory (Azure AD) Application Proxy
  • D..Azure Application Insights
Suggested Answer:
B.. Desired State Configuration (DSC) extension

Explanations of Right Option

B.. Desired State Configuration (DSC) extension
The DSC extension is used to automate configuration tasks on Azure virtual machines, including the installation and configuration of applications like NGINX. By adding the DSC extension to your Azure Resource Manager template, you can ensure that every VM in the scale set will have NGINX installed after deployment.

Explain the Other Options

A..the New-AzConfigurationAssignment cmdlet
This cmdlet is used to assign Azure Policy or guest configuration assignments, not to install software like NGINX on VMs after deployment.
C..Azure Active Directory (Azure AD) Application Proxy
This service is for publishing on-premises applications to external users and does not assist with installing or configuring NGINX on VMs.
D..Azure Application Insights
This is a monitoring tool for applications and does not provide a method for configuring or installing software like NGINX on a VM.
