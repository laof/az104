You deploy an Azure Kubernetes Service (AKS) cluster named AKS1.
You need to deploy a YAML file to AKS1.
Solution: From Azure Cloud Shell, you run az aks.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B..No
B..No - Running az aks alone in Azure Cloud Shell does not deploy a YAML file to an AKS cluster. The az aks command is used for managing AKS clusters, not for deploying workloads. To deploy a YAML file, you need to use kubectl apply -f filename.yaml after configuring kubectl to connect to the AKS cluster.
-----------------
A..Yes - Incorrect because the az aks command does not apply or deploy YAML manifests to the cluster. You need kubectl for deploying YAML files to AKS.
