You have an Azure virtual machine named VM1 that runs Windows Server 2016.
You need to create an alert in Azure when more than two error events are logged to the System event log on VM1 within an hour.
Solution: You create an Azure Log Analytics workspace and configure the data settings. You add the Microsoft Monitoring Agent VM extension to VM1. You create an alert in Azure Monitor and specify the Log Analytics workspace as the source.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B..No
B..No – The solution as stated does not meet the goal, because it refers to the "Microsoft Monitoring Agent VM extension," which is not the correct, current name for the supported monitoring agent in Azure. The correct extension for new deployments is the "Azure Monitor Agent VM extension." Microsoft is deprecating the Microsoft Monitoring Agent (MMA) and recommends using the Azure Monitor Agent (AMA) and its corresponding extension.
Note: For the most accurate and supported solution, you should use the “Azure Monitor Agent VM extension” to connect the VM to Azure Monitor and the Log Analytics workspace.
