You have an Azure virtual machine named VM1 that runs Windows Server 2016.
You need to create an alert in Azure when more than two error events are logged to the System event log on VM1 within an hour.
Solution: You create an Azure Log Analytics workspace and configure the data settings. You install the Microsoft Monitoring Agent on VM1. You create an alert in
Azure Monitor and specify the Log Analytics workspace as the source.
Does this meet the goal?

  • CorrectA..Yes
  • B..No
Suggested Answer:
A..Yes
A..Yes – Installing the Microsoft Monitoring Agent on VM1, connecting it to a Log Analytics workspace, and creating an alert in Azure Monitor using that workspace as the source will allow you to create an alert when more than two error events are logged to the System event log within an hour. This approach is supported and meets the requirements, even though Microsoft recommends the newer Azure Monitor Agent for new deployments. The Microsoft Monitoring Agent is still valid and functional for this scenario.
-----------------
B..No
B..No – Incorrect because the Microsoft Monitoring Agent (MMA) is still supported for use with Log Analytics and Azure Monitor alerts, and the described steps are sufficient to meet the goal. The only scenario where this would be wrong is if the question strictly required the latest agent (Azure Monitor Agent), but the MMA still fulfills the requirements as described.
