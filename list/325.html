HOTSPOT -

You have an Azure subscription.
You need to use an Azure Resource Manager (ARM) template to create a virtual machine that will have multiple data disks.
How should you complete the template? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
First dropdown: copy
Second dropdown: [copyIndex('dataDisks')]
First dropdown: copy – In an Azure Resource Manager (ARM) template, to create multiple instances of a resource or property (such as multiple data disks), you use the copy element. This allows the template to iterate and create the specified number of data disks based on the parameter value.
Second dropdown: [copyIndex('dataDisks')] – The copyIndex() function returns the current iteration index of the copy loop. By specifying [copyIndex('dataDisks')], each data disk gets a unique Logical Unit Number (LUN), which is required for proper disk assignment in Azure VMs.
-----------------
[copyIndex] and [dependsOn] – Incorrect because [copyIndex] without an argument does not specify which copy loop’s index to use (important when using nested copies), and [dependsOn] is used for resource dependencies, not for iteration or LUN assignment in this context.
