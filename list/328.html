You have an Azure subscription named Subscription1 that contains the resources shown in the following table.
Subscription1 also includes a virtual network named VNET2. VM1 connects to a virtual network named VNET2 by using a network interface named NIC1.
You need to create a new network interface named NIC2 for VM1.
Solution: You create NIC2 in RG2 and West US.
Does this meet the goal?

  • CorrectA..Yes
  • B..No
Suggested Answer:
A..Yes
A..Yes – The solution meets the goal. In Azure, a resource group is simply a logical container and does not restrict the region of the resources it contains. The important requirement is that the network interface (NIC2) must be created in the same region as the virtual machine (VM1), which is West US. Creating NIC2 in RG2 and specifying West US as the location is valid and supported. RG2's original location (West Europe) does not prevent you from creating resources in other regions within it. As long as NIC2 is in the West US region, it can be attached to VM1.
-----------------
B..No
B..No – Incorrect because the resource group's location does not limit where the NIC can be created. The correct region for the NIC is West US, matching VM1, regardless of the resource group location.
