Question : 81 Topic: 4

You have an Azure subscription that contains three virtual machines named VM1, VM2, and VM3. All the virtual machines are in an availability set named AVSet1.

You need to scale up VM1 to a new virtual machine size, but the intended size is unavailable.
What should you do first?

  • A..Create a proximity placement group.
  • CorrectB..Deallocate VM1.
  • C..Convert AvSet1 into a managed availability set.
  • D..Shut down VM3 and VM3.
Suggested Answer:
B..Deallocate VM1
Explanation of right option:
Deallocate VM1 – Before resizing a virtual machine in an availability set, you must deallocate (stop) the VM. This releases the compute resources and allows you to resize the VM to a different size, including those sizes that may not currently be available due to capacity constraints. Resizing cannot be performed while the VM is running.
-----------------
A..Create a proximity placement group – Proximity placement groups are used to ensure that resources are physically close for low latency, but they do not affect resizing or VM size availability.
C..Convert AvSet1 into a managed availability set – Converting to a managed availability set does not address the issue of resizing or resource availability for a specific size.
D..Shut down VM3 and VM3 – Shutting down other VMs in the set does not help with resizing VM1; you must deallocate the specific VM you intend to resize.
