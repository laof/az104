You are creating an Azure Kubernetes Services (AKS) cluster as shown in the following exhibit.
Use the drop-down menus to select the answer choice that completes each statement based on the information presented in the graphic.
NOTE: Each correct selection is worth one point.

Suggested Answer:
To ensure that you can create Windows containers in AKS1, you must modify the Network configuration setting.
To ensure that you can integrate AKS1 with an Azure container registry, you must modify the Authentication method setting.
Explanation:
The AKS cluster must use the Azure CNI network plugin (not Kubenet) to support Windows node pools, so the network configuration must be changed.
For Azure Container Registry (ACR) integration, you must configure the Authentication method (such as a service principal or managed identity) for AKS, and then assign the AcrPull role to that identity. This allows the AKS cluster nodes to authenticate to ACR and pull container images securely.
