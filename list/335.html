HOTSPOT -

You have an Azure subscription that contains an Azure Kubernetes Service (AKS) cluster named Cluster1. Cluster1 hosts a node pool named Pool1 that has four nodes.
You need to perform a coordinated upgrade of Cluster1. The solution must meet the following requirements:
• Deploy two new nodes to perform the upgrade.
• Minimize costs.
How should you complete the command? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.

Suggested Answer:
update -n pool1 -g RG1 --cluster-name cluster1 --max-surge 2
Explanation of right options:
update: This command is used to change the configuration of an existing node pool, which is required for performing a coordinated upgrade.
--max-surge 2: This parameter specifies that during the upgrade, up to two additional nodes can be temporarily added. This meets the requirement to "deploy two new nodes to perform the upgrade" while minimizing costs by limiting the number of surge nodes.
-----------------
add: This is used to create a new node pool, not to change or upgrade an existing one.
get-updates: This is not a valid az aks nodepool command; it is not used for upgrades or scaling.
scale: This command changes the number of nodes in the pool, not the rolling upgrade behavior.
updates: This is not a valid az aks nodepool command; updates are performed with the upgrade command.
--max-count 2: This parameter sets the upper limit of nodes in the pool, not the number of extra nodes during an upgrade.
--max-pods 2: This parameter limits the number of pods per node, not the number of nodes during upgrade.
--node-count 2: This parameter sets the total number of nodes in the pool, which would not satisfy the requirement to temporarily add two nodes during an upgrade.
