HOTSPOT -

You have an Azure container registry named contoso2023 as shown in the following exhibit.
You need to enable contoso2023 to use a dedicated data endpoint.
Which two settings should you configure for contoso2023? To answer, select the appropriate settings in the answer area.
NOTE: Each correct answer is worth one point.

Suggested Answer:>
Networking
Geo-replications
-----------------
Explanation of Correct Options:
Networking: To enable a dedicated data endpoint for Azure Container Registry, you must configure the Networking settings. This allows you to enable private endpoints and control data access using a dedicated endpoint.
Geo-replications: Enabling geo-replication can allow you to configure dedicated regional endpoints for your registry, which supports the use of dedicated data endpoints in different Azure regions.
Explanation of Other Options:
Access keys: This option is used to manage authentication credentials, not to configure endpoints.
Encryption: This setting is for configuring encryption at rest for the registry, not for endpoint management.
Identity: This option is related to managed identities for Azure resources, not endpoint configuration.
Microsoft Defender for Cloud: This integrates security monitoring and is not related to endpoint settings.
Properties: This only displays registry properties and does not allow endpoint configuration.
Locks: Locks are for preventing accidental deletion or modification of the registry, not endpoint management.
Repositories: This is for managing container images, not for endpoints.
Webhooks: Webhooks are used to trigger events and are unrelated to data endpoints.
Tasks: Tasks automate container image builds and deployments, not endpoints.
Connected registries (Preview): This is for connecting registries across environments, not for dedicated data endpoints.
Cache: Caching is for performance and availability, not for configuring dedicated endpoints.
