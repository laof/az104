Question : 90 Topic: 4

You have an Azure subscription that has the public IP addresses shown in the following table.

You plan to deploy an Instance of Azure Firewall Premium named FW1.
Which IP addresses can you use?

  • A..IP2 only
  • B..IP1 and IP2 only
  • CorrectC..IP1, IP2, and IP5 only
  • D..IP1, IP2, IP4, and IP5 only
Suggested Answer:
C. IP1, IP2, and IP5 only
-----------------
Explanation of Correct Option:
Azure Firewall Premium requires that public IP addresses must use the Standard SKU and be Static assignment. Both IPv4 and IPv6 addresses are supported, and the tier can be either Regional or Global. Therefore, IP1 (Standard, Regional, Static, IPv4), IP2 (Standard, Global, Static, IPv4), and IP5 (Standard, Regional, Static, IPv6) meet these requirements.
Explanation of Other Options:
A. IP2 only
IP2 is eligible, but this option ignores the other valid Standard SKU, Static addresses (IP1 and IP5), so it's incomplete.
B. IP1 and IP2 only
While both IP1 and IP2 are valid, this option omits IP5, which is also valid as it is Standard SKU, Static, and Regional (IPv6 is supported).
D. IP1, IP2, IP4, and IP5 only
IP4 uses the Basic SKU, which is not supported by Azure Firewall Premium, so including it makes this option incorrect.
Additional:
Azure Firewall only supports Static IPv4 addresses, and does not support Dynamic IPv4, Dynamic IPv6, or Static IPv6. This is accurate for Azure Firewall Standard.
However, Azure Firewall Premium (which is referenced in the previous question) now supports Static IPv6 addresses as well, according to Azure updates and documentation. Therefore, for Azure Firewall Premium, you can use both Static IPv4 and Static IPv6 addresses.
