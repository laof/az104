Question : 35 Topic: 1

Your company's Azure subscription includes Azure virtual machines (VMs) that run Windows Server 2016.

One of the VMs is backed up every day using Azure Backup Instant Restore.
When the VM becomes infected with data encrypting ransomware, you are required to restore the VM.
Which of the following actions should you take?

  • A..You should restore the VM after deleting the infected VM.
  • B..You should restore the VM to any VM within the company's subscription.
  • CorrectC..You should restore the VM to a new Azure VM.
  • D..You should restore the VM to an on-premise Windows device.
Suggested Answer:
C..You should restore the VM to a new Azure VM.
-----------------
Explanation of Correct Option:
C..You should restore the VM to a new Azure VM. — This is the correct answer. When a VM is infected with ransomware, the safest and most effective recovery method is to restore the VM to a new instance using Azure Backup. This avoids any risk of reintroducing the malware from the compromised VM and ensures a clean environment.
-----------------
Explanation of Other Options:
A..You should restore the VM after deleting the infected VM. — Incorrect. While deleting the infected VM may seem logical, it's not necessary and could result in loss of diagnostic data. Restoring to a new VM is safer and allows for side-by-side comparison or investigation.
B..You should restore the VM to any VM within the company's subscription. — Incorrect. You cannot overwrite another VM with a restore; Azure Backup restores to a new VM or the original one.
D..You should restore the VM to an on-premise Windows device. — Incorrect. Azure Backup for VMs is designed to restore within Azure. Restoring to on-premises would require a different backup and recovery approach.
