Question : 98 Topic: 4

You have an Azure AD tenant named contoso.com.

You have an Azure subscription that contains an Azure App Service web app named App1 and an Azure key vault named KV1. KV1 contains a wildcard certificate for contoso.com.
You have a user named [email protected] that is assigned the Owner role for App1 and KV1.
You need to configure App1 to use the wildcard certificate of KV1.
What should you do first?

  • A..Create an access policy for KV1 and assign the Microsoft Azure App Service principal to the policy.
  • CorrectB..Assign a managed user identity to App1.
  • C..Configure KV1 to use the role-based access control (RBAC) authorization system.
  • D..Create an access policy for KV1 and assign the policy to User1.
Suggested Answer:
B..Assign a managed user identity to App1. (Assigning a managed identity to App1 is the first step so the app can securely access Azure Key Vault resources such as certificates. The managed identity will later be granted permission to retrieve the certificate from KV1.)
-----------------
A..Create an access policy for KV1 and assign the Microsoft Azure App Service principal to the policy. (You must assign a managed identity to App1 first; only then can you assign the identity to an access policy. The "Microsoft Azure App Service principal" is not directly used—App1's managed identity is.)
C..Configure KV1 to use the role-based access control (RBAC) authorization system. (Configuring RBAC is not the first step. Before you can grant permissions using RBAC or access policies, App1 needs a managed identity.)
D..Create an access policy for KV1 and assign the policy to User1. (Assigning the policy to User1 only allows the user to access the Key Vault, not App1. The requirement is for App1—an application—not a user—to access the certificate, so this does not meet the scenario.)
