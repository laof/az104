Question : 107 Topic: 4

You have an Azure subscription.

You plan to create an Azure container registry named ContReg1.
You need to ensure that you can push and pull signed images for ContReg1.
What should you do for ContReg1?

  • A..Enable encryption by using a customer-managed key.
  • B..Create a connected registry.
  • C..Add a token.
  • CorrectD..Enable content trust.
Suggested Answer:
D..Enable content trust.
-----------------
Explanation of right option: D..Enable content trust Yes — Content trust in Azure Container Registry enables image signing and verification. Enabling content trust ensures that only signed images can be pushed to and pulled from the registry, which is required for securing the supply chain and verifying image integrity.
Other options explained:
A..Enable encryption by using a customer-managed key — This option protects data at rest but does not relate to image signing or verification.
B..Create a connected registry — Connected registries are used for edge scenarios, not for image signing.
C..Add a token — Adding a token allows for scoped authentication but does not enable image signing or verification.
