Question : 109 Topic: 4

You plan to deploy several Azure virtual machines that will run Windows Server 2022 in a virtual machine scale set by using an Azure Resource Manager template.

You need to ensure that NGINX is available on all the virtual machines after they are deployed.
What should you use?

  • CorrectA..Azure Custom Script Extension
  • B..Deployment Center in Azure App Service
  • C..Microsoft Entra Application Proxy
  • D..the Publish-AzVMDscConfiguration cmdlet
Suggested Answer:
A..Azure Custom Script Extension
-----------------
Explanation of right option:
A..Azure Custom Script Extension — The Azure Custom Script Extension allows you to run scripts (such as installing NGINX) on Azure VMs after deployment. It is the standard way to automate configuration steps like software installation for all VMs in a scale set deployed via ARM templates.
Other options explained:
B..Deployment Center in Azure App Service — This is for deploying app code to Azure App Service, not for configuring VMs or scale sets.
C..Microsoft Entra Application Proxy — This is used for secure remote access to applications, not for software deployment on VMs.
D..the Publish-AzVMDscConfiguration cmdlet — This cmdlet publishes DSC configurations but does not directly install software like NGINX on all VMs in a scale set; Custom Script Extension is simpler and more direct for this use case.
