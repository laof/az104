You have an Azure container registry named Registry1 that contains an image named image1.
You receive an error message when you attempt to deploy a container instance by using image1.
You need to be able to deploy a container instance by using image1.
Solution: You assign the AcrPull role to ACR-Tasks-Network for Registry1.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B..No
-----------------
Explanation of right option:
B..NoAcrPull role to ACR-Tasks-Network only grants pull permissions to that specific identity. To deploy a container instance from Registry1, the identity used by the Azure Container Instance resource must have AcrPull on Registry1, not ACR-Tasks-Network. Therefore, this solution does not meet the goal.
