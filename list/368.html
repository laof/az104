Question : 116 Topic: 4

You have a Standard Azure App Service plan named Plan1.

You need to ensure that Plan1 will scale automatically when the CPU usage of the web app exceeds 80 percent.
What should you select for Plan1?

  • A..Automatic in the Scale out method settings
  • CorrectB..Rules Based in the Scale out method settings
  • C..Premium P1 in the Scale up (App Service plan) settings
  • D..Standard S1 in the Scale up (App Service plan) settings
  • E..Manual in the Scale out method settings
Suggested Answer:
B..Rules Based in the Scale out method settings
Explanation of right option:
B..Rules Based in the Scale out method settings — By selecting Rules Based in the Scale out method settings, you can configure custom rules to scale out automatically based on metrics such as CPU usage, which allows Plan1 to scale when CPU exceeds 80%.
Other options explained:
A..Automatic in the Scale out method settings — There is no Automatic option for custom rules in Standard plans; automatic is not available here.
C..Premium P1 in the Scale up (App Service plan) settings — This changes the pricing tier, not the scaling method.
D..Standard S1 in the Scale up (App Service plan) settings — This also refers to the pricing tier, not how scaling is configured.
E..Manual in the Scale out method settings — Manual scaling requires you to set the instance count yourself and does not scale automatically.
