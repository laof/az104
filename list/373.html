Which PowerShell cmdlet and key should you use?

  • A..Set-AzVMDiskEncryptionExtension and Key2.
  • B..Set-AzDiskEncryptionKey and Key2.
  • C..Set-AzDiskDiskEncryptionKey and Key1.
  • CorrectD..Set-AzVMDiskEncryptionExtension and Key1.
Suggested Answer:
D. Set-AzVMDiskEncryptionExtension and Key1.
Explanation of right option:
D. Set-AzVMDiskEncryptionExtension and Key1. — To meet the technical requirements for using a Key Encryption Key (KEK) with Azure Disk Encryption on virtual machines, you must use the Set-AzVMDiskEncryptionExtension PowerShell cmdlet. This cmdlet is specifically designed for enabling disk encryption on Azure VMs and allows you to specify a KEK from Azure Key Vault. For Azure Disk Encryption, only RSA keys (such as Key1, which has a key size of 4096) are supported as KEKs. EC (Elliptic Curve) keys like Key2 are not supported as KEKs.
-----------------
Other options explained:
A. Set-AzVMDiskEncryptionExtension and Key2. — Incorrect. While the cmdlet is correct, Key2 is an EC key, which is not supported as a KEK for Azure Disk Encryption.
B. Set-AzDiskEncryptionKey and Key2. — Incorrect. This is not a valid PowerShell cmdlet for VM disk encryption, and Key2 (EC) is not a supported KEK type.
C. Set-AzDiskDiskEncryptionKey and Key1. — Incorrect. This cmdlet does not exist; the correct cmdlet for VM disk encryption is Set-AzVMDiskEncryptionExtension.
