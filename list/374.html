HOTSPOT -

You have an Azure subscription named Sub1.
You plan to deploy a multi-tiered application that will contain the tiers shown in the following table.
You need to recommend a networking solution to meet the following requirements:
✑ Ensure that communication between the web servers and the business logic tier spreads equally across the virtual machines.
✑ Protect the web servers from SQL injection attacks.
Which Azure resource should you recommend for each requirement? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
1. Ensure that communication between the web servers and the business logic tier spreads equally across the virtual machines:
an internal load balancer
2. Protect the web servers from SQL injection attacks:
an application gateway that uses the WAF tier
Explanation of right options:
Ensure that communication between the web servers and the business logic tier spreads equally across the virtual machines: an internal load balancer ✔ — An internal load balancer distributes network traffic among virtual machines in a private network, ensuring even distribution and high availability for the business logic tier, which is not exposed to the Internet.
Protect the web servers from SQL injection attacks: an application gateway that uses the WAF tier ✔ — An Application Gateway with the WAF (Web Application Firewall) tier provides centralized protection against common web vulnerabilities such as SQL injection attacks. WAF inspects incoming traffic and blocks malicious requests before they reach your web servers.
-----------------
Other options explained:
an application gateway that uses the Standard tier — Incorrect. The Standard tier does not provide WAF capabilities for SQL injection protection.
a network security group (NSG) — Incorrect. NSGs control network traffic at the subnet or NIC level but do not provide load balancing or application-layer protection against SQL injection.
a public load balancer — Incorrect. A public load balancer distributes traffic from the Internet and is not suitable for internal-only communication between tiers.
