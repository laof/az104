Your company has three offices. The offices are located in Miami, Los Angeles, and New York. Each office contains datacenter.

You have an Azure subscription that contains resources in the East US and West US Azure regions. Each region contains a virtual network. The virtual networks are peered.
You need to connect the datacenters to the subscription. The solution must minimize network latency between the datacenters.
What should you create?

  • A..three Azure Application Gateways and one On-premises data gateway
  • CorrectB..three virtual hubs and one virtual WAN
  • C..three virtual WANs and one virtual hub
  • D..three On-premises data gateways and one Azure Application Gateway
Suggested Answer:
B. three virtual hubs and one virtual WAN.
— Azure Virtual WAN is a networking service that provides optimized and automated branch connectivity to, and through, Azure. By creating three virtual hubs (one for each datacenter location) and connecting them through a single virtual WAN, you ensure a low-latency, optimized, and scalable network topology. This approach is designed for connecting multiple on-premises sites to Azure and to each other with minimal latency.
-----------------
Other options explained:
A. three Azure Application Gateways and one On-premises data gateway — Incorrect. Application Gateways are used for web traffic load balancing, not for connecting datacenters or minimizing latency.
C. three virtual WANs and one virtual hub — Incorrect. A virtual WAN is a global resource that can contain multiple hubs; creating three WANs with a single hub does not reflect the correct architecture for multi-site connectivity.
D. three On-premises data gateways and one Azure Application Gateway — Incorrect. On-premises data gateways are used for Power BI and other Azure data services, not for network connectivity. Application Gateway does not serve as a WAN or VPN solution.
Additional:
Virtual WAN: The virtual WAN resource represents a virtual overlay of your Azure network and is a collection of multiple resources. It contains links to all your virtual hubs that you would like to have within the virtual WAN. Virtual WANs are isolated from each other and can't contain a common hub. Virtual hubs in different virtual WANs don't communicate with each other.
