You plan to create new inbound NAT rules that meet the following requirements:
✑ Provide Remote Desktop access to VM1 from the internet by using port 3389.
✑ Provide Remote Desktop access to VM2 from the internet by using port 3389.
What should you create on LB1 before you can create the new inbound NAT rules?

  • A..a frontend IP address
  • CorrectB..a load balancing rule
  • C..a health probe
  • D..a backend pool
Suggested Answer:
B. a load balancing rule
Explanation of right option:
Option B (a load balancing rule) is selected because, among the available options, it is the closest to a selectable answer, even though what you actually need to add are inbound NAT rules. The correct configuration for providing RDP access to VM1 and VM2 involves adding NAT rules, but since "NAT rule" is not a listed option, option B is chosen by process of elimination. Note that load balancing rules and NAT rules are different features. Inbound NAT rules can coexist with load balancing rules on the same frontend IP of the load balancer, but you do not need to add a new load balancing rule to use NAT rules. The actual action should be adding new NAT rules, but the options do not provide this choice.
-----------------
Explanation of other options:
A. a frontend IP address — Not needed because the frontend IP address already exists, as confirmed by the prior discussion and the presence of a load balancing rule.
B. a load balancing rule — Selected only because NAT rules are not an option. In reality, you do NOT need to add or modify a load balancing rule to create NAT rules.
C. a health probe — Not required for NAT rules. Health probes monitor VM health for load balancing, not for NAT rules.
D. a backend pool — Not required for NAT rules. Backend pools are used for load balancing rules, but NAT rules map frontend ports to individual VM NICs and do not require a backend pool.
