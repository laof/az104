HOTSPOT -

You have Azure virtual machines that run Windows Server 2019 and are configured as shown in the following table.
You create a private Azure DNS zone named adatum.com. You configure the adatum.com zone to allow auto registration from VNET1.
Which A records will be added to the adatum.com zone for each virtual machine? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
For VM1: Private IP address only — VM1 is in VNET1, and its Windows Server DNS suffix is set to adatum.com. Since the private Azure DNS zone adatum.com is configured to allow auto registration from VNET1, VM1 will automatically register its private IP (10.1.0.4) as an A record in adatum.com. Public IP addresses are not registered in private DNS zones.
For VM2: None — Although VM2 is also in VNET1, its DNS suffix is set to contoso.com, not adatum.com. Only VMs with a DNS suffix that matches the DNS zone (adatum.com) will auto-register A records in that zone. Therefore, VM2 will not register any A record in adatum.com.
-----------------
Explanation of other options:
Public IP address only — Public IPs are never registered in a private DNS zone. Azure private DNS zones only register private IP addresses for auto-registration.
Private IP address and public IP address — Only the private IP is registered in the private DNS zone, never both.
None (for VM1) — Incorrect, because VM1 meets the criteria for auto-registration (VNET1, DNS suffix adatum.com).
Private IP address only/Public IP address only/Private IP address and public IP address (for VM2) — Incorrect, because VM2's DNS suffix does not match adatum.com, so no record will be registered for VM2 in that zone.
