You have the Azure virtual networks shown in the following table.

To which virtual networks can you establish a peering connection from VNet1?

  • A..VNet2 andVNet3 only
  • B..VNet2 only
  • CorrectC..VNet3 and VNet4 only
  • D..VNet2, VNet3, and VNet4
Suggested Answer:
C. VNet3 and VNet4 only
Explanation of right option:
Option C is correct because you cannot peer virtual networks that have overlapping address spaces. VNet1 uses 10.11.0.0/16. VNet2 uses 10.11.0.0/17, which overlaps with VNet1, so peering is not possible. VNet3 (10.10.0.0/22) and VNet4 (192.168.16.0/22) do not overlap with VNet1, so peering is allowed.
-----------------
Explanation of other options:
A. VNet2 and VNet3 only — Incorrect because VNet2's address space overlaps with VNet1, so peering is not allowed.
B. VNet2 only — Incorrect because VNet2's address space overlaps, and VNet3 and VNet4 are actually peering-capable.
D. VNet2, VNet3, and VNet4 — Incorrect because VNet2's address space overlaps with VNet1, so it cannot be peered.
