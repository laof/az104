You have an Azure subscription that contains a virtual network named VNet1. VNet1 contains four subnets named Gateway, Perimeter, NVA, and Production.

The NVA subnet contains two network virtual appliances (NVAs) that will perform network traffic inspection between the Perimeter subnet and the Production subnet.
You need to implement an Azure load balancer for the NVAs. The solution must meet the following requirements:
✑ The NVAs must run in an active-active configuration that uses automatic failover.
✑ The load balancer must load balance traffic to two services on the Production subnet. The services have different IP addresses.
Which three actions should you perform? Each correct answer presents part of the solution.
NOTE: Each correct selection is worth one point.

  • A..Deploy a basic load balancer
  • CorrectB..Deploy a standard load balancer
  • CorrectC..Add two load balancing rules that have HA Ports and Floating IP enabled
  • D..Add two load balancing rules that have HA Ports enabled and Floating IP disabled
  • E..Add a frontend IP configuration, a backend pool, and a health probe
  • CorrectF..Add a frontend IP configuration, two backend pools, and a health probe
Suggested Answer:
B. Deploy a standard load balancer
C. Add two load balancing rules that have HA Ports and Floating IP enabled
E. Add a frontend IP configuration, a backend pool, and a health probe
Explanation of right options:
B. Deploy a standard load balancer — Standard load balancers support high availability and advanced features like HA Ports and Floating IP, which are required for active-active NVA setups and automatic failover. Basic load balancers do not support these features.
C. Add two load balancing rules that have HA Ports and Floating IP enabled — HA Ports are needed for scenarios requiring all-port load balancing, typical for NVAs. Floating IP enables direct server return, which is required for some NVA architectures and for supporting active-active failover.
E. Add a frontend IP configuration, a backend pool, and a health probe — These are the minimum components for a load balancer to distribute traffic and monitor the health of backend NVAs.
-----------------
Explanation of other options:
A. Deploy a basic load balancer — Incorrect because basic load balancers do not support HA Ports or Floating IP features required for this scenario.
D. Add two load balancing rules that have HA Ports enabled and Floating IP disabled — Incorrect because Floating IP must be enabled for active-active NVA and failover scenarios.
F. Add a frontend IP configuration, two backend pools, and a health probe — Incorrect because you only need one backend pool (both NVAs in the same backend pool) to achieve load balancing across both NVAs.
Additional:
You can similarly configure your load balancer to use a load-balancing rule with HA Port with a single front end by setting the Floating IP to Enabled.
With this configuration, you can add more floating IP load-balancing rules and/or a public load balancer. However, you can't use a nonfloating IP, HA-ports load-balancing configuration on top of this configuration.
