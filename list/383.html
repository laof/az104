HOTSPOT -

You have an Azure subscription. The subscription contains virtual machines that run Windows Server 2016 and are configured as shown in the following table.
You create a public Azure DNS zone named adatum.com and a private Azure DNS zone named contoso.com.
You create a virtual network link for contoso.com as shown in the following exhibit.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
When VM1 starts, a record for VM1 is added to the contoso.com DNS zone. — Yes — VM1 has its DNS suffix set to contoso.com and VNET2 is linked to the private contoso.com DNS zone with auto registration enabled, so a record for VM1 will be automatically added to the contoso.com DNS zone.
-----------------
When VM2 starts, a record for VM2 is added to the contoso.com DNS zone. — No — VM2 does not have any DNS suffix configured, so it will not be automatically registered in the contoso.com private DNS zone.
When VM3 starts, a record for VM3 is added to the adatum.com DNS zone. — No — VM3's DNS suffix is adatum.com, but only a public DNS zone named adatum.com was created, and auto registration does not apply to public DNS zones, so VM3 will not be auto-registered in adatum.com.
