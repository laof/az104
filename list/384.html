You have an Azure subscription that contains the resources in the following table.

To which subnets can you apply NSG1?

  • A..the subnets on VNet1 only
  • B..the subnets on VNet2 and VNet3 only
  • C..the subnets on VNet2 only
  • D..the subnets on VNet3 only
  • CorrectE..the subnets on VNet1, VNet2, and VNet3
Suggested Answer:
E..the subnets on VNet1, VNet2, and VNet3 — You can apply a Network Security Group (NSG) to any subnet in any virtual network within the same Azure subscription, regardless of Azure region or resource group. NSGs are not restricted by region or resource group; they are only scoped to the subscription level.
-----------------
A..the subnets on VNet1 only — Incorrect. NSGs are not restricted to a single virtual network or resource group. You can apply NSGs to subnets in any VNet within the subscription.
B..the subnets on VNet2 and VNet3 only — Incorrect. NSGs are not limited to these VNets. They can be applied to all VNets in the subscription.
C..the subnets on VNet2 only — Incorrect. There is no restriction that limits NSG application to only VNet2.
D..the subnets on VNet3 only — Incorrect. NSGs are not limited to a single VNet or region, and can be used across all VNets in the subscription.
