HOTSPOT -

You have an Azure subscription that contains the resource groups shown in the following table.
RG1 contains the resources shown in the following table.
VM1 is running and connects to NIC1 and Disk1. NIC1 connects to VNET1.
RG2 contains a public IP address named IP2 that is in the East US location. IP2 is not assigned to a virtual machine.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
You can move storage1 to RG2. Yes — You can move a storage account to another resource group as long as both resource groups are in the same subscription, even if they are in different regions. The storage account will retain its original region (West US) after the move.
You can move NIC1 to RG2. Yes — You can move a network interface to a different resource group within the same subscription, regardless of region. The NIC will remain in its original region (West US) after the move.
If you move IP2 to RG1, the location of IP2 will change. No — Moving a public IP address to another resource group does not change its region. The resource location (East US) stays the same even if you move IP2 to RG1.
-----------------
You can move storage1 to RG2. No — Incorrect. The move is allowed and does not change the region.
You can move NIC1 to RG2. No — Incorrect. The move is allowed and does not change the region.
If you move IP2 to RG1, the location of IP2 will change. Yes — Incorrect. The region stays the same after moving a resource between resource groups.
Additional:
Each NIC attached to a VM must exist in the same location and subscription as the VM. Each NIC must be connected to a VNet that exists in the same Azure location and subscription as the NIC. You can change the subnet a VM is connected to after it's created. You can't change the virtual network.
