Question : 14 Topic: 5

You have an Azure web app named webapp1.

You have a virtual network named VNET1 and an Azure virtual machine named VM1 that hosts a MySQL database. VM1 connects to VNET1.
You need to ensure that webapp1 can access the data hosted on VM1.
What should you do?

  • A..Deploy an internal load balancer
  • B..Peer VNET1 to another virtual network
  • CorrectC..Connect webapp1 to VNET1
  • D..Deploy an Azure Application Gateway
Suggested Answer:
C. Connect webapp1 to VNET1
This is the correct option. To allow an Azure web app (webapp1) to access resources (such as a MySQL database) hosted on a VM (VM1) within a virtual network (VNET1), you must use Azure's Virtual Network Integration feature for App Service. This enables private communication between the web app and the VM, so the web app can securely access the MySQL database without exposing it to the public internet.
-----------------
A. Deploy an internal load balancer
This is not required for this scenario. An internal load balancer is used to distribute traffic among multiple VMs within a virtual network, but it does not connect a web app to a VNet.
B. Peer VNET1 to another virtual network
Peering VNET1 with another VNet is not necessary. The requirement is for the web app to access VM1 in VNET1, not to establish connectivity between VNets.
D. Deploy an Azure Application Gateway
Application Gateway is used for HTTP/HTTPS load balancing and web traffic management. It does not provide the required network-level integration to allow the web app to access the VM directly within the VNet.
