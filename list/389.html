You have five Azure virtual machines that run Windows Server 2016. The virtual machines are configured as web servers.

You have an Azure load balancer named LB1 that provides load balancing services for the virtual machines.
You need to ensure that visitors are serviced by the same web server for each request.
What should you configure?

  • A..Floating IP (direct server return) to Disabled
  • B..Session persistence to None
  • C..Floating IP (direct server return) to Enabled
  • CorrectD..Session persistence to Client IP
Suggested Answer:
D. Session persistence to Client IP
This is the correct option. Setting Session persistence to Client IP ensures that requests from the same client IP address are always directed to the same backend server. This is also known as "sticky sessions" and is required when you want a user's session state to remain on a single web server during their visit.
-----------------
A. Floating IP (direct server return) to Disabled
This setting controls whether the load balancer uses direct server return, not session persistence. It does not guarantee users will be directed to the same server.
B. Session persistence to None
With session persistence set to None, requests from the same client could be routed to any backend server, which does not meet the requirement.
C. Floating IP (direct server return) to Enabled
This setting is for specific scenarios like high-performance and port reuse, and does not control session persistence or sticky sessions.
