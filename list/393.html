HOTSPOT -

You have a virtual network named VNet1 that has the configuration shown in the following exhibit.
Use the drop-down menus to select the answer choice that completes each statement based on the information presented in the graphic.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Before a virtual machine on VNet1 can receive an IP address from 192.168.1.0/24, you must first add an address space
Before a virtual machine on VNet1 can receive an IP address from 10.2.1.0/24, you must first add a subnet
-----------------
Explanation of right options:
add an address space: VNet1 currently only has the address space 10.2.0.0/16 (see Image 1). To allow an IP from 192.168.1.0/24, you must first add 192.168.1.0/24 to the virtual network's address space.
add a subnet: The address space 10.2.0.0/16 already includes 10.2.1.0/24, but there is no subnet defined for 10.2.1.0/24. You must add a subnet with this range before assigning an IP from this range.
-----------------
Other options:
add a network interface: This is required for VM connectivity, but does not affect the availability of IP ranges within the VNet.
delete a subnet: This would remove existing subnets and is not required to add or use new IP ranges.
delete an address space: This would remove address spaces and does not enable assignment of new IP ranges.
