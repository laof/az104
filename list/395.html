You have an Azure subscription that contains the resources shown in the following table.

The Not allowed resource types Azure policy that has policy enforcement enabled is assigned to RG1 and uses the following parameters:
Microsoft.Network/virtualNetworks
Microsoft.Compute/virtualMachines
In RG1, you need to create a new virtual machine named VM2, and then connect VM2 to VNET1.
What should you do first?

  • CorrectA..Remove Microsoft.Compute/virtualMachines from the policy.
  • B..Create an Azure Resource Manager template
  • C..Add a subnet to VNET1.
  • D..Remove Microsoft.Network/virtualNetworks from the policy.
Suggested Answer:
A. Remove Microsoft.Compute/virtualMachines from the policy.
-----------------
Explanation of right option:
A. Remove Microsoft.Compute/virtualMachines from the policy.: The current Azure policy explicitly blocks the creation of both virtual networks and virtual machines in RG1. To create a new VM (VM2) in RG1, you must first remove the restriction on Microsoft.Compute/virtualMachines in the policy, otherwise the creation will be denied by policy enforcement.
-----------------
Other options:
B. Create an Azure Resource Manager template: This is a deployment method, but does not address the policy restriction that prevents VM creation.
C. Add a subnet to VNET1: This is not needed for the initial step, since the main blocker is the policy preventing VM creation.
D. Remove Microsoft.Network/virtualNetworks from the policy: This would allow you to create virtual networks, but the question requires you to create a virtual machine, so the first step must be to allow VM creation.
