Your company has an Azure subscription named Subscription1.

The company also has two on-premises servers named Server1 and Server2 that run Windows Server 2016. Server1 is configured as a DNS server that has a primary DNS zone named adatum.com. Adatum.com contains 1,000 DNS records.
You manage Server1 and Subscription1 from Server2. Server2 has the following tools installed:
✑ The DNS Manager console
✑ Azure PowerShell
✑ Azure CLI 2.0
You need to move the adatum.com zone to an Azure DNS zone in Subscription1. The solution must minimize administrative effort.
What should you use?

  • A..Azure CLI
  • CorrectB..Azure PowerShell
  • C..the Azure portal
  • D..the DNS Manager console
Suggested Answer:
B. Azure PowerShell: Azure PowerShell provides cmdlets that can automate the export and import of DNS zones and records. You can use the Export-DnsServerZone cmdlet to export the DNS zone from Server1, and then use Azure PowerShell cmdlets (like New-AzDnsZone and New-AzDnsRecordSet) to create the zone and import records into Azure DNS. This approach is efficient for handling a large number of DNS records (1,000 in this case) and minimizes manual effort.
-----------------
Other options:
A. Azure CLI: While Azure CLI can manage DNS zones, it does not provide built-in commands to import large sets of DNS records directly from Windows DNS exports; it would require more scripting and manual processing.
C. the Azure portal: The Azure portal is not suitable for bulk DNS record import/export; it is only practical for small numbers of records and manual entry.
D. the DNS Manager console: The DNS Manager console cannot move or import DNS zones directly into Azure DNS; it is meant for managing on-premises Windows DNS servers.
