You have a public load balancer that balances ports 80 and 443 across three virtual machines named VM1, VM2, and VM3.

You need to direct all the Remote Desktop Protocol (RDP) connections to VM3 only.
What should you configure?

  • CorrectA..an inbound NAT rule
  • B..a new public load balancer for VM3
  • C..a frontend IP configuration
  • D..a load balancing rule
Suggested Answer:
A. an inbound NAT rule
-----------------
Explanation of right option:
A. an inbound NAT rule: An inbound NAT (Network Address Translation) rule on the public load balancer allows you to direct RDP (typically TCP port 3389) traffic from the public IP to a specific backend VM (VM3). This way, only VM3 will receive RDP connections, even though other traffic (ports 80 and 443) is load balanced across all three VMs.
-----------------
Other options:
B. a new public load balancer for VM3: Unnecessary, as you can use an inbound NAT rule on the existing load balancer.
C. a frontend IP configuration: This defines the public IP endpoint, but does not control how RDP traffic is routed.
D. a load balancing rule: This would distribute RDP connections across all VMs, rather than directing them to VM3 only.
