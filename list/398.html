HOTSPOT -

You have an Azure subscription named Subscription1 that contains the virtual networks in the following table.
Subscription1 contains the virtual machines in the following table.
In Subscription1, you create a load balancer that has the following configurations:
✑ Name: LB1
✑ SKU: Basic
✑ Type: Internal
✑ Subnet: Subnet12
✑ Virtual network: VNET1
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
LB1 can balance the traffic between VM1 and VM2. Yes
Explanation of right options: LB1 can balance the traffic between VM1 and VM2 if both are in the same VNet as LB1 and either both are in the same availability set or both are standalone VMs, according to Basic Load Balancer backend pool rules. Subnet does not restrict backend pool membership as long as all VMs are in the same VNet.
Other options: If VM1 and VM2 are in different VNets, different availability sets (for Basic SKU), or mixed (one in availability set, one standalone), then LB1 cannot balance traffic between them.
-----------------
LB1 can balance the traffic between VM3 and VM4. Yes
Explanation of right options: LB1 can balance the traffic between VM3 and VM4 if both are in the same VNet as LB1 and both satisfy the backend pool membership requirements (same availability set or both standalone). Subnet is not a restriction as long as they are in the same VNet.
Other options: If VM3 and VM4 are not in the same VNet as LB1, or do not meet Basic Load Balancer backend pool requirements, LB1 cannot balance their traffic.
-----------------
LB1 can balance the traffic between VM5 and VM6. Yes
Explanation of right options: LB1 can balance the traffic between VM5 and VM6 if both are in the same VNet as LB1 and both are standalone VMs (i.e., not in any availability set or scale set), since Basic Load Balancer backend pool can contain only standalone VMs if not using availability sets or scale sets.
Other options: If VM5 and VM6 are not both standalone, or not in the same VNet as LB1, or mixed (one standalone, one in an availability set), LB1 cannot balance their traffic.
