DRAG DROP -

You have an on-premises network that you plan to connect to Azure by using a site-so-site VPN.
In Azure, you have an Azure virtual network named VNet1 that uses an address space of 10.0.0.0/16 VNet1 contains a subnet named Subnet1 that uses an address space of 10.0.0.0/24.
You need to create a site-to-site VPN to Azure.
Which four actions should you perform in sequence? To answer, move the appropriate actions from the list of actions to the answer area and arrange them in the correct order.
NOTE: More than one order of answer choice is correct. You will receive credit for any of the correct orders you select.
Select and Place:

Suggested Answer:
1.Create a gateway subnet.
2.Create a VPN gateway.
3.Create a local gateway.
4.Create a VPN connection.
Explanation of right options: To establish a site-to-site VPN between an on-premises network and Azure, you must first create a gateway subnet in your VNet (required for hosting the VPN gateway). Next, create a VPN gateway in the gateway subnet (this is the Azure endpoint for the VPN tunnel). Then, create a local gateway to represent your on-premises VPN device and its public IP in Azure. Finally, create a VPN connection to link the Azure VPN gateway to the local gateway, establishing the tunnel.
-----------------
Other options: Create a custom DNS server is not required for a VPN connection and relates to name resolution, not VPN setup. Create an Azure Content Delivery Network (CDN) profile is unrelated to networking or VPN configuration, and is for global content delivery, not private site-to-site connectivity.
