You have an Azure subscription that contains three virtual networks named VNET1, VNET2, and VNET3.
Peering for VNET1 is configured as shown in the following exhibit.
Peering for VNET2 is configured as shown in the following exhibit.
Peering for VNET3 is configured as shown in the following exhibit.
How can packets be routed between the virtual networks? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Packets from VNET1 can be routed to: VNET2 and VNET3:
VNET1 has direct peering connections with both VNET2 and VNET3, so it can send packets to both.
Packets from VNET2 can be routed to: VNET1 only:
VNET2 is only peered with VNET1, so it can only route packets to VNET1.
-----------------
Packets from VNET1 can be routed to: VNET2 only, VNET3 only: These are incorrect because VNET1 has direct peerings to both VNET2 and VNET3, allowing it to send packets to both, not just one.
Packets from VNET2 can be routed to: VNET3 only, VNET1 and VNET3: These are incorrect because VNET2 does not have a peering with VNET3, so it cannot route packets to VNET3 or to both VNET1 and VNET3.
