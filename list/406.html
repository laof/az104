You have a computer named Computer1 that has a point-to-site VPN connection to an Azure virtual network named VNet1. The point-to-site connection uses a self-signed certificate.
From Azure, you download and install the VPN client configuration package on a computer named Computer2.
You need to ensure that you can establish a point-to-site VPN connection to VNet1 from Computer2.
Solution: You modify the Azure Active Directory (Azure AD) authentication policies.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
Solution: You modify the Azure Active Directory (Azure AD) authentication policies.
Answer: B. No
Explanation: Modifying Azure AD authentication policies does not solve the issue because the point-to-site VPN connection is using certificate-based authentication (self-signed certificate), not Azure AD authentication. To allow Computer2 to connect, you must ensure that the client certificate (the same one used by Computer1, or a client certificate generated from the self-signed root certificate) is installed on Computer2. Changing Azure AD policies does not affect certificate-based VPN authentication.
The solution should be export the client certificate from Computer1 and install the certificate on Computer2
