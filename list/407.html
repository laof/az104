You have a computer named Computer1 that has a point-to-site VPN connection to an Azure virtual network named VNet1. The point-to-site connection uses a self-signed certificate.
From Azure, you download and install the VPN client configuration package on a computer named Computer2.
You need to ensure that you can establish a point-to-site VPN connection to VNet1 from Computer2.
Solution: You join Computer2 to Azure Active Directory (Azure AD).
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
Solution: You join Computer2 to Azure Active Directory (Azure AD).
Answer: B. No
Explanation: Joining Computer2 to Azure AD does not allow it to authenticate using a point-to-site VPN connection that requires a self-signed certificate. For certificate-based authentication, the appropriate client certificate (the same one used by Computer1, or a client certificate generated from the self-signed root certificate) must be installed on Computer2. Azure AD join is not required or sufficient for this type of VPN authentication.
