You have an Azure subscription that contains 10 virtual networks. The virtual networks are hosted in separate resource groups.
Another administrator plans to create several network security groups (NSGs) in the subscription.
You need to ensure that when an NSG is created, it automatically blocks TCP port 8080 between the virtual networks.
Solution: You create a resource lock, and then you assign the lock to the subscription.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
Solution: You create a resource lock, and then you assign the lock to the subscription.
Answer: B. No
Explanation: A resource lock in Azure (either Read-only or Delete) is used to prevent resources from being deleted or modified, but it does not apply or enforce any network security rules. To automatically block TCP port 8080 between virtual networks, you would need to configure an NSG rule that denies traffic on port 8080, not a resource lock. Resource locks are not used for security rule enforcement.
