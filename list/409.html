You have an Azure subscription named Subscription1. Subscription1 contains a virtual machine named VM1.

You have a computer named Computer1 that runs Windows 10. Computer1 is connected to the Internet.
You add a network interface named vm1173 to VM1 as shown in the exhibit. (Click the Exhibit tab.)
From Computer1, you attempt to connect to VM1 by using Remote Desktop, but the connection fails.
You need to establish a Remote Desktop connection to VM1.
What should you do first?

  • A..Change the priority of the RDP rule
  • B..Attach a network interface
  • C..Delete the DenyAllInBound rule
  • CorrectD..Start VM1
Suggested Answer:
D..Start VM1
Explanation: The RDP rule has the highest priority (lowest number, 300), so it is evaluated first and will allow Remote Desktop traffic before the DenyAllInBound rule is processed. Since the NSG rules are correct, the most likely reason for the connection failure is that VM1 is not running. Therefore, you should start VM1 to establish a Remote Desktop connection.
-----------------
A..Change the priority of the RDP rule
Explanation: The RDP rule already has the highest priority, so changing its priority will not affect the result. The rule is already correctly positioned to allow RDP traffic.
B..Attach a network interface
Explanation: The network interface (vm1173) is already attached to VM1 as shown in the exhibit, so attaching another interface is unnecessary.
C..Delete the DenyAllInBound rule
Explanation: Deleting the DenyAllInBound rule is not required because the RDP rule already permits the necessary traffic before the deny rule is evaluated. Removing DenyAllInBound could reduce security by allowing unwanted inbound traffic.
