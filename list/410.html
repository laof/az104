You have the Azure virtual machines shown in the following table.

A DNS service is installed on VM1.
You configure the DNS servers settings for each virtual network as shown in the following exhibit.
You need to ensure that all the virtual machines can resolve DNS names by using the DNS service on VM1.
What should you do?

  • A..Configure a conditional forwarder on VM1
  • B..Add service endpoints on VNET1
  • C..Add service endpoints on VNET2 and VNET3
  • CorrectD..Configure peering between VNET1, VNET2, and VNET3
Suggested Answer:
D..Configure peering between VNET1, VNET2, and VNET3
Explanation: To allow all virtual machines in different virtual networks to resolve DNS names using the DNS service installed on VM1 (in VNET1), network connectivity is required between all VNETs. Configuring peering between VNET1, VNET2, and VNET3 enables cross-VNET communication so that DNS requests from VMs in VNET2 and VNET3 can reach VM1.
-----------------
A..Configure a conditional forwarder on VM1
Explanation: A conditional forwarder directs DNS queries for specific domains to another DNS server but does not provide the required network connectivity across VNETs for all VMs to reach VM1.
B..Add service endpoints on VNET1
Explanation: Service endpoints provide secure, direct connectivity to Azure services, not to a custom DNS server running on a VM. This does not help VMs in other VNETs reach VM1 for DNS.
C..Add service endpoints on VNET2 and VNET3
Explanation: Service endpoints do not enable connectivity to custom VMs in other VNETs. They are designed for Azure PaaS services, so this does not solve the DNS resolution requirement.
