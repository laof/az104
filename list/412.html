Question : 39 Topic: 5

You have the Azure virtual network named VNet1 that contains a subnet named Subnet1. Subnet1 contains three Azure virtual machines. Each virtual machine has a public IP address.

The virtual machines host several applications that are accessible over port 443 to users on the Internet.
Your on-premises network has a site-to-site VPN connection to VNet1.
You discover that the virtual machines can be accessed by using the Remote Desktop Protocol (RDP) from the Internet and from the on-premises network.
You need to prevent RDP access to the virtual machines from the Internet, unless the RDP connection is established from the on-premises network. The solution must ensure that all the applications can still be accessed by the Internet users.
What should you do?

  • A..Modify the address space of the local network gateway
  • CorrectB..Create a deny rule in a network security group (NSG) that is linked to Subnet1
  • C..Remove the public IP addresses from the virtual machines
  • D..Modify the address space of Subnet1
Suggested Answer:
B..Create a deny rule in a network security group (NSG) that is linked to Subnet1
Explanation: The best solution is to create a deny rule in an NSG on Subnet1 that blocks inbound RDP (port 3389) access from all sources except the on-premises network's IP range. This allows only RDP connections from your on-premises network while still allowing Internet users to access the applications over port 443.
-----------------
A..Modify the address space of the local network gateway
Explanation: Changing the address space of the local network gateway does not control RDP access and would disrupt your VPN.
C..Remove the public IP addresses from the virtual machines
Explanation: Removing public IPs would block all Internet access, including the required application access over port 443.
D..Modify the address space of Subnet1
Explanation: Modifying the subnet address space does not affect NSG rules or RDP access control.
