You have an Azure subscription that contains the resources in the following table.

Subnet1 is associated to VNet1. NIC1 attaches VM1 to Subnet1.
You need to apply ASG1 to VM1.
What should you do?

  • CorrectA..Associate NIC1 to ASG1
  • B..Modify the properties of ASG1
  • C..Modify the properties of NSG1
Suggested Answer:
A..Associate NIC1 to ASG1 (Correct)
Explanation: To apply ASG1 to VM1, you must associate NIC1 (the network interface attached to VM1) with ASG1. This joins VM1 to the application security group, allowing NSG rules to reference ASG1 as a source or destination. This action is performed from the VM's Networking blade or the NIC's properties in the Azure portal.
-----------------
B..Modify the properties of ASG1
Explanation: You do not directly modify ASG1 to add VMs; instead, you assign NICs to ASGs. The ASG itself is just a container for grouping NICs.
C..Modify the properties of NSG1
Explanation: While you reference ASGs in NSG rules to control traffic, this does not "apply" an ASG to a VM. The association step is done by joining the NIC to the ASG first, then using ASG in NSG rules as needed.
