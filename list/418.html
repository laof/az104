Question : 45 Topic: 5

You have an Azure subscription that contains the resources shown in the following table.

You need to create a network interface named NIC1.
In which location can you create NIC1?

  • A..East US and North Europe only
  • CorrectB..East US only
  • C..East US, West Europe, and North Europe
  • D..East US and West Europe only
Suggested Answer:
B..East US only
— A network interface (NIC1) must be created in the same region as the associated virtual network (VNet). In this scenario, VNET1 is in East US, so NIC1 must also be created in East US to be able to connect to VNET1. Azure requires that the NIC and the VNet are in the same region for association.
-----------------
A..East US and North Europe only
— Incorrect: The NIC must be in the same location as the VNet (East US); North Europe is not valid because there is no VNet in North Europe.
C..East US, West Europe, and North Europe
— Incorrect: NIC1 cannot be created in West Europe or North Europe since the only VNet is in East US; NIC must be in the VNet's region.
D..East US and West Europe only
— Incorrect: Only East US is valid because the NIC must match the VNet's region; West Europe is not an option unless there is a VNet there.
Additional:
Before creating a network interface, you must have an existing virtual network in the same location and subscription you create a network interface in. If you try to create a NIC on a location that does not have any VNets you will get the following error: "The currently selected subscription and location lack any existing virtual networks. Create a virtual network first."
