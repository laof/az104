You have Azure virtual machines that run Windows Server 2019 and are configured as shown in the following table.

You create a public Azure DNS zone named adatum.com and a private Azure DNS zone named contoso.com.
For controso.com, you create a virtual network link named link1 as shown in the exhibit. (Click the Exhibit tab.)
You discover that VM1 can resolve names in contoso.com but cannot resolve names in adatum.com. VM1 can resolve other hosts on the Internet.
You need to ensure that VM1 can resolve host names in adatum.com.
What should you do?

  • A..Update the DNS suffix on VM1 to be adatum.com
  • CorrectB..Configure the name servers for adatum.com at the domain registrar
  • C..Create an SRV record in the contoso.com zone
  • D..Modify the Access control (IAM) settings for link1
Suggested Answer:
B..Configure the name servers for adatum.com at the domain registrar
— VM1 can resolve names in contoso.com (private Azure DNS zone) because VNET1 is linked to that private zone. However, for adatum.com, which is a public Azure DNS zone, VM1 uses standard Internet DNS resolution. If names in adatum.com cannot be resolved, it's likely because the public DNS zone is not properly delegated at the domain registrar. To ensure global DNS resolution, you must configure the name servers for adatum.com at the domain registrar to point to the Azure DNS-provided name servers.
-----------------
A..Update the DNS suffix on VM1 to be adatum.com
— Incorrect: The DNS suffix only affects the default domain for unqualified name resolution; it does not configure DNS servers or delegation, so it won't help VM1 resolve adatum.com if DNS is not delegated.
C..Create an SRV record in the contoso.com zone
— Incorrect: SRV records are for service location, not for cross-zone name resolution. This would not allow VM1 to resolve adatum.com.
D..Modify the Access control (IAM) settings for link1
— Incorrect: IAM settings control who can manage the link resource, not DNS resolution behavior. This does not address the DNS delegation required for adatum.com.
