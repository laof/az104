HOTSPOT -

You have an Azure subscription that contains the Azure virtual machines shown in the following table.
You configure the network interfaces of the virtual machines to use the settings shown in the following table.
From the settings of VNET1 you configure the DNS servers shown in the following exhibit.
The virtual machines can successfully connect to the DNS server that has an IP address of 192.168.10.15 and the DNS server that has an IP address of
193.77.134.10.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
VM1 connects to 193.77.134.10 for DNS queries: Yes
VM2 connects to 193.77.134.10 for DNS queries: No
VM3 connects to 192.168.10.15 for DNS queries: Yes
-----------------
A..VM1 connects to 193.77.134.10 for DNS queries: Yes — VM1 has its NIC DNS set to None, so it uses the custom DNS server configured on the VNET1 level, which is 193.77.134.10.
B..VM2 connects to 193.77.134.10 for DNS queries: No — VM2 has its NIC DNS server set to 192.168.10.15, which overrides the VNET DNS server. Therefore, VM2 does not use 193.77.134.10 for DNS queries.
C..VM3 connects to 192.168.10.15 for DNS queries: Yes — VM3 has its NIC DNS server set to 192.168.10.15, so it uses this DNS server for all queries instead of the VNET DNS.
NIC-level DNS settings always override VNET-level settings. Only VMs without a custom DNS on their NIC (like VM1) will default to the VNET's DNS configuration.
