HOTSPOT -

You have an Azure subscription that contains the resource groups shown in the following table.
RG1 contains the resources shown in the following table.
You need to identify which resources you can move from RG1 to RG2, and which resources you can move from RG2 to RG1.
Which resources should you identify? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Resources that you can move from RG1 to RG2: IP1 and storage1 only
Resources that you can move from RG2 to RG1: IP2 and storage2 only
A..Resources that you can move from RG1 to RG2: IP1 and storage1 only — IP1 has no lock, so it can be moved. storage1 has a Delete lock, but a Delete lock does not prevent moving; it only prevents deletion. VNET1 has a Read-only lock, which blocks all modifications, including moving the resource.
B..Resources that you can move from RG2 to RG1: IP2 and storage2 only — If IP2 and storage2 are in RG2, both can be moved back to RG1 for the same reasons: neither have a Read-only lock, and a Delete lock does not prevent movement.
-----------------
C..None (from RG1 to RG2 and from RG2 to RG1): Incorrect — At least IP2 and storage2 can be moved, so "None" is not correct.
D..IP2 only (from RG1 to RG2 or RG2 to RG1): Incorrect — storage2 (with a Delete lock) can also be moved, so this option is incomplete.
E..IP2 and VNET2 only / IP2, VNET2, and storage2 (from RG1 to RG2 or RG2 to RG1): Incorrect — VNET2 has a Read-only lock, which prevents movement.
Read-only locks prevent moving resources because they block all modifications. Delete locks do not block movement; they only prevent deletion.
