You have an Azure subscription that contains the virtual machines shown in the following table.
You deploy a load balancer that has the following configurations:
✑ Name: LB1
✑ Type: Internal
✑ SKU: Standard
✑ Virtual network: VNET1
You need to ensure that you can add VM1 and VM2 to the backend pool of LB1.
Solution: You create a Basic SKU public IP address, associate the address to the network interface of VM1, and then start VM1.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B..No
-----------------
A..Yes: Incorrect — The solution does not meet the goal because the load balancer (LB1) is a Standard SKU, but the solution creates a Basic SKU public IP address. Standard SKU resources (such as Standard Load Balancer) are NOT compatible with Basic SKU public IP addresses; you must use a Standard SKU public IP for association.
B..No: Correct — Standard SKU Load Balancer requires Standard SKU public IPs. Associating a Basic SKU public IP to a VM intended for use with a Standard Load Balancer backend pool will NOT work.
Key Point:
- SKU types must match between Azure resources. Mixing Basic and Standard SKUs is not supported for load balancer backend pools.
- You can only attach the VMs that are in the same location and on the same virtual network as the LB.
