✑ Type: Internal
✑ SKU: Standard
✑ Virtual network: VNET1
You need to ensure that you can add VM1 and VM2 to the backend pool of LB1.

Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B..No — The solution does not meet the goal because VM2 has a Basic SKU public IP, which is not compatible with a Standard SKU load balancer; all backend pool members must use Standard SKU public IPs if a public IP is assigned.

A..Yes — Incorrect: Although associating a Standard SKU public IP to VM1 would make VM1 eligible, VM2 still has a Basic SKU public IP, which cannot be added to a Standard SKU load balancer backend pool. The SKU types must match for all backend pool resources.
