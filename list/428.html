You have an Azure subscription that contains 10 virtual networks. The virtual networks are hosted in separate resource groups.
Another administrator plans to create several network security groups (NSGs) in the subscription.
You need to ensure that when an NSG is created, it automatically blocks TCP port 8080 between the virtual networks.
Solution: From the Resource providers blade, you unregister the Microsoft.ClassicNetwork provider.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B..No — Unregistering the Microsoft.ClassicNetwork provider does not meet the goal. This action only prevents the use of classic (legacy) networking resources in Azure. It does not affect the behavior of network security groups (NSGs) or automatically block TCP port 8080 between virtual networks. NSGs and their rules are part of the Azure Resource Manager (ARM) model, not the classic model governed by Microsoft.ClassicNetwork.
-----------------
A..Yes — Incorrect: Unregistering the Microsoft.ClassicNetwork provider does not create or enforce any NSG rules, nor does it affect traffic between ARM-based virtual networks. It only disables the ability to create or manage classic networking resources.
