HOTSPOT -

You manage two Azure subscriptions named Subscription1 and Subscription2.
Subscription1 has following virtual networks:
The virtual networks contain the following subnets:
Subscription2 contains the following virtual network:
✑ Name: VNETA
✑ Address space: 10.10.128.0/17
✑ Location: Canada Central
VNETA contains the following subnets:
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
A Site-to-Site connection can be established between VNET1 and VNET2.   No
VNET1 and VNET2 can be peered.   Yes
VNET1 and VNETA can be peered.   No
-----------------
Explanation of right options:
A Site-to-Site connection can be established between VNET1 and VNET2. No: To create a VNet-to-VNet VPN (Site-to-Site connection), you need a special Gateway Subnet. In this scenario, the VNet has no sufficient address space to create a Gateway Subnet and thus cannot establish a VNet-to-VNet VPN connection.
VNET1 and VNET2 can be peered. Yes: Peering is supported because the address spaces of VNET1 (10.10.10.0/24) and VNET2 (172.16.0.0/16) do not overlap.
-----------------
Explanation of the other option:
VNET1 and VNETA can be peered. No: Peering is not allowed between VNET1 and VNETA due to overlapping address spaces. VNETA uses 10.10.128.0/17 which covers a much larger range including 10.10.10.0/24, causing the overlap and making peering not possible.
