Question : 61 Topic: 5

You have an Azure subscription.

You plan to deploy an Azure Kubernetes Service (AKS) cluster to support an app named App1. On-premises clients connect to App1 by using the IP address of the pod.
For the AKS cluster, you need to choose a network type that will support App1.
What should you choose?

  • A..kubenet
  • CorrectB..Azure Container Networking Interface (CNI)
  • C..Hybrid Connection endpoints
  • D..Azure Private Link
Suggested Answer:
B.. Azure Container Networking Interface (CNI)
-----------------
Explanation of right option:
B.. Azure Container Networking Interface (CNI): To allow on-premises clients to connect directly to the IP address of a pod in AKS, you must use Azure CNI networking. With Azure CNI, pods get routable IP addresses from the virtual network, making them directly accessible from on-premises networks if routing is configured. This fulfills the scenario requirement.
-----------------
Explanation of the other options:
A.. kubenet: With kubenet, pods receive IP addresses from an internal range that is not directly routable from on-premises, so it does not meet the requirement.
C.. Hybrid Connection endpoints: Hybrid Connections are used for app services to connect to on-premises resources, not for exposing pod IPs for direct access.
D.. Azure Private Link: Azure Private Link is used to provide private connectivity to Azure platform services, not for directly exposing pod IPs.
