You have an Azure subscription that contains the virtual machines shown in the following table.
You deploy a load balancer that has the following configurations:
✑ Name: LB1
✑ Type: Internal
✑ SKU: Standard
✑ Virtual network: VNET1
You need to ensure that you can add VM1 and VM2 to the backend pool of LB1.
Solution: You disassociate the public IP address from the network interface of VM2.
Does this meet the goal?

  • CorrectA..Yes
  • B..No
Suggested Answer:
A.. Yes
-----------------
Explanation of right option:
A.. Yes: For a Standard SKU (internal) load balancer, virtual machines can only be added to the backend pool if they have a Standard SKU public IP or no public IP at all. VM2 currently has a Basic SKU public IP, which is not supported for backend pool membership with a Standard SKU load balancer. By disassociating the public IP address from VM2, both VM1 (which already has no public IP) and VM2 (after removal) will meet the requirements and can be added to the backend pool.
-----------------
Explanation of the other options:
B.. No: This is incorrect because removing the Basic SKU public IP from VM2 is necessary for it to be eligible for the backend pool of a Standard SKU load balancer; otherwise, it cannot be added.
