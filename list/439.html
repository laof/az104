HOTSPOT -

You have an on-premises data center and an Azure subscription. The data center contains two VPN devices. The subscription contains an Azure virtual network named VNet1. VNet1 contains a gateway subnet.
You need to create a site-to-site VPN. The solution must ensure that if a single instance of an Azure VPN gateway fails, or a single on-premises VPN device fails, the failure will not cause an interruption that is longer than two minutes.
What is the minimum number of public IP addresses, virtual network gateways, and local network gateways required in Azure? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Public IP addresses: 2
Virtual network gateways: 1
Local network gateways: 1
Explanation of right options:
Public IP addresses: 2 — For high availability and to avoid an outage longer than two minutes if an Azure VPN gateway instance fails, you must use an active-active VPN gateway configuration, which requires two public IP addresses.
Virtual network gateways: 1 — Only a single virtual network gateway (in active-active mode) is needed on the Azure side for redundancy.
Local network gateways: 1 — On the Azure side, you only need one local network gateway to represent the on-premises VPN endpoint(s), even if there are two VPN devices, as both can be defined under the same local network gateway resource.
-----------------
Explanation of the other options:
Using only 1 public IP address does not provide redundancy. More than 2 public IPs are unnecessary for a single active-active Azure gateway.
More than 1 virtual network gateway is not needed because a single gateway can be configured for high availability.
More than 1 local network gateway is only required if you have multiple on-premises address spaces or endpoints to represent separately, but for a standard setup with two on-premises devices, a single local network gateway can suffice on the Azure side.
