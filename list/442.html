Question : 69 Topic: 5

You have an Azure subscription that contains a policy-based virtual network gateway named GW1 and a virtual network named VNet1.

You need to ensure that you can configure a point-to-site connection from an on-premises computer to VNet1.
Which two actions should you perform? Each correct answer presents part of the solution.
NOTE: Each correct selection is worth one point.

  • A..Add a service endpoint to VNet1
  • B..Reset GW1
  • CorrectC..Create a route-based virtual network gateway
  • D..Add a connection to GW1
  • CorrectE..Delete GW1
  • F..Add a public IP address space to VNet1
Suggested Answer:
C.. Create a route-based virtual network gateway — Point-to-site (P2S) VPN connections in Azure require a route-based virtual network gateway. Policy-based gateways do not support point-to-site connections. Therefore, you must create a new route-based gateway.
E.. Delete GW1 — Since each virtual network can only have one virtual network gateway, and P2S requires a route-based gateway, you must delete the existing policy-based gateway (GW1) before creating the required route-based gateway.
-----------------
Explanation of the other options:
A.. Add a service endpoint to VNet1 — Service endpoints are used to secure Azure service resources to a virtual network, not for configuring VPN gateways or point-to-site connectivity.
B.. Reset GW1 — Resetting the gateway does not change its type and does not enable point-to-site connections.
D.. Add a connection to GW1 — You cannot add a point-to-site connection to a policy-based gateway. This option is not possible until you have a route-based gateway.
F.. Add a public IP address space to VNet1 — Public IP address spaces are not required or recommended for VPN connections; Azure VNets typically use private address spaces for secure communication.
