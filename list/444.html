HOTSPOT -

You have an Azure subscription that contains the virtual networks shown in the following table.
The subscription contains the private DNS zones shown in the following table.
You add virtual network links to the private DNS zones as shown in the following table.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Statement 1: You can enable auto registration for Link2. Yes
Explanation: You can change the "Enable auto registration" setting for an existing virtual network link. Link2 currently has auto registration set to "No," but this setting can be switched to "Yes" if needed.
Statement 2: You can add a virtual network link for VNET1 to Zone3.com. Yes
Explanation: There is no virtual network link connecting VNET1 to Zone3.com yet. Azure allows you to create additional virtual network links between any virtual network and any private DNS zone, regardless of region.
Statement 3: You can add a virtual network link for VNET2 to Zone1.com and enable auto registration. No
Explanation: A private DNS zone can only have one virtual network link with auto registration enabled. Zone1.com already has VNET1 linked with auto registration enabled, so you cannot enable auto registration for a new VNET2 link to Zone1.com.
-----------------
Option 1: You can enable auto registration for Link2. Yes — The "Enable auto registration" setting for a virtual network link can be changed after the link is created.
Option 2: You can add a virtual network link for VNET1 to Zone3.com. Yes — There is no restriction on adding additional virtual network links between networks and DNS zones unless a link already exists for that combination.
Option 3: You can add a virtual network link for VNET2 to Zone1.com and enable auto registration. No — A private DNS zone can have only one auto registration link; Zone1.com already has one with VNET1.
