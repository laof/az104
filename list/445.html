HOTSPOT -

You have an Azure subscription.
You plan to use an Azure Resource Manager template to deploy a virtual network named VNET1 that will use Azure Bastion.
How should you complete the template? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Statement 1: "name" AzureBastionSubnet
Statement 2: "addressPrefix" 10.10.10.0/27
Explanation: When deploying Azure Bastion in a virtual network, you must create a subnet named AzureBastionSubnet exactly (case-sensitive) and its subnet prefix must be at least /27 or larger (as per Azure Bastion requirements). "10.10.10.0/27" satisfies this, while /29 or /30 would be too small.
-----------------
Option 1: "name": AzureBastionSubnet — This is the required and reserved name for the subnet Azure Bastion uses. Any other name (such as AzureFirewallSubnet, LAN01, or RemoteAccessSubnet) would not work for Bastion deployment.
Option 2: "addressPrefix": 10.10.10.0/27 — This prefix is the smallest allowed for Azure Bastion. Prefixes /29 and /30 are too small to support Bastion and would not be accepted by Azure.
Additional:
Azure Bastion requires a dedicated subnet: AzureBastionSubnet. You must create this subnet in the same virtual network that you want to deploy Azure Bastion to.
