You manage a virtual network named VNet1 that is hosted in the West US Azure region.
VNet1 hosts two virtual machines named VM1 and VM2 that run Windows Server.
You need to inspect all the network traffic from VM1 to VM2 for a period of three hours.
Solution: From Azure Network Watcher, you create a connection monitor.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
B..No
Explanation: Azure Network Watcher's Connection Monitor is designed to monitor connectivity and network performance between two endpoints (such as VM1 and VM2). However, it does not capture or inspect the actual network traffic (packets) between the VMs. To inspect all network traffic, you would need to use the Packet Capture feature instead.

The connection monitor capability monitors communication at a regular interval and informs you of reachability, latency, and network topology changes between the VM and the endpoint.
Connection monitor also provides the minimum, average, and maximum latency observed over time. After learning the latency for a connection, you may find that you're able to decrease the latency by moving your Azure resources to different Azure regions.
