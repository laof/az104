Question 2-5 Answer:
1. You can create a virtual network in Subscription1. No
2. You can create a virtual machine in Subscription2. Yes
3. You can add Subscription1 to ManagementGroup11. No
-----------------
Explanation:
1. You can create a virtual network in Subscription1. — No, because the Not allowed resource types policy at the Tenant Root Group blocks virtualNetworks, and this applies to all child groups including Subscription1.
2. You can create a virtual machine in Subscription2. — The policy at the Tenant Root Group blocks only virtualNetworks, not virtual machines. The policy at ManagementGroup12 allows virtualNetworks but does not restrict other types, so virtual machines are allowed.
3. You can add Subscription1 to ManagementGroup11. — No, because Subscription1 is already under ManagementGroup21, which is a child of ManagementGroup11. You cannot directly add it to ManagementGroup11 without first removing it from its current group.
Additional:
The most restrictive policy always applies when there is a conflict between Azure policy rules at different scopes, such as between a subscription and the tenant root group. If the tenant-level policy denies a resource type and the subscription-level policy allows it, the deny at the tenant level takes precedence and the resource will be blocked.
