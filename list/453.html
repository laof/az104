Server2 resolves host2.contoso.com to 131.107.50.50: No
Server2 resolves host1.contoso.com to 131.107.10.15: Yes
Server3 resolves host2.contoso.com to 131.107.50.50: No
-----------------
Explanation of right options:
Server2 resolves host2.contoso.com to 131.107.50.50: No — Server2 uses DNS server 10.10.0.4 (Server1), which only has a primary zone for contoso.com with an A record for host1.contoso.com and does not have a record for host2.contoso.com. There is no evidence of DNS forwarding to Azure Private DNS, so Server2 cannot resolve host2.contoso.com.
Server2 resolves host1.contoso.com to 131.107.10.15: Yes — Server1 (10.10.0.4) contains an A record for host1.contoso.com with the value 131.107.10.15, so Server2 can resolve this name via its configured DNS.
Server3 resolves host2.contoso.com to 131.107.50.50: No — Server3 inherits the DNS server from VNET2, which is set to 10.10.0.4 (Server1). Server1 does not have a record for host2.contoso.com, and there is no DNS forwarding configured, so Server3 cannot resolve host2.contoso.com.
-----------------
Explanation of other options:
Any option marking Server2 resolves host2.contoso.com to 131.107.50.50: Yes or Server3 resolves host2.contoso.com to 131.107.50.50: Yes is incorrect, because both Server2 and Server3 use Server1 (10.10.0.4) as their DNS, and Server1 does not have a record for host2.contoso.com nor is there any evidence of forwarding to the Azure Private DNS zone. Only Server2 resolves host1.contoso.com to 131.107.10.15: Yes is correct, as that record is present in Server1’s DNS.
