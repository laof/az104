Question : 81 Topic: 5

You have a virtual network named VNet1 as shown in the exhibit. (Click the Exhibit tab.)

No devices are connected to VNet1.
You plan to peer VNet1 to another virtual network named VNet2. VNet2 has an address space of 10.2.0.0/16.
You need to create the peering.
What should you do first?

  • CorrectA..Modify the address space of VNet1.
  • B..Add a gateway subnet to VNet1.
  • C..Create a subnet on VNet1 and VNet2.
  • D..Configure a service endpoint on VNet2.
Suggested Answer:
A. Modify the address space of VNet1. Yes — Both VNet1 and VNet2 have the same address space (10.2.0.0/16). Azure virtual network peering requires that the address spaces of the peered VNets do not overlap. Therefore, you must first modify the address space of VNet1 so that it does not overlap with VNet2 before you can successfully create the peering.
-----------------
Explanation of other options:
B. Add a gateway subnet to VNet1. No — Adding a gateway subnet is only necessary for VPN gateway connections, not for basic VNet peering.
C. Create a subnet on VNet1 and VNet2. No — Creating subnets is not required to establish VNet peering; the issue is the overlapping address space.
D. Configure a service endpoint on VNet2. No — Service endpoints are unrelated to VNet peering and do not resolve address space overlap.
