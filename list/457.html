You have a computer named Computer1 that has a point-to-site VPN connection to an Azure virtual network named VNet1. The point-to-site connection uses a self-signed certificate.
From Azure, you download and install the VPN client configuration package on a computer named Computer2.
You need to ensure that you can establish a point-to-site VPN connection to VNet1 from Computer2.
Solution: On Computer2, you set the Startup type for the IPSec Policy Agent service to Automatic.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:
Answer: B..No
Explanation: Changing the IPSec Policy Agent service Startup type to Automatic on Computer2 does not meet the goal. For a point-to-site VPN connection using a self-signed certificate, you must install the root certificate (that was used to generate the client certificate for Computer1) in the Trusted Root Certification Authorities store on Computer2, and install the appropriate client certificate in the Personal store. Setting the IPSec Policy Agent service to Automatic alone does not enable authentication or certificate validation for the VPN connection.
