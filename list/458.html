You have five Azure virtual machines that run Windows Server 2016. The virtual machines are configured as web servers.

You have an Azure load balancer named LB1 that provides load balancing services for the virtual machines.
You need to ensure that visitors are serviced by the same web server for each request.
What should you configure?

  • CorrectA..Session persistence to Client IP and protocol
  • B..Protocol to UDP
  • C..Session persistence to None
  • D..Floating IP (direct server return) to Enabled
Suggested Answer:
A..Session persistence to Client IP and protocol
Explanation: To ensure that each visitor is always serviced by the same web server for every request (also known as "session persistence" or "sticky sessions"), you should configure the Azure Load Balancer's session persistence setting to "Client IP and protocol." This setting directs all requests from the same client IP address and protocol to the same backend VM, maintaining user session consistency. None of the other options provide this functionality.
