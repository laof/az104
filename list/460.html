Question : 87 Topic: 5

You have an Azure subscription.

You are deploying an Azure Kubernetes Service (AKS) cluster that will contain multiple pods. The pods will use kubernet networking.
You need to restrict network traffic between the pods.
What should you configure on the AKS cluster?

  • A..the Azure network policy
  • CorrectB..the Calico network policy
  • C..pod security policies
  • D..an application security group
Suggested Answer:
B..the Calico network policy
Explanation: To restrict network traffic between pods in an AKS cluster using kubenet networking, you must enable and configure the Calico network policy, which is a supported network policy provider for AKS with kubenet. Calico lets you define rules to control traffic between pods.
Other Options Explained:
A. the Azure network policy: Incorrect. Azure network policy is only supported with Azure CNI networking, not kubenet.
C. pod security policies: Incorrect. Pod security policies control pod-level security settings, not network traffic between pods.
D. an application security group: Incorrect. Application security groups are used for Azure VMs, not for AKS pods or Kubernetes networking.
