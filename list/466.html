HOTSPOT -

You have an Azure subscription that contains the virtual machines shown in the following table.
The subscription contains a storage account named contoso2024 as shown in the following exhibit.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.

Suggested Answer:
VM1 can connect to contoso2024 by using 131.107.10.10. Yes
VM2 can connect to contoso2024 by using 150.120.10.10. Yes
VM3 must use its private IP address to connect to contoso2024. No
Explanation: All public IP addresses for VM1, VM2, and VM3 are allowed in the storage account firewall, so they can connect using their public IPs and VM3 does not need to use its private IP.
Additional explanation:
The configuration enables a service endpoint for Azure Storage in Subnet1 of VNet1, allowing VMs in that subnet to securely access contoso2024 using private IP addresses. Any resources in Subnet1 of VNet1 can access the storage account using their private IPs, providing secure, direct connectivity from the subnet to the Azure Storage service.
