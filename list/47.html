You have an Azure subscription that contains the resources shown in the following table:
You assign a policy to RG6 as shown in the following table:
To RG6, you apply the tag: RGroup: RG6.
You deploy a virtual network named VNET2 to RG6.
Which tags apply to VNET1 and VNET2? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Question 2-7 Answer:
VNET1: Department: D1 only
VNET2: Label: Value1 only
-----------------
Explanation of Correct Options:
VNET1: — This resource was created before the policy was assigned. It already has the tag Department: D1, and since Azure policies do not retroactively apply to existing resources unless a remediation task is run, it retains only this tag.
VNET2: — This resource was created after the policy was assigned. The policy enforces the tag Label: Value1 on all new resources in RG6. The tag RGroup: RG6 applied to the resource group is not inherited by default, and the policy does not specify inheritance from the resource group, so VNET2 only gets Label: Value1.
Additional:
The policy assigned to RG6 is of type "Apply tag and its default value", which means it enforces the presence of a specific tag (Label) with a default value (Value1) on newly created resources. This policy does not copy or inherit tags like RGroup: RG6 from the resource group. If VNET2 ends up with RGroup: RG6, it must have been applied manually or through another mechanism, not by the policy shown.
To configure Azure Policy to inherit tags from a parent resource group, use the built-in policy definition called "Inherit a tag from the resource group if missing". This policy uses the modify effect to apply a tag from the resource group to any new or updated resource that does not already have that tag.
Steps to Configure:
1. In the Azure portal, go to Policy and select Definitions.
2. Search for and select the built-in policy named "Inherit a tag from the resource group if missing".
3. Click Assign and set the Scope to the subscription or resource group where you want the policy to apply.
4. In Parameters, specify the tag name you want to inherit.
5. Complete the assignment and optionally create a remediation task to apply the tag to existing resources.
-----------------
Notes: - This policy only applies the tag if the resource does not already have it.
- It does not overwrite existing tag values.
- To apply the tag to existing resources, you must run a remediation task.
- This policy does not inherit all tags—only the one specified in the parameters.
