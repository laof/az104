You plan to deploy route-based Site-to-Site VPN connections between several on-premises locations and an Azure virtual network.

Which tunneling protocol should you use?

  • A..IKEv1
  • B..PPTP
  • CorrectC..IKEv2
  • D..L2TP
Suggested Answer:
C..IKEv2<>br>
Explanation: For route-based Site-to-Site VPN connections in Azure, you should use IKEv2 as the tunneling protocol. IKEv2 is the recommended and supported protocol for Azure route-based VPNs due to its enhanced security, reliability, and compatibility.
---------------------
A..IKEv1
Explanation: IKEv1 is only supported for policy-based VPNs, not for route-based VPNs in Azure.
B..PPTP
Explanation: PPTP is not supported for Azure VPN connections and is considered insecure.
D..L2TP
Explanation: L2TP is not used for Azure Site-to-Site VPNs and is generally used for point-to-site scenarios.
Additional:
IKEv2 vs IKEv1 Differences:
IKEv2 is a newer, more secure, and more efficient version of the Internet Key Exchange protocol compared to IKEv1.
Key Differences:
- IKEv2 supports stronger encryption algorithms and better security.
- IKEv2 is more resilient to network changes (such as IP address changes or network roaming).
- IKEv2 has built-in support for NAT traversal and mobility.
- IKEv2 is faster and has improved reliability due to fewer message exchanges.
- IKEv1 is older, less secure, and generally only used for legacy or policy-based VPNs, while IKEv2 is recommended for new, route-based VPN deployments.
- IKEv2 supports up to 250 Site-to-Site (S2S) VPN connections per VPN gateway in Azure, while IKEv1 supports up to 100 Site-to-Site (S2S) VPN connections per VPN gateway in Azure.
