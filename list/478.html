Question : 105 Topic: 5

You have five Azure virtual machines that run Windows Server 2016. The virtual machines are configured as web servers.

You have an Azure load balancer named LB1 that provides load balancing services for the virtual machines.
You need to ensure that visitors are serviced by the same web server for each request.
What should you configure?

  • A..Floating IP (direct server return) to Disabled
  • CorrectB..Session persistence to Client IP
  • C..Protocol to UDP
  • D..Idle Time-out (minutes) to 20
Suggested Answer:
B..Session persistence to Client IP
Explanation: Configuring Session persistence to Client IP ensures that requests from the same user (client IP) are always directed to the same backend VM, maintaining session affinity and providing a consistent experience for each visitor.
-----------------
A..Floating IP (direct server return) to Disabled
Explanation: Floating IP is used for scenarios like SQL Always On or for direct server return; it does not control session affinity for web servers.
C..Protocol to UDP
Explanation: Changing the protocol to UDP does not affect how sessions are persisted or how web requests are routed for session affinity.
D..Idle Time-out (minutes) to 20
Explanation: Modifying the idle time-out only changes how long a connection can stay idle before being closed, but does not ensure that a client always connects to the same backend server.
