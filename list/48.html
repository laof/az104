You have an Azure subscription named AZPT1 that contains the resources shown in the following table:

You create a new Azure subscription named AZPT2.
You need to identify which resources can be moved to AZPT2.
Which resources should you identify?

  • A..VM1, storage1, VNET1, and VM1Managed only
  • B..VM1 and VM1Managed only
  • CorrectC..VM1, storage1, VNET1, VM1Managed, and RVAULT1
  • D..RVAULT1 only
Suggested Answer:

Question 2-8 Answer:

A. VM1, storage1, VNET1, and VM1Managed only
-----------------

Explanation of Correct Option:

A. VM1, storage1, VNET1, and VM1Managed only — These resources are eligible to be moved between subscriptions within the same tenant. Azure supports moving virtual machines, their managed disks, virtual networks, and storage accounts across subscriptions as long as both subscriptions are in the same Azure Active Directory tenant.
-----------------

Explanation of Other Options:

B. VM1 and VM1Managed only — Incorrect because storage1 and VNET1 can also be moved.
C. VM1, storage1, VNET1, VM1Managed, and RVAULT1 — Incorrect because Recovery Services vaults (RVAULT1) cannot be moved across subscriptions.
D. RVAULT only — Incorrect because Recovery Services vaults are not supported for subscription moves.

Azure Services That Can't Be Moved Across Subscriptions:

Recovery Services vaults, Azure AD Domain Services, diagnostic settings, support providers, AD Hybrid Health Service components, Azure Advisor configurations, certain alert types, API Management in Consumption SKU, App Service components like apiapps and gateways, HDInsight cluster networking resources, and any child resources whose parent cannot be moved [1](https://learn.microsoft.com/en-us/azure/azure-resource-manager/management/move-support-resources).
