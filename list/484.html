HOTSPOT -

You plan to deploy the following Azure Resource Manager (ARM) template.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:
Option 1: No — LB1 will be connected to a subnet named VNET1/API, not VNET1/netname. The ARM template sets the subnet name variable to "API", so the full subnet path is VNET1/API.
Option 2: No — LB1 can be deployed in a different resource group than VNET1 as long as both the load balancer and its backend resources are in the same virtual network. For example, VNET1 can be in Resource Group A, VM1 in Resource Group B, and LB1 in Resource Group C, all connected to VNET1. The requirement is same virtual network, not same resource group.
Option 3: No — The value of the sku variable cannot be provided as a parameter at deployment, since sku is defined as a variable in the template, not a parameter. Only parameters can be set or overridden at deployment time; variables are calculated inside the template.
