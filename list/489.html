You have five Azure virtual machines that run Windows Server 2016. The virtual machines are configured as web servers.

You have an Azure load balancer named LB1 that provides load balancing services for the virtual machines.
You need to ensure that visitors are serviced by the same web server for each request.
What should you configure?

  • A..Session persistence to None
  • B..a health probe
  • CorrectC..Session persistence to Client IP and protocol
  • D..Idle Time-out (minutes) to 20
Suggested Answer:
C..Session persistence to Client IP and protocol
-----------------
A..Session persistence to None: Incorrect — If session persistence is set to None, each request from a client can be routed to any backend server, so visitors would not be serviced by the same web server for each request.
B..a health probe: Incorrect — A health probe is used to determine which backend servers are healthy and available, but it does not control whether a client is always routed to the same server.
C..Session persistence to Client IP and protocol: Correct — Session persistence (also known as "sticky sessions") set to Client IP and protocol ensures that requests from the same client IP address using the same protocol are always routed to the same backend server, meeting the requirement.
D..Idle Time-out (minutes) to 20: Incorrect — Changing the idle timeout setting only affects how long a TCP or HTTP connection can remain idle before being closed, and does not influence session persistence or server selection.
