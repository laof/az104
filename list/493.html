You have a Windows 11 device named Device and an Azure subscription that contains the resources shown in the following table.
Device1 has Azure PowerShell and Azure Command-Line Interface (CLI) installed.
From Device1, you need to establish a Remote Desktop connection to VM1.
Which three actions should you perform in sequence? To answer, move the appropriate actions from the list of actions to the answer area and arrange them in the correct order.

Suggested Answer:
From Bastion1, select Native Client Support.
From Azure CLI on Device1, run az network bastion rdp.
On Device1, run mstsc.exe.
From Bastion1, select Native Client Support. — This enables the use of native RDP/SSH clients with Azure Bastion, which is required for remote desktop connections via Bastion.
From Azure CLI on Device1, run az network bastion rdp. — This command establishes the Bastion tunnel from Device1 to VM1 through Bastion1 using the Azure CLI.
On Device1, run mstsc.exe. — This opens the Remote Desktop Connection app on Device1 to connect to the VM using the established Bastion tunnel.
-----------------
From Bastion1, enable Kerberos authentication. — Enabling Kerberos authentication is not required for a standard Bastion RDP session.
From VM1, enable just-in-time (JIT) VM access. — JIT is used for controlling access windows but is not required if Bastion and NSG rules already allow access.
Upgrade Bastion1 to the Standard SKU. — The Basic SKU supports native client RDP connections, so upgrading is not required in this scenario.
