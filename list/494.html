You have five Azure virtual machines that run Windows Server 2016. The virtual machines are configured as web servers.

You have an Azure load balancer named LB1 that provides load balancing services for the virtual machines.
You need to ensure that visitors are serviced by the same web server for each request.
What should you configure?

  • A..Floating IP (direct server return) to Enabled
  • CorrectB..Session persistence to Client IP
  • C..Protocol to UDP
  • D..Idle Time-out (minutes) to 20
Suggested Answer:
B..Session persistence to Client IP — Setting Session persistence to Client IP ensures that each visitor is consistently routed to the same backend virtual machine based on their IP address, providing sticky session behavior required in this scenario.
-----------------
A..Floating IP (direct server return) to Enabled — Floating IP is used for scalable multi-instance applications like SQL Always On or NLB clusters, but does not control session stickiness.
C..Protocol to UDP — Changing to UDP only affects the transport protocol and does not ensure that users are always routed to the same backend server.
D..Idle Time-out (minutes) to 20 — Adjusting the Idle Time-out only affects how long inactive connections are kept open and does not influence which server handles each request.
