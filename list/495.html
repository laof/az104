You have an Azure subscription that has the public IP addresses shown in the following table.

You plan to deploy an Azure Bastion Basic SKU host named Bastion1.
Which IP addresses can you use?

  • CorrectA..IP1 only
  • B..IP1 and IP2 only
  • C..IP3, IP4, and IP5 only
  • D..IP1, IP2, IP4, and IP5 only
  • E..IP1, IP2, IP3, IP4, and IP5
Suggested Answer:
A..IP1 only — According to the requirement for Azure Bastion (as shown in https://learn.microsoft.com/en-us/azure/bastion/configuration-settings), the public IP address must be Standard SKU and Static assignment. In the table, only IP1 is Standard SKU, Regional tier, IPv4, and Static.
-----------------
B..IP1 and IP2 only — IP2 is Standard SKU and Static, but its tier is Global. Bastion requires a Regional tier, so IP2 is not valid.
C..IP3, IP4, and IP5 only — These are all Basic SKU, but Bastion requires Standard SKU for public IP, so these are not valid.
D..IP1, IP2, IP4, and IP5 only — Only IP1 meets all requirements; the others fail on tier or SKU.
E..IP1, IP2, IP3, IP4, and IP5 — Only IP1 meets all requirements; the others fail on tier or SKU.
