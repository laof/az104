You have five Azure virtual machines that run Windows Server 2016. The virtual machines are configured as web servers.

You have an Azure load balancer named LB1 that provides load balancing services for the virtual machines.
You need to ensure that visitors are serviced by the same web server for each request.
What should you configure?

  • A..Floating IP (direct server return) to Disabled
  • B..Floating IP (direct server return) to Enabled
  • C..a health probe
  • CorrectD..Session persistence to Client IP
Suggested Answer:
D..Session persistence to Client IP — Setting Session persistence to Client IP ensures that each visitor is consistently routed to the same backend virtual machine based on their IP address, providing sticky session behavior required in this scenario.
-----------------
A..Floating IP (direct server return) to Disabled — This setting is unrelated to session stickiness and is typically used for direct server return scenarios.
B..Floating IP (direct server return) to Enabled — Enabling floating IP is intended for certain high-availability scenarios (like SQL Always On) and does not ensure clients are routed to the same backend.
C..a health probe — A health probe is required to monitor backend health, but does not provide session persistence or sticky session functionality.
