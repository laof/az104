You are planning to deploy an Ubuntu Server virtual machine to your company's Azure subscription.

You are required to implement a custom deployment that includes adding a particular trusted root certification authority (CA).
Which of the following should you use to create the virtual machine?

  • A..The New-AzureRmVm cmdlet.
  • B..The New-AzVM cmdlet.
  • C..The Create-AzVM cmdlet.
  • CorrectD..The az vm create command.
Suggested Answer:
Correct Option: D. The az vm create command.
Explanation: The az vm create command is used in the Azure CLI to create virtual machines. It allows for custom configurations, including adding trusted root certification authorities (CAs) during the deployment process.
-----------------
Explanation of Other Options:
A. The New-AzureRmVm cmdlet: This cmdlet is part of the older AzureRM module, which has been replaced by the Az module. It is not recommended for new deployments.
B. The New-AzVM cmdlet: This cmdlet is used in the Az PowerShell module to create virtual machines, but it does not provide the same level of customization as the az vm create command.
C. The Create-AzVM cmdlet: This cmdlet does not exist in the Az PowerShell module. The correct cmdlet for creating VMs in PowerShell is New-AzVM.

Additional:

Example for Creating an Ubuntu Server VM with a Trusted Root CA:
To create an Ubuntu Server virtual machine and add a trusted root certification authority (CA) using the Azure CLI, you can use the following command:
az vm create --resource-group RG1 --name MyUbuntuVM --image UbuntuLTS --admin-username azureuser --generate-ssh-keys --custom-data cloud-init.txt
In this example, cloud-init.txt is a file that contains the cloud-init configuration to add the trusted root CA. The content of cloud-init.txt might look like this:
#cloud-config
ca-certs:
trusted:
- content: |
-----BEGIN CERTIFICATE-----
MIIDdzCCAl+gAwIBAgIEU1iO/jANBgkqhkiG9w0BAQsFADBoMQswCQYDVQQGEwJV
...
-----END CERTIFICATE-----
filename: /usr/local/share/ca-certificates/my-ca.crt
-----------------
Explanation: This command creates a VM named MyUbuntuVM in the resource group RG1 using the UbuntuLTS image. The cloud-init.txt file is used to configure the VM to include the trusted root CA.
