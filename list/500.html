You have five Azure virtual machines that run Windows Server 2016. The virtual machines are configured as web servers.

You have an Azure load balancer named LB1 that provides load balancing services for the virtual machines.
You need to ensure that visitors are serviced by the same web server for each request.
What should you configure?

  • A..Floating IP (direct server return) to Disabled
  • B..Idle Time-out (minutes) to 20
  • C..a health probe
  • CorrectD..Session persistence to Client IP
Suggested Answer:
D..Session persistence to Client IP — Setting session persistence to "Client IP" ensures that each visitor is consistently routed to the same backend VM for each request, providing the required sticky session behavior.
-----------------
A..Floating IP (direct server return) to Disabled — This setting is not related to session stickiness.
B..Idle Time-out (minutes) to 20 — This only controls how long idle connections are kept open, not which backend server is used.
C..a health probe — Health probes are for monitoring backend health, not for sticky sessions.
