You have five Azure virtual machines that run Windows Server 2016. The virtual machines are configured as web servers.

You have an Azure load balancer named LB1 that provides load balancing services for the virtual machines.
You need to ensure that visitors are serviced by the same web server for each request.
What should you configure?

  • CorrectA..Session persistence to Client IP
  • B..Idle Time-out (minutes) to 20
  • C..Session persistence to None
  • D..Protocol to UDP
Suggested Answer:
A..Session persistence to Client IP — Setting session persistence to "Client IP" ensures that each visitor is consistently routed to the same backend VM for each request, providing the required sticky session behavior.
-----------------
B..Idle Time-out (minutes) to 20 — This only controls how long idle connections are kept open, not which backend server is used.
C..Session persistence to None — This disables sticky sessions, allowing requests to be distributed to any backend.
D..Protocol to UDP — This changes the transport protocol but does not provide session stickiness.
