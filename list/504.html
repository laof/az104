Question : 131 Topic: 5

You have an Azure subscription that contains a Recovery Services vault named Vault1.

You need to enable multi-user authorization (MAU) for Vault1.
Which resource should you create first?

  • A..an administrative unit
  • B..a managed identity
  • CorrectC..a resource guard
  • D..a custom Azure role
Suggested Answer:
C..a resource guard — To enable multi-user authorization (MAU) for a Recovery Services vault, you must first create a resource guard. The resource guard is used to enforce additional authorization requirements for critical operations.
---------------------------
A..an administrative unit — Administrative units are for organizing Azure AD resources and access management, but are not involved in enabling MAU for Recovery Services vaults.
B..a managed identity — Managed identities allow Azure resources to authenticate to services, but are not used to enable MAU for vaults.
D..a custom Azure role — Custom roles define permissions, but enabling MAU specifically requires a resource guard to authorize critical operations.
Additional:
Resource Guard is an Azure feature that helps protect critical resources, such as Recovery Services Vaults and Backup Vaults, from unauthorized modifications. It does this by adding an extra layer of authorization to critical operations.
Resource Guard works by associating a vault with a Resource Guard. When you try to perform a critical operation on the vault, Azure first checks to see if you have the appropriate permissions on the Resource Guard. If you do not have the appropriate permissions, the operation will fail.
Creating a Resource Guard is a simple process that can be done in the Azure portal. You can create a Resource Guard in the same subscription or a different subscription as the vault you want to protect.
