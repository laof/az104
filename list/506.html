You have an Azure subscription that contains the resources shown in the following table.

You create a route table named RT1 in the East US Azure region.
To which resources can you associate RT1?

  • A..VNet1 only
  • CorrectB..Subnet1 only
  • C..VNet1 and NIC1 only
  • D..Subnet1 and NIC1 only
  • E..VNet1, Subnet1, and NIC1
Suggested Answer:
B.. Subnet1 only
— In Azure, route tables can only be associated with subnets and not with virtual networks (VNets) or network interfaces (NICs). When you create a route table like RT1, you can associate it only with Subnet1.
-----------------
A.. VNet1 only — You cannot associate a route table directly with a virtual network (VNet); route tables are associated at the subnet level only.
C.. VNet1 and NIC1 only — Route tables cannot be associated with a VNet or with an individual network interface (NIC).
D.. Subnet1 and NIC1 only — Route tables are not associated with network interfaces, only with subnets.
E.. VNet1, Subnet1, and NIC1 — Only Subnet1 is valid for route table association; VNets and NICs are not supported for direct association.
