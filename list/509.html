You have an Azure subscription that contains the virtual networks shown in the following table.

All the virtual networks are peered. Each virtual network contains nine virtual machines.
You need to configure secure RDP connections to the virtual machines by using Azure Bastion.
What is the minimum number of Bastion hosts required?

  • A..1
  • CorrectB..3
  • C..9
  • D..10
Suggested Answer:
B: 3 — Azure Bastion must be deployed in each Azure region where you want to provide secure RDP/SSH connectivity through Bastion. Since all VNets are peered (including global peering, which connects VNets across regions), this enables communication between VNets in different regions. However, Bastion hosts cannot be shared across regions even with global peering. You still need one Bastion host per Azure region. The table lists three regions: US East, UK South, and Asia East, so the minimum number of Bastion hosts required is 3.
-----------------
A: 1 — Incorrect, because Bastion cannot span multiple Azure regions; you need one per region even if VNets are globally peered.
C: 9 — Incorrect, as this would imply more than one Bastion per region, which is unnecessary due to VNet peering.
D: 10 — Incorrect, as this would mean deploying a Bastion in every VNet, which is not required because peered VNets in the same region (and across regions with global peering) can share a single Bastion, but not across regions.
