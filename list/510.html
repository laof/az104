HOTSPOT -

You have an Azure subscription that contains the virtual networks shown in the following table.
The subscription contains the virtual machines shown in the following table.
Each virtual machine contains only a private IP address.
You create an Azure bastion for VNet1 as shown in the following exhibit.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:
The Remote Desktop Connection client (mstsc.exe) can be used to connect to VM1 through Bastion1. — No
Azure Bastion only supports RDP/SSH access through the Azure portal; native RDP clients like mstsc.exe are not supported for direct Bastion connections.
The Azure portal can use SSH to connect to VM2 through Bastion1. — Yes
Bastion1 in VNet1 can be used to connect to VM2 in VNet2 with SSH via the Azure portal, since VNet1 and VNet2 are peered and in the same region.
The Azure portal can be used to connect to VM3 through Bastion1. — No
VM3 is in VNet3 (West US), and Bastion1 is in East US. Bastion cannot be used across regions, even with peering.
-----------------
mstsc.exe is not supported with Bastion; only browser-based connections via Azure portal are supported.
SSH via Azure portal is supported for VMs in peered VNets in the same region, so VM2 is accessible.
Cross-region Bastion access is not supported, so you cannot access VM3 in West US from a Bastion in East US.
