Question : 138 Topic: 5

HOTSPOT -

You have an Azure subscription that contains the virtual networks shown in the following table.
The subscription contains the subnets shown in the following table.
The subscription contains the storage accounts shown in the following table.
You create a service endpoint policy named Policy1 in the South Central US Azure region to allow connectivity to all the storage accounts in the subscription.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:

Correct Options

Policy1 can be applied to Subnet3. No
Only storage1 and storage2 can be accessed from VNet2. No
Only storage2 can be accessed from VNet3. No
-----------------

Explanation of Right Options

Policy1 can be applied to Subnet3. No
Service endpoint policies can only be applied to subnets in the same region where the policy is created and for supported storage account kinds (StorageV2). Policy1 is in South Central US and Subnet3 is in South Central US, but storage2 is BlobStorage (not supported by endpoint policy), and the policy is to allow connectivity to all storage accounts, so applying to Subnet3 would not restrict access to only StorageV2. The correct answer is No.
Only storage1 and storage2 can be accessed from VNet2. No
Subnet2 in VNet2 has a Microsoft.Storage service endpoint, which allows access to any storage account with Microsoft.Storage endpoint, including storage3. There is no restriction that only storage1 and storage2 can be accessed; all storage accounts in the subscription can be accessed unless restricted by policies, which is not the case here.
Only storage2 can be accessed from VNet3. No
Subnet3 in VNet3 has a Microsoft.Storage service endpoint. It can access all storage accounts with a Microsoft.Storage endpoint, including storage1 and storage3 (both StorageV2), as well as storage2 (BlobStorage). There is no such restriction that only storage2 can be accessed from VNet3.

Explanation of Other Options

All statements are No because the region, storage account kind, and service endpoint configurations do not restrict access in the way the statements describe. Service endpoint policies require both the subnet and the policy to be in the same region and only support StorageV2, and service endpoint configuration allows access to all storage accounts with Microsoft.Storage endpoints unless explicitly limited by policy.
