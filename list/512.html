Question : 139 Topic: 5

You have an Azure virtual network named VNet1 that contains the following settings:

• IPv4 address space: 172.16.10.0/24
• Subnet name: Subnet1
• Subnet address range: 172.16.10.0/25
What is the maximum number of virtual machines that can connect to Subnet1?

  • A..24
  • B..25
  • CorrectC..123
  • D..128
  • E..251
Suggested Answer:

Correct Option

C..123
-----------------

Explanation of Right Option

C..123
Subnet1 uses the address range 172.16.10.0/25, which provides 128 IP addresses (2^(32-25) = 128). Azure reserves 5 IP addresses in each subnet (the first four and the last one), leaving 128 - 5 = 123 usable IP addresses for VMs.

Explanation of Other Options

A..24 and B..25: These are much lower than the available usable IPs for a /25 subnet.
D..128: This is the total number of addresses before Azure reserves 5 for internal use.
E..251: This would require a much larger subnet; /25 only provides 128 total addresses.
