Question : 140 Topic: 5

You have an Azure subscription that contains a resource group named RG1 and a virtual network named VNet1.

You plan to create an Azure container instance named container1.
You need to be able to configure DNS name label scope reuse for container1.
What should you configure for container1?

  • A..the private networking type
  • CorrectB..the public networking type
  • C..a new subnet on VNet1
  • D..a confidential SKU
Suggested Answer:
B..the public networking type
-----------------

Explanation of Right Option

B..the public networking type
To configure DNS name label scope reuse for an Azure Container Instance, you must deploy the container with a public networking type. DNS name labels are used for containers with public IP addresses so that they can be accessed over the internet using a unique DNS name. Private networking does not use public DNS labels.

Explanation of Other Options

A..the private networking type: Private networking does not support public DNS name labels, so scope reuse is not relevant.
C..a new subnet on VNet1: Creating a new subnet affects private IP addressing, not DNS name label scope reuse for public endpoints.
D..a confidential SKU: Confidential SKUs are for enhanced security and do not impact DNS name label configuration or reuse.
