Question : 141 Topic: 5

HOTSPOT -

You have the Azure virtual machines shown in the following table.
VNET1, VNET2, and VNET3 are peered.
VM4 has a DNS server that is authoritative for a zone named contoso.com and contains the records shown in the following table.
The virtual networks are configured to use the DNS servers shown in the following table.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:
From VM1, server1.contoso.com resolves to 131.107.3.3.: No
From VM2, server1.contoso.com resolves to 131.107.3.3.: Yes
From VM3, server2.contoso.com resolves to 131.107.2.4.: No
-----------------

Explanation of Right Options

From VM2, server1.contoso.com resolves to 131.107.3.3.: Yes VNET2 uses custom DNS 192.168.0.5 (VM4), which is authoritative for contoso.com and has an A record for server1.contoso.com pointing to 131.107.3.3, so VM2's queries are resolved by VM4.

Explanation of Other Options

From VM1, server1.contoso.com resolves to 131.107.3.3.: No VNET1 uses the default Azure DNS and does not use the custom DNS server on VM4, so it cannot resolve names from the contoso.com zone hosted by VM4.
From VM3, server2.contoso.com resolves to 131.107.2.4.: No VNET3 uses custom DNS 192.168.0.5 (VM4), but server2.contoso.com in VM4's DNS zone resolves to 131.107.3.4, not 131.107.2.4.
