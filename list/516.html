Question : 143 Topic: 5

You have an Azure subscription.

You are creating a new Azure container instance that will have the following settings:
• Container name: cont1
• SKU: Standard
• OS type: Windows
• Networking type: Public
• Memory (GiB): 2.5
• Number of CPU cores: 2
You discover that the Private setting for Networking type is unavailable.
You need to ensure that cont1 can be configured to use private networking.
Which setting should you change?

  • A..Memory (GiB)
  • CorrectB..Networking type
  • C..Number of CPU cores
  • D..OS type
  • E..SKU
Suggested Answer:
D..OS type
-----------------

Explanation of Right Options

D..OS type The Private networking type is only supported for container instances running the Linux OS type. Windows-based container instances do not support private networking. To enable private networking, you must change the OS type from Windows to Linux.

Explanation of Other Options

A..Memory (GiB) Adjusting memory does not affect the networking type options.
B..Networking type You are trying to enable a private networking type, but it is unavailable because of the OS type, not because of this setting itself.
C..Number of CPU cores The number of CPU cores does not affect the availability of private networking.
E..SKU The SKU (Standard) does not determine whether private networking is available; the limitation is due to the OS type.
