You have an Azure subscription that contains the virtual networks shown in the following table.

The subscription contains the virtual machines shown in the following table.
All the virtual machines have only private IP addresses.
You deploy an Azure Bastion host named Bastion1 to VNet1.
To which virtual machines can you connect through Bastion1?

  • A..VM1 only
  • CorrectB..VM1 and VM2 only
  • C..VM1 and VM3 only
  • D..VM1, VM2, and VM3
Suggested Answer:
B..VM1 and VM2 only
-----------------

Explanation of Right Options

B..VM1 and VM2 only Bastion1 is deployed in VNet1. You can connect to VMs in peered VNets as long as peering allows traffic. VNet1 is peered with VNet2 (and VNet2 with VNet3), so you can connect to VM1 (in VNet1) and VM2 (in VNet2) through Bastion1.

Explanation of Other Options

A..VM1 only is incorrect because you can also reach VM2 in the directly peered VNet2.
C..VM1 and VM3 only is incorrect because there is no direct peering between VNet1 and VNet3; VNet3 is only peered with VNet2, so Bastion1 in VNet1 cannot reach VM3.
D..VM1, VM2, and VM3 is incorrect because, due to peering configuration, Bastion1 cannot connect to VM3 in VNet3.
