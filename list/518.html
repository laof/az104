You have the Azure virtual networks shown in the following table.

Which virtual networks can you peer with VNet1?

  • A..VNet2, VNet3, and VNet4
  • B..VNet2 only
  • CorrectC..VNet3 and VNet4 only
  • D..VNet2 and VNet3 only
Suggested Answer:
C..VNet3 and VNet4 only
-----------------

Explanation of Right Options

C..VNet3 and VNet4 only Peering is only possible if virtual networks have non-overlapping address spaces. VNet3 (10.10.0.0/22) and VNet4 (192.168.16.0/22) do not overlap with VNet1 (10.11.0.0/16), so both can be peered with VNet1.

Explanation of Other Options

A..VNet2, VNet3, and VNet4 is incorrect because VNet2's address space (10.11.0.0/17) overlaps with VNet1's (10.11.0.0/16), so peering is not allowed.
B..VNet2 only is incorrect because VNet2 cannot be peered with VNet1 due to overlapping address spaces.
D..VNet2 and VNet3 only is incorrect because VNet2 overlaps and cannot be peered; only VNet3 and VNet4 are valid.
