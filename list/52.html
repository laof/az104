Question : 12 Topic: 2

You have an Azure subscription named Subscription1 and an on-premises deployment of Microsoft System Center Service Manager.

Subscription1 contains a virtual machine named VM1.
You need to ensure that an alert is set in Service Manager when the amount of available memory on VM1 is below 10 percent.
What should you do first?

  • A..Create an automation runbook
  • B..Deploy a function app
  • CorrectC..Deploy the IT Service Management Connector (ITSM)
  • D..Create a notification
Suggested Answer:

Question 2-12 Answer:

C. Deploy the IT Service Management Connector (ITSM)
-----------------

Explanation of Correct Option:

C. Deploy the IT Service Management Connector (ITSM) — The ITSM Connector integrates Azure with System Center Service Manager, allowing alerts from Azure Monitor to be sent as incidents to Service Manager. This is the required first step to ensure that alerts like low memory on VM1 are forwarded to Service Manager.
-----------------

Explanation of Other Options:

A. Create an automation runbook — Incorrect because automation runbooks are used for remediation or automation tasks, not for integrating with Service Manager.
B. Deploy a function app — Incorrect because function apps are used for serverless code execution, not for ITSM integration.
D. Create a notification — Incorrect because notifications alert users but do not integrate with Service Manager or create incidents.
