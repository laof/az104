HOTSPOT

-
You have an Azure subscription that contains the virtual networks shown in the following table.
Each virtual network has 50 connected virtual machines.
You need to implement Azure Bastion. The solution must meet the fallowing requirements:
• Support host scaling.
• Support uploading and downloading files.
• Support the virtual machines on both VNet1 and VNet2.
• Minimize the number of addresses on the Azure Bastion subnet.
How should you configure Azure Bastion? To answer, select the options in the answer area.
NOTE: Each correct answer is worth one point.

Suggested Answer:
Subnet size: /26
Public IP: Standard SKU with a static allocation
-----------------

Explanation of Right Options

Subnet size: /26 — To support host scaling and minimize the number of addresses, a /26 subnet is the minimum required for Azure Bastion Standard SKU with host scaling enabled. This is recommended by Microsoft documentation for scenarios with many VMs and scaling.
Public IP: Standard SKU with a static allocation — The Standard SKU is required for features like scaling and file transfer (upload and download support). Standard SKU requires a static public IP and enables all the required features.

Explanation of Other Options

/24, /28, /29 — /24 allocates more addresses than needed (not minimizing address usage); /28 and /29 are too small for scaling requirements. Only /26 meets all constraints.
Basic SKU with a dynamic allocation, Basic SKU with a static allocation — The Basic SKU does not support scaling or file transfer features, and dynamic allocation is not supported for Bastion public IPs. Only "Standard SKU with a static allocation" meets all requirements.
