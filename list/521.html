Question : 148 Topic: 5

You have an Azure subscription that contains the virtual networks shown in the following table.

You need to ensure that all the traffic between VNet1 and VNet2 traverses the Microsoft backbone network.
What should you configure?

  • A..a private endpoint
  • CorrectB..peering
  • C..Express Route
  • D..a route table
Suggested Answer:
B..peering
-----------------

Explanation of Right Options

B..peering — Configuring VNet peering between VNet1 and VNet2 ensures that all traffic between the two VNets traverses the Microsoft backbone network, even if the VNets are in different Azure regions. This is the recommended and simplest approach for secure, low-latency inter-VNet communication within Azure.

Explanation of Other Options

A..a private endpoint — Private endpoints are used to connect to Azure services privately, not for connecting two VNets directly.
C..Express Route — ExpressRoute is used to extend on-premises networks into Azure over a private connection, not for VNet-to-VNet traffic within Azure.
D..a route table — Route tables are used for custom routing rules but by themselves do not enforce the use of the Microsoft backbone between VNets.
