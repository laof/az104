Question : 149 Topic: 5

You have an Azure subscription that contains two peered virtual networks named VNet1 and VNet2. VNet1 has a VPN gateway that uses static routing,

The on-premises network has a VPN connection that uses the VPN gateway of VNet1.
You need to configure access for users on the on-premises network to connect to a virtual machine on VNet2. The solution must minimize costs.
Which type of connectivity should you use?

  • A..Azure Firewall with a private IP address
  • CorrectB..service chaining and user-defined routes (UDRs)
  • C..Azure Application Gateway
  • D..ExpressRoute circuits to VNet2
Suggested Answer:
B..service chaining and user-defined routes (UDRs)
-----------------

Explanation of Right Options

B..service chaining and user-defined routes (UDRs) — Since VNet1 and VNet2 are peered and VNet1 already has a VPN gateway, you can use UDRs (User-Defined Routes) to configure traffic from the on-premises network to VNet2 via the peered connection. This approach allows on-premises users to access VMs in VNet2 without deploying additional costly resources, thus minimizing costs.

Explanation of Other Options

A..Azure Firewall with a private IP address — Deploying Azure Firewall adds extra cost and is unnecessary for simple routing between peered VNets.
C..Azure Application Gateway — Application Gateway is used for web traffic load balancing, not for routing VPN or VM connectivity.
D..ExpressRoute circuits to VNet2 — Deploying ExpressRoute is expensive and not required since VPN connectivity is already established with VNet1 and peering connects VNet2.
