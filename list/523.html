Question : 150 Topic: 5

You have an Azure subscription that contains two peered virtual networks named VNet1 and VNet2.

You have a Network Virtual Appliance (NVA) named NetVA1.
You need to ensure that the traffic from VNet1 to VNet2 is inspected by using NetVA1.
What should you use?

  • A..a local network gateway
  • CorrectB..a route table that has custom routes
  • C..a service endpoint
  • D..IP address reservations
Suggested Answer:
B..a route table that has custom routes
-----------------

Explanation of Right Options

B..a route table that has custom routes — To force traffic from VNet1 to VNet2 to be inspected by the Network Virtual Appliance (NetVA1), you must use a route table with custom (user-defined) routes. By configuring the route table, you can direct inter-VNet traffic to pass through NetVA1 for inspection before reaching its destination.

Explanation of Other Options

A..a local network gateway — Local network gateways are used for on-premises connectivity and are not relevant for routing between VNets within Azure.
C..a service endpoint — Service endpoints are used to provide private access to Azure services, not for routing or traffic inspection between VNets.
D..IP address reservations — IP address reservations are for reserving IP addresses, not for routing or traffic inspection purposes.
