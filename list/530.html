HOTSPOT -

You have the web apps shown in the following table.
You need to monitor the performance and usage of the apps by using Azure Application Insights. The solution must minimize modifications to the application code.
What should you do on each app? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
App1: Install the Application Insights Agent
App2: Install the Application Insights Agent

Explanation of Right Options

For App1 (on-premises ASP.NET app on IIS), and App2 (Azure VM ASP.NET Core app on IIS), the best way to monitor performance and usage with minimal code changes is to Install the Application Insights Agent (formerly "Status Monitor v2"). This agent can collect telemetry for .NET and .NET Core applications running on IIS, without requiring code changes or recompilation. It is supported for both on-premises servers and Azure VMs running Windows Server with IIS.
-----------------

Explanation of Other Options

Install the Log Analytics agent: This agent collects performance and event data from the operating system, not application-level telemetry required for Application Insights.
Install the Azure Monitor agent: This is primarily for collecting VM-level metrics and logs; it does not directly enable Application Insights telemetry for web apps without code changes.
Use the Application Insights SDK: This option requires modifying the application code and redeploying, which does not meet the requirement to minimize code changes.
Install the Application Insights Agent is the only option that enables Application Insights for both web apps with minimal modification.
