Question : 10 Topic: 6

Your company has a main office in London that contains 100 client computers.

Three years ago, you migrated to Azure Active Directory (Azure AD).
The company's security policy states that all personal devices and corporate-owned devices must be registered or joined to Azure AD.
A remote user named User1 is unable to join a personal device to Azure AD from a home network.
You verify that User1 was able to join devices to Azure AD in the past.
You need to ensure that User1 can join the device to Azure AD.
What should you do?

  • A..Assign the User administrator role to User1.
  • CorrectB..From the Device settings blade, modify the Maximum number of devices per user setting.
  • C..Create a point-to-site VPN from the home network of User1 to Azure.
  • D..From the Device settings blade, modify the Users may join devices to Azure AD setting.
Suggested Answer:
B..From the Device settings blade, modify the Maximum number of devices per user setting.
Explanation: is correct because if a user was able to join devices to Azure AD in the past but now cannot, it is likely they have reached the limit set by the "Maximum number of devices per user" setting. Increasing this limit allows User1 to join additional devices.
-----------------
A..Assign the User administrator role to User1 is incorrect because joining devices to Azure AD does not require administrative privileges; any user can join devices as long as policy allows.
C..Create a point-to-site VPN from the home network of User1 to Azure is incorrect because joining devices to Azure AD is an internet-based process and does not require VPN connectivity to the corporate network.
D..From the Device settings blade, modify the Users may join devices to Azure AD setting is incorrect because if User1 was able to join devices in the past, this setting is already enabled for the user or group.
