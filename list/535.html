HOTSPOT -

You have an Azure subscription that contains an Azure Active Directory (Azure AD) tenant named contoso.com. The tenant is synced to the on-premises Active
Directory domain. The domain contains the users shown in the following table.
You enable self-service password reset (SSPR) for all users and configure SSPR to have the following authentication methods:
✑ Number of methods required to reset: 2
✑ Methods available to users: Mobile phone, Security questions
✑ Number of questions required to register: 3
✑ Number of questions required to reset: 3
You select the following security questions:
✑ What is your favorite food?
✑ In what city was your first job?
✑ What was the name of your first pet?
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
SecAdmin1 must answer the following question during the self-service password reset: In what city was your first job? No
SecAdmin1 is a Security administrator. In Azure AD, all administrator roles—including Security administrator and Billing administrator—are not allowed to use security questions as an authentication method for self-service password reset. Only non-admin users can use security questions.
BillAdmin1 must answer the following question during the self-service password reset: What is your favorite food? No
Since Billing administrator is also considered an admin role in Azure AD, BillAdmin1 cannot use security questions for password reset, just like SecAdmin1.
User1 must answer the following question during the self-service password reset: What was the name of your first pet? Yes
User1 is a Reports reader (a non-admin user) and can use security questions as an authentication method for self-service password reset as per the policy.
