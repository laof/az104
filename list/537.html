Question : 14 Topic: 6

You have an existing Azure subscription that contains 10 virtual machines.

You need to monitor the latency between your on-premises network and the virtual machines.
What should you use?

  • A..Service Map
  • B..Connection troubleshoot
  • CorrectC..Network Performance Monitor
  • D..Effective routes
Suggested Answer:
C..Network Performance Monitor
Network Performance Monitor (NPM) is the correct choice because it is specifically designed to monitor and visualize network performance, including measuring latency and packet loss between your on-premises environment and Azure virtual machines. NPM provides end-to-end network monitoring capabilities, which is exactly what's needed for monitoring latency.
-----------------
A..Service Map This tool visualizes dependencies between VMs and services but does not provide direct network latency measurements between on-premises and Azure VMs.
B..Connection troubleshoot This is used for troubleshooting specific connection issues and testing connectivity but is not suitable for continuous latency monitoring.
D..Effective routes This feature shows the routes that are being used by a network interface but does not monitor latency or network performance.
Additional:
Network Watcher is a Suite of tools offering but not limited to the following: - Connection Monitor - latency and network issues with IaaS devices over a PERIOD OF TIME
- Connection troubleshoot - latency and network issues with IaaS devices ONE-TIME
- IP Flow - latency and network issues at the VM LEVEL
- Network Performance Monitor - latency and network issues in hybrid, ON-PREM, across environments.
