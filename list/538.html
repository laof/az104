HOTSPOT -

You have an Azure App Service plan named ASP1.
CPU usage for ASP1 is shown in the following exhibit.
Use the drop-down menus to select the answer choice that completes each statement based on the information presented in the graphic.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
1. The average CPU percentage is calculated [four times] per day.

Explanation of Right Option

The chart is set to "Automatic - 6 hours," meaning the CPU percentage is aggregated every 6 hours. Since there are 24 hours in a day, this results in the average CPU percentage being calculated four times per day (24 ÷ 6 = 4).
2. ASP1 must be [scaled up] to optimize CPU usage.

Explanation of Right Option

The maximum CPU percentage frequently reaches 100%, indicating that the current App Service Plan is under-provisioned. To optimize CPU usage and avoid performance issues, scaling up (increasing the plan’s resources) is required.

once, six times, 24 times These options are incorrect because, with a 6-hour aggregation, the average is not calculated once, six times, or 24 times per day.
scaled down This would reduce available resources and worsen performance, not optimize CPU usage.
scaled out Scaling out adds more instances, but since each instance would still hit 100% CPU, the best solution is to scale up.
