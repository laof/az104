You have an Azure web app named webapp1.

Users report that they often experience HTTP 500 errors when they connect to webapp1.
You need to provide the developers of webapp1 with real-time access to the connection errors. The solution must provide all the connection error details.
What should you do first?

  • CorrectA..From webapp1, enable Web server logging
  • B..From Azure Monitor, create a workbook
  • C..From Azure Monitor, create a Service Health alert
  • D..From webapp1, turn on Application Logging
Suggested Answer:
A..From webapp1, enable Web server logging

Explanation of Right Option

Web server logging captures detailed information about HTTP requests and responses, including HTTP 500 errors that originate from the web server or hosting platform. Enabling this allows developers to access all server-level error data, which is critical for diagnosing both application and server-side issues that cause HTTP 500 errors in real time.
-----------------
D..From webapp1, turn on Application Logging
Application Logging mainly records errors and exceptions from the application code; it may not capture all server- or platform-level issues that result in HTTP 500 errors.
B..From Azure Monitor, create a workbook
Azure Monitor workbooks are used for visualizing and analyzing logged data, but they require detailed logging (such as web server logging) to be enabled first in order to display relevant error information.
C..From Azure Monitor, create a Service Health alert
Service Health alerts notify you of Azure platform incidents or outages, not application-level or web server HTTP 500 errors.
