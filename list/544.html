Question : 21 Topic: 6

You have an Azure web app named App1.

You need to monitor the availability of App1 by using a multi-step web test.
What should you use in Azure Monitor?

  • A..Azure Service Health
  • CorrectB..Azure Application Insights
  • C..the Diagnostic settings
  • D..metrics
Suggested Answer:
B..Azure Application Insights

Explanation of Right Option

Azure Application Insights provides availability monitoring, including multi-step web tests (also called multi-step availability tests), which simulate user actions to monitor your web app's availability and responsiveness.
-----------------
A..Azure Service Health
Azure Service Health provides information about Azure service issues and planned maintenance, not specific web app availability monitoring.
C..the Diagnostic settings
Diagnostic settings are used to configure the collection of logs and metrics, but do not directly provide multi-step web tests.
D..metrics
Metrics allow you to view performance and usage statistics, but do not support multi-step web availability testing.
Additional:
To implement a multi-step web test in Azure, follow these steps using Azure Application Insights:
1. In the Azure portal, go to your Application Insights resource.
2. From the left-hand menu, select Availability.
3. Click Add test and choose Multi-step web test.
4. Upload or write your multi-step test script (in .webtest format or using PowerShell/Visual Studio Web Test tools).
5. Configure test locations, frequency, and alert criteria.
6. Save and enable the test to monitor your web appâ€™s availability and simulate user interactions.
-----------------
Key Points: Multi-step web tests require Application Insights, and the test can simulate sequences like login, navigation, and data entry, providing detailed availability and performance monitoring.
