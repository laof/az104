HOTSPOT -

You have an Azure subscription that has diagnostic logging enabled and is configured to send logs to a Log Analytics workspace.
You are investigating a service outage.
You need to view the event time, the event name, and the affected resources.
How should you complete the query? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
1. AzureActivity
2. project

Explanation of Right Option

AzureActivity is the correct table because it contains records of management events in your Azure subscription, including service health events, outages, and operational actions. Filtering with Where Level == 'Critical' selects only critical events.
Using project lets you select and display only the required columns: TimeGenerated (event time), OperationNameValue (event name), and _ResourceId (affected resources).
-----------------
Heartbeat
Heartbeat tracks VM health/availability but does not log management or outage events.
NetworkMonitoring
NetworkMonitoring contains network-related monitoring data, not general Azure activity or outage logs.
Perf
Perf reports performance metrics (CPU, memory, etc.), not service outage or event details.
extend, join, print
extend adds calculated columns, not used here. join combines tables, which is not required. print outputs static values, not used for selecting columns.
