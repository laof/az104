HOTSPOT -

You have the following custom role-based access control (RBAC) role.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
No - Users that are assigned Role1 can assign Role1 to users.: The actions section does not include Microsoft.Authorization/*/Write or Microsoft.Authorization/roleAssignments/write, and the notAction section explicitly denies write and delete actions for Microsoft.Authorization. Therefore, users with Role1 cannot assign roles to others.
Yes - Users that are assigned Role1 can deploy new virtual machines.: The actions section includes Microsoft.Compute/virtualMachines/*, which grants all permissions (read, write, create, delete, etc.) on virtual machines, so users with Role1 can deploy new VMs.
Yes - Users that are assigned Role1 can set a static IP address on a virtual machine.: The actions section includes Microsoft.Network/networkInterfaces/*, which grants all permissions (including write) on network interfaces. This means users with Role1 can set a static IP address on a VM.
