Question : 15 Topic: 2

You have an Azure subscription that contains a resource group named RG26.

RG26 is set to the West Europe location and is used to create temporary resources for a project. RG26 contains the resources shown in the following table.
SQLDB01 is backed up to RGV1.
When the project is complete, you attempt to delete RG26 from the Azure portal. The deletion fails.
You need to delete RG26.
What should you do first?

  • A..Delete VM1
  • B..Stop VM1
  • CorrectC..Stop the backup of SQLDB01
  • D..Delete sa001
Suggested Answer:

Question 2-15 Answer:

C. Stop the backup of SQLDB01
-----------------

Explanation of Correct Option:

C. Stop the backup of SQLDB01 — Azure does not allow deletion of a resource group that contains a Recovery Services vault with active backup items. Since SQLDB01 is backed up to RGV1, you must first stop the backup and delete the backup data before you can delete the vault and the resource group.
-----------------

Explanation of Other Options:

A. Delete VM1 — Incorrect because the VM is not preventing the deletion; the backup configuration is.
B. Stop VM1 — Incorrect because stopping a VM does not affect the backup configuration or deletion process.
D. Delete sa001 — Incorrect because the storage account is not the cause of the deletion failure.

Dependencies That Prevent Deletion of a Recovery Services Vault:

You cannot delete an Azure Recovery Services vault if it has any of the following dependencies:
- Protected data sources (e.g., IaaS VMs, SQL databases, Azure file shares)
- Backup items that have not been deleted
- Backup items in the soft deleted state (retained for 14 days after deletion) - Registered storage accounts
- Active backup policies with associated items
- Multi-User Authorization (MUA) enabled without proper permissions
-----------------

Notes:

To delete the vault, you must first stop all backups, delete backup items, wait for soft delete retention to expire (or disable soft delete if allowed), unregister storage accounts, and remove all associated policies [1](https://learn.microsoft.com/en-us/azure/backup/backup-azure-delete-vault).
