Question : 28 Topic: 6

You have an Azure subscription named Subscription1 that contains two Azure virtual networks named VNet1 and VNet2. VNet1 contains a VPN gateway named

VPNGW1 that uses static routing. There is a site-to-site VPN connection between your on-premises network and VNet1.
On a computer named Client1 that runs Windows 10, you configure a point-to-site VPN connection to VNet1.
You configure virtual network peering between VNet1 and VNet2. You verify that you can connect to VNet2 from the on-premises network. Client1 is unable to connect to VNet2.
You need to ensure that you can connect Client1 to VNet2.
What should you do?

  • A..Select Use the remote virtual network's gateway or Route Server on VNet1 to VNet2 peering.
  • B..Select Use the remote virtual network s gateway or Route Server on VNet2 to VNet1 peering.
  • CorrectC..Download and re-install the VPN client configuration package on Client1.
  • D..Enable BGP on VPNGW1.
Suggested Answer:
C..Download and re-install the VPN client configuration package on Client1.

Explanation of Right Option

C..Download and re-install the VPN client configuration package on Client1.: If there were changes in the address pool, DNS settings, or routing configuration in the VPN gateway, downloading and reinstalling the VPN client configuration would update Client1 with the latest settings. This could help if client routes were missing, but it does not solve gateway transit or peering routing issues.
-----------------
A..Select Use the remote virtual network's gateway or Route Server on VNet1 to VNet2 peering.: Incorrect, because although this is the correct solution to allow point-to-site clients to connect to VNet2, it is not what is being chosen in this answer.
B..Select Use the remote virtual network's gateway or Route Server on VNet2 to VNet1 peering.: Incorrect, as this option does not enable the required gateway transit for Client1.
D..Enable BGP on VPNGW1.: Incorrect, because BGP is not required for static routing or the described connectivity scenario.
