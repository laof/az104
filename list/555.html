Question : 32 Topic: 6

You have a subnet named Subnet1 that contains Azure virtual machines. A network security group (NSG) named NSG1 is associated to Subnet1. NSG1 only contains the default rules.

You need to create a rule in NSG1 to prevent the hosts on Subnet1 form connecting to the Azure portal. The hosts must be able to connect to other internet hosts.
To what should you set Destination in the rule?

  • A..Application security group
  • B..IP Addresses
  • CorrectC..Service Tag
  • D..Any
Suggested Answer:
C. Service Tag:
You should set Destination to Service Tag in the NSG rule. Service tags represent groups of IP addresses for Azure services. By selecting the appropriate service tag (such as AzurePortal or AzureCloud), you can block access to the Azure portal while still allowing other internet traffic. This is the recommended, scalable, and maintainable approach since Azure manages the underlying IP ranges for service tags.
-----------------
Explanation of Other Options:
Application security group: This is used to group and manage VM resources within your own environment, not to represent external or Microsoft-managed services, so it cannot be used to block the Azure portal.
IP Addresses: While technically possible, blocking by IP address is not recommended for the Azure portal because its IP ranges change frequently, making this approach hard to maintain and unreliable.
Any: Setting the destination to "Any" would block all outbound traffic from Subnet1, preventing all internet access, which does not meet the requirement to allow connectivity to other internet hosts.
