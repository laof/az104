Question : 16 Topic: 2

You have an Azure subscription named Subscription1 that contains a virtual network named VNet1. VNet1 is in a resource group named RG1.

Subscription1 has a user named User1. User1 has the following roles:
✑ Reader
✑ Security Admin
✑ Security Reader
You need to ensure that User1 can assign the Reader role for VNet1 to other users.
What should you do?

  • A..Remove User1 from the Security Reader and Reader roles for Subscription1.
  • CorrectB..Assign User1 the User Access Administrator role for VNet1.
  • C..Assign User1 the Network Contributor role for VNet1.
  • D..Assign User1 the Network Contributor role for RG1.
Suggested Answer:

Question 2-16 Answer:

B. Assign User1 the User Access Administrator role for VNet1
-----------------

Explanation of Correct Option:

B. Assign User1 the User Access Administrator role for VNet1 — This role allows a user to manage role assignments in Azure RBAC. To assign roles like Reader to others, a user must have the User Access Administrator role at the appropriate scope (in this case, VNet1).
-----------------

Explanation of Other Options:

A. Remove User1 from the Security Reader and Reader roles for Subscription1 — Incorrect because removing roles does not grant additional permissions.
C. Assign User1 the Network Contributor role for VNet1 — Incorrect because this role allows managing network resources but not role assignments.
D. Assign User1 the Network Contributor role for RG1 — Incorrect for the same reason as option C; it does not grant permission to assign roles.
