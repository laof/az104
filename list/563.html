Question : 40 Topic: 6

You have an Azure subscription that contains 10 network security groups (NSGs), 10 virtual machines, and a Log Analytics workspace named Workspace1. Each NSG is connected to a virtual machine.

You need to configure an Azure Monitor Network Insights alert that will be triggered when suspicious network traffic is detected.
What should you do first?

  • A..Deploy Connection Monitor.
  • B..Configure data collection endpoints.
  • C..Configure a private link.
  • CorrectD..Configure NSG flow logs.
Suggested Answer:
D. Configure NSG flow logs.
-----------------
Explanation of Right Option:
To detect suspicious network traffic using Azure Monitor Network Insights, you must first enable NSG flow logs. NSG flow logs capture information about IP traffic flowing through network security groups. This data is then analyzed (often via a Log Analytics workspace like Workspace1), allowing Network Insights to detect suspicious activity and trigger alerts.
-----------------
Explanation of Other Options:
A: Deploying Connection Monitor is for monitoring connectivity and performance, but it does not provide flow log data needed for traffic analysis.
B: Data collection endpoints are not required for NSG flow log analysis in this scenario.
C: Configuring a private link is unrelated to collecting or analyzing NSG flow data.
