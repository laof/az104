Question : 42 Topic: 6

You have an Azure subscription that contains a storage account named storage1 in the North Europe Azure region.

You need to ensure that when blob data is added to storage1, a secondary copy is created in the East US region. The solution must minimize administrative effort.
What should you configure?

  • A..operational backup
  • CorrectB..object replication
  • C..geo-redundant storage (GRS)
  • D..a lifecycle management rule
Suggested Answer:
B. Object replication
-----------------
Explanation of Right Option:
Object replication allows you to automatically and asynchronously copy blobs between storage accounts in different Azure regions. This is the correct solution to ensure that when blob data is added to storage1 in North Europe, a secondary copy is created in East US, with minimal administrative effort.
-----------------
Explanation of Other Options:
A: Operational backup is for backing up data, not for cross-region replication.
C: Geo-redundant storage (GRS) replicates data to a secondary region, but the secondary copy is not directly accessible; it's only available if Microsoft initiates a failover.
D: Lifecycle management rules are for automating data movement within a storage account (e.g., to archive or delete), not for copying data to another region.
