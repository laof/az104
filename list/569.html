Question : 46 Topic: 6

You deploy Azure virtual machines to three Azure regions

Each region contains a virtual network. Each virtual network contains multiple subnets peered in a full mesh topology.
Each subnet contains a network security group (NSG) that has defined rules.
A user reports that he cannot use port 33000 to connect from a virtual machine in one region to a virtual machine in another region.
Which two options can you use to diagnose the issue? Each correct answer presents a complete solution.
NOTE: Each correct selection is worth one point.

  • A..Azure Virtual Network Manager
  • CorrectB..IP flow verify
  • C..Azure Monitor Network Insights
  • CorrectD..Connection troubleshoot
  • E..elective security rules
Suggested Answer:
B. IP flow verify;
D. Connection troubleshoot
Explanation:
IP flow verify is a feature of Azure Network Watcher that helps determine if traffic is allowed or denied by NSG rules for a specific VM, IP, and port.
Connection troubleshoot (also in Network Watcher) can test connectivity between endpoints, diagnose where traffic is blocked, and provide hop-by-hop analysis, making both tools ideal for diagnosing inter-region connection issues.
-----------------
A. Azure Virtual Network Manager: Used for managing and configuring network topologies and connectivity at scale, not for diagnosing specific port or traffic issues.
C. Azure Monitor Network Insights: Provides visualization and monitoring but does not offer specific diagnostic tools for port connectivity issues.
E. Elective security rules: Refers to user-defined NSG rules, not a diagnostic tool.
