Question : 17 Topic: 2

You have an Azure Active Directory (Azure AD) tenant named contosocloud.onmicrosoft.com.

Your company has a public DNS zone for contoso.com.
You add contoso.com as a custom domain name to Azure AD.
You need to ensure that Azure can verify the domain name.
Which type of DNS record should you create?

  • CorrectA..MX
  • B..NSEC
  • C..PTR
  • D..RRSIG
Suggested Answer:

Question 2-17 Answer:

A. MX
-----------------

Explanation of Correct Option:

A. MX — When adding a custom domain to Azure Active Directory, Microsoft requires you to create a DNS record to verify ownership. The most common method is to create an MX record with specific values provided by Azure. This record does not affect mail flow but is used solely for domain verification.
-----------------

Explanation of Other Options:

B. NSEC — Incorrect because NSEC records are used in DNSSEC to prove the nonexistence of a DNS record, not for domain verification.
C. PTR — Incorrect because PTR records are used for reverse DNS lookups, not for verifying domain ownership.
D. RRSIG — Incorrect because RRSIG records are also part of DNSSEC and are used to digitally sign DNS records, not for domain verification.
