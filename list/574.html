Question : 51 Topic: 6

You have an Azure subscription. The subscription contains virtual machines that connect to a virtual network named VNet1.

You plan to configure Azure Monitor for VM Insights.
You need to ensure that all the virtual machines only communicate with Azure Monitor through VNet1.
What should you create first?

  • A..a data collection rule (DCR)
  • B..a Log Analytics workspace
  • CorrectC..an Azure Monitor Private Link Scope (AMPLS)
  • D..a private endpoint
Suggested Answer:
C. an Azure Monitor Private Link Scope (AMPLS)
Explanation: To ensure that all virtual machines only communicate with Azure Monitor through VNet1, you must first create an Azure Monitor Private Link Scope (AMPLS). This scope allows you to configure private endpoints for Azure Monitor, so that monitoring traffic remains within your virtual network and is not exposed to the public internet.
-----------------

Other Options Explanation

A. a data collection rule (DCR): DCRs define what data to collect and where to send it, but do not control network access or restrict traffic to a specific VNet.
B. a Log Analytics workspace: This is required to store monitoring data, but it does not enforce network-level communication restrictions for Azure Monitor traffic.
D. a private endpoint: A private endpoint is used to connect services privately, but it must be linked to an AMPLS first. Thus, the AMPLS must be created before configuring the private endpoint.

Difference Between Azure Private Link and Private Endpoint

Azure Private Link is a service that enables you to access Azure services (such as Azure Storage, SQL Database, or your own services) privately over a private network, effectively bringing the service into your own VNet.
Private Endpoint is a specific network interface with a private IP address from your VNet, created as part of Private Link, which is used to connect privately to the Azure service.

Other

Azure Private Link provides the overall capability for private connectivity, while a Private Endpoint is the actual resource (network interface) that enables this private connection for a specific service instance.
You use Private Link to enable private connectivity, and Private Endpoint is the technical implementation of that connectivity within your VNet.
