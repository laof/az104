HOTSPOT

-
You have an Azure subscription that contains the vaults shown in the following table.
You create a storage account that contains the resources shown in the following table.
To which vault can you back up cont1 and share1? To answer, select the appropriate options in the answer area.
NOTE: Each correct answer is worth one point.

Suggested Answer:
cont1: Backup1 only
share1: Recovery1 only
Explanation:
- Azure Blob containers (like cont1) can only be backed up to a Backup vault, so only Backup1 is valid.
- Azure File shares (like share1) can only be backed up to a Recovery Services vault, so only Recovery1 is valid.

Other Options Explanation

Backup1 or Recovery1: This is incorrect because each vault supports different resource types: Backup vaults support Blob containers but not File shares, while Recovery Services vaults support File shares but not Blob containers.
Cannot be backed up to Backup1 or Recovery1: This is incorrect because both cont1 and share1 have valid backup solutions with the appropriate vault types.
Recovery1 only (for cont1) / Backup1 only (for share1): These are incorrect because Blob containers are not supported by Recovery Services vaults and File shares are not supported by Backup vaults.

Difference Between Recovery Services vault and Backup vault

Recovery Services vault supports backing up:
- Azure Virtual machines,
- SQL in Azure VM,
- Azure Files (Azure Storage),
- SAP HANA in Azure VM,
- Azure Backup Server,
- Azure Backup Agent,
- DPM.
Backup vault supports backing up:
- Azure Database for PostgreSQL servers (Preview),
- Azure Blobs (Azure Storage),
- Azure Disks.

Other

Recovery Services vault is generally used for VM backups and file shares, while Backup vault is used for newer workloads like blobs, disks, and PostgreSQL servers; each vault type supports different data sources and is not interchangeable.
