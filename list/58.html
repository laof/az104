You have an Azure Directory (Azure AD) tenant named Adatum and an Azure Subscription named Subscription1. Adatum contains a group named Developers.
Subscription1 contains a resource group named Dev.
You need to provide the Developers group with the ability to create Azure logic apps in the Dev resource group.
Solution: On Subscription1, you assign the DevTest Labs User role to the Developers group.
Does this meet the goal?

  • A..Yes
  • CorrectB..No
Suggested Answer:

Question 2-18

Correct Answer: B..No
Explanation: The DevTest Labs User role is intended for managing lab environments in Azure DevTest Labs. It does not grant permissions to create Azure Logic Apps. To meet the goal, the Developers group would need a role like Contributor or a custom role with the necessary permissions to create Logic Apps in the Dev resource group.
-----------------
A..Yes: This is incorrect because the DevTest Labs User role does not include permissions to create Logic Apps. It is limited to managing lab resources, which is unrelated to Logic Apps.

Azure Role Privileges

DevTest Labs User Role: This role is designed for users working within Azure DevTest Labs. It allows users to:
- Create their own virtual machines (VMs) in a lab
- Start, stop, and delete their own VMs
- Apply artifacts to their VMs
- Use custom images and formulas to create VMs
- View lab resources such as policies and virtual networks
Note: This role does not grant permissions to create or manage Azure Logic Apps[1](https://learn.microsoft.com/en-us/azure/devtest-labs/devtest-lab-add-devtest-user)
-----------------
Logic App Contributor Role: This role allows users to manage Azure Logic Apps. It includes permissions to:
- Create, edit, and delete Logic Apps
- Manage workflows, triggers, and actions
- Access and modify Logic App resources
This role is ideal for developers who need full control over Logic App development and maintenance [2](https://learn.microsoft.com/en-us/azure/role-based-access-control/built-in-roles)
