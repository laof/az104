Question : 57 Topic: 6

You have an Azure subscription that has Traffic Analytics configured.

You deploy a new virtual machine named VM1 that has the following settings:
• Region: East US
• Virtual network: VNet1
• NIC network security group: NSG1
You need to monitor VM1 traffic by using Traffic Analytics.
Which settings should you configure?

  • A..Diagnostic settings for VM1
  • CorrectB..NSG flow logs for NSG1
  • C..Diagnostic settings for NSG1
  • D..Insights for VM1
Suggested Answer:
B. NSG flow logs for NSG1
To monitor VM1 traffic using Traffic Analytics, you must enable NSG flow logs on the network security group (NSG1) associated with VM1's network interface. Traffic Analytics analyzes the flow logs generated by NSGs to provide insights into network traffic patterns, threats, and anomalous flows.
-----------------

Explanation of Other Options

A. Diagnostic settings for VM1
These settings are used for collecting diagnostics data like metrics and logs from the VM, but they do not generate the NSG flow logs required for Traffic Analytics.
C. Diagnostic settings for NSG1
While these can collect some logs and metrics, Traffic Analytics specifically requires NSG flow logs, not just diagnostic logs.
D. Insights for VM1
Insights provides VM performance and health monitoring, but it does not enable the traffic flow data required for Traffic Analytics.
Additional:
NSG flow logs are a feature of Azure Network Watcher that allows logging of information about IP traffic flowing through a network security group. This data can be used by Traffic Analytics to analyze network traffic in your environment. By enabling NSG flow logs for NSG1, the Network Security Group associated with VM1, you would be able to monitor the traffic of VM1 using Traffic Analytics.
