Question : 58 Topic: 6

You have an Azure subscription. The subscription contains 10 virtual machines that run Windows Server. Each virtual machine hosts a website in IIS and has the Azure Monitor Agent installed.

You need to collect the IIS logs from each virtual machine and store them in a Log Analytics workspace.
What should you configure first?

  • CorrectA..a data collection endpoint
  • B..an Azure Monitor Private Link Scope (AMPLS)
  • C..Diagnostic settings
  • D..VM insights
  • E..a private endpoint
Suggested Answer:
A. a data collection endpoint
To collect custom logs such as IIS logs from Azure virtual machines using the Azure Monitor Agent, you must first configure a data collection endpoint (DCE). The DCE acts as a routing point for data collected by the Azure Monitor Agent and is required before you can create data collection rules that specify the sources (like IIS logs) and the destination (Log Analytics workspace).
-----------------

Explanation of Other Options

B. an Azure Monitor Private Link Scope (AMPLS)
This is used to securely connect monitoring data sources to Azure Monitor over a private link, but it is not required for collecting IIS logs and is not the first step.
C. Diagnostic settings
Diagnostic settings are used to configure platform logs and metrics for Azure resources, but collecting custom logs from VMs with Azure Monitor Agent requires a data collection endpoint first.
D. VM insights
VM insights provides monitoring for VM performance and dependencies but does not configure the collection of custom log files like IIS logs.
E. a private endpoint
A private endpoint is used for private connectivity to Azure services, but it is not needed to start collecting IIS logs from VMs.
