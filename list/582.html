Question : 59 Topic: 6

HOTSPOT -

You have an Azure subscription that contains two storage accounts named contoso101 and contoso102.
The subscription contains the virtual machines shown in the following table.
VNet1 has service endpoints configured as shown in the Service endpoints exhibit. (Click the Service endpoints tab.)
The Microsoft.Storage service endpoint has the service endpoint policy shown in the Microsoft.Storage exhibit. (Click the Microsoft.Storage tab.)
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.

Suggested Answer:
No for "VM1 can access contoso102."
No for "VM2 can access contoso101."
Yes for "VM2 uses a private IP address to access Azure AD."

Explanations for Right Options

VM1 can access contoso102. - No: Subnet1 has a Microsoft.Storage service endpoint, but a service endpoint policy restricts access to only contoso101. When a policy is enforced, it blocks access to other storage accounts like contoso102, even if public access is available.
VM2 can access contoso101. - No: Subnet2 does not have a Microsoft.Storage service endpoint, so VM2 cannot use private connectivity to reach contoso101. Service endpoint restriction will prevent access if contoso101's firewall only allows traffic from the service endpoint.
VM2 uses a private IP address to access Azure AD. - Yes: Subnet2 has a Microsoft.AzureActiveDirectory service endpoint, so VM2 communicates with Azure AD using its private IP address over the Azure backbone.
Additional:
default, subnets in the same Virtual Network can access each other.
