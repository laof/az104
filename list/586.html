Question : 2 Topic: 7

You need to meet the user requirement for Admin1.

What should you do?

  • A..From the Azure Active Directory blade, modify the Groups
  • B..From the Azure Active Directory blade, modify the Properties
  • CorrectC..From the Subscriptions blade, select the subscription, and then modify the Access control (IAM) settings
  • D..From the Subscriptions blade, select the subscription, and then modify the Properties
Suggested Answer:
The User requirement:
- Designate a new user named Admin1 as the service admin for the Azure subscription.
- Admin1 must receive email alerts regarding service outages.
C. From the Subscriptions blade, select the subscription, and then modify the Access control (IAM) settings
Explanation of the Right OptionOption C is correct because to assign or modify Azure roles for a user (such as Admin1) at the subscription level, you must go to the subscription in the Azure portal and use the Access control (IAM) settings. This is where you grant or change Azure RBAC permissions, allowing you to meet the user requirement for Admin1.

Explanation of the Other Options

Option A: From the Azure Active Directory blade, modify the Groups – This option is used to manage group memberships in Azure AD, but it does not assign Azure RBAC roles at the subscription level.
Option B: From the Azure Active Directory blade, modify the Properties – The Properties blade in Azure AD is for tenant-wide settings and does not control role assignments for subscriptions.
Option D: From the Subscriptions blade, select the subscription, and then modify the Properties – The Properties section contains metadata about the subscription (like name, ID, directory) but not access control settings.
