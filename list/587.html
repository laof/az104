HOTSPOT -
You need to configure Azure Backup to back up the file shares and virtual machines.
What is the minimum number of Recovery Services vaults and backup policies you should create? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:
Recovery Services vaults: 3
Backup policies: 2
Explanations of Right Options
Recovery Services vaults: 3 – Azure Backup requires that Recovery Services vaults be in the same region as the resources they protect. In this scenario, resources needing backup are in West US, East US, and Central US. Therefore, you need a minimum of one vault per region, totaling three vaults.
Backup policies: 2 – You need at least two backup policies: one for VMs and one for file shares. You can apply the same policy to all VMs and a different policy to all file shares, provided the backup schedule/retention requirements are the same.
Explanation of Other Options
Recovery Services vaults: 1, 2, 4, 7 – One or two vaults are insufficient because Azure Backup does not allow cross-region backups; each region (West US, East US, Central US) requires its own vault. More than three vaults is unnecessary unless you have additional requirements like separation by workload or department.
Backup policies: 1 – You cannot use a single policy for both VMs and file shares, as they require different backup policy types. More than two policies would only be needed if you had different schedules or retention for different resources or groups.
