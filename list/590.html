You need to ensure that you can grant Group4 Azure RBAC read only permissions to all the Azure file shares.
What should you do?

  • CorrectA..On storage2, enable identity-based access for the file shares.
  • B..Recreate storage2 and set Hierarchical namespace to Enabled.
  • C..On storage1 and storage4, change the Account kind type to StorageV2 (general purpose v2).
  • D..Create a shared access signature (SAS) for storage1, storage2, and storage4.
Suggested Answer:
A. On storage2, enable identity-based access for the file shares.
Explanation:
Enabling identity-based access for Azure file shares on storage2 allows assigning Azure RBAC roles (like reader) to Group4, which is the required method for secure, manageable, and scalable access control using Azure AD groups.
Explanation of Other Options
B. Recreate storage2 and set Hierarchical namespace to Enabled: This is for Azure Data Lake Storage features, not for Azure Files RBAC permissions, so it does not help with assigning read-only access to file shares.
C. On storage1 and storage4, change the Account kind type to StorageV2 (general purpose v2): Changing account kind does not enable RBAC for file shares; identity-based access is still required for RBAC permissions.
D. Create a shared access signature (SAS) for storage1, storage2, and storage4: SAS provides token-based access, not Azure RBAC, and cannot assign permissions to Azure AD groups like Group4.
