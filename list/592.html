Question : 2 Topic: 10

Scenario:

- Move the existing product blueprint files to Azure Blob storage.
- Copy the blueprint files to Azure over the Internet.
- Ensure that the blueprint files are stored in the archive storage tier.
- Ensure that partner access to the blueprint files is secured and temporary.
- Minimize administrative effort whenever possible.You need to move the blueprint files to Azure.
What should you do?

  • A..Generate an access key. Map a drive, and then copy the files by using File Explorer.
  • CorrectB..Use Azure Storage Explorer to copy the files.
  • C..Use the Azure Import/Export service.
  • D..Generate a shared access signature (SAS). Map a drive, and then copy the files by using File Explorer.
Suggested Answer:
B. Use Azure Storage Explorer to copy the files.
Explanation: Azure Storage Explorer is the best choice because it is simple, secure, minimizes administrative effort, supports uploading files directly to Azure Blob storage (including the archive tier) over the Internet, and allows you to easily manage secure, temporary access for partners.
Explanation of Other Options
A. Generate an access key. Map a drive, and then copy the files by using File Explorer: This is less secure, harder to manage, and does not easily support archive tier or temporary access.
C. Use the Azure Import/Export service: This is for large, offline transfers, not suitable for Internet-based uploads or minimizing admin effort.
D. Generate a shared access signature (SAS). Map a drive, and then copy the files by using File Explorer: This method is unnecessarily complex for this scenario and does not directly support setting the archive tier or simplifying secure, temporary access for partners.
