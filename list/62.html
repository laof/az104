You have an Azure subscription named Subscription1 that contains an Azure Log Analytics workspace named Workspace1.

You need to view the error events from a table named Event.
Which query should you run in Workspace1?

  • A..Get-Event Event | where {$_.EventType == "error"}
  • CorrectB..search in (Event) "error"
  • C..select * from Event where EventType == "error"
  • D..search in (Event) * | where EventType -eq "error"
Suggested Answer:

Question 2-22

Correct Answer: B..search in (Event) "error"
Explanation: This is a valid Kusto Query Language (KQL) query used in Azure Log Analytics. It performs a full-text search for the word "error" within the Event table. While it is not as precise as filtering by a specific field, it is syntactically correct and will return all records that contain the word "error" in any column.
-----------------
A..Get-Event Event | where {$_.EventType == "error"}: Incorrect. This is a PowerShell command, not a KQL query, and cannot be used in Azure Log Analytics.
B..search in (Event) "error": Correct. This is a valid KQL query that searches for the term "error" across all fields in the Event table.
C..select * from Event where EventType == "error": Incorrect. This is SQL syntax, not KQL, and is not valid in Azure Log Analytics.
D..search in (Event) * | where EventType -eq "error": Incorrect. Although the structure resembles KQL, it uses the -eq operator, which is from PowerShell, not KQL. The correct KQL operator is ==.
