You have an Azure subscription that contains a virtual network named VNET1 in the East US 2 region. A network interface named VM1-NI is connected to
VNET1.
You successfully deploy the following Azure Resource Manager template.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Question 2-23

Correct Answers:
VM1 and VM2 can connect to VNET1: Yes
If an Azure datacenter becomes unavailable, VM1 or VM2 will be available: Yes
If the East US 2 region becomes unavailable, VM1 or VM2 will be available: No
Explanation:
VM1 and VM2 can connect to VNET1: Yes — The template shows that VM1 is connected to VNI-NI1, which is already connected to VNET1. It is implied that VM2 is similarly configured. INC and VNET must is in the same region.
If an Azure datacenter becomes unavailable, VM1 or VM2 will be available: Yes — The use of availability zones (e.g., "zones": ["2"]) means the VMs are distributed across different datacenters within the same region, providing high availability.
If the East US 2 region becomes unavailable, VM1 or VM2 will be available: No — Availability zones provide resiliency within a region, but if the entire region (East US 2) goes down, all VMs in that region will be unavailable.
