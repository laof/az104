Question : 26 Topic: 2

You have an Azure subscription.

Users access the resources in the subscription from either home or from customer sites. From home, users must establish a point-to-site VPN to access the Azure resources. The users on the customer sites access the Azure resources by using site-to-site VPNs.
You have a line-of-business-app named App1 that runs on several Azure virtual machine. The virtual machines run Windows Server 2016.
You need to ensure that the connections to App1 are spread across all the virtual machines.
What are two possible Azure services that you can use? Each correct answer presents a complete solution.
NOTE: Each correct selection is worth one point.

  • CorrectA..an internal load balancer
  • B..a public load balancer
  • C..an Azure Content Delivery Network (CDN)
  • D..Traffic Manager
  • CorrectE..an Azure Application Gateway
Suggested Answer:

Question 2-26

Correct Answers: A..an internal load balancer
E..an Azure Application Gateway
Explanation:
A..an internal load balancer: Correct. An internal load balancer distributes traffic across virtual machines within a virtual network. Since users connect via VPN (point-to-site or site-to-site), the traffic is internal to Azure, making an internal load balancer suitable.
E..an Azure Application Gateway: Correct. Application Gateway is a layer 7 load balancer that can distribute HTTP/HTTPS traffic across multiple VMs. It also supports features like SSL termination and session affinity, making it ideal for web-based applications like App1.
-----------------
B..a public load balancer: Incorrect. A public load balancer is used for distributing traffic from the internet. Since users connect through VPNs, the traffic is not public.
C..an Azure Content Delivery Network (CDN): Incorrect. CDN is used for caching and delivering static content globally, not for distributing traffic across VMs.
D..Traffic Manager: Incorrect. Traffic Manager is a DNS-based load balancer used for distributing traffic across regions or endpoints, not for balancing traffic across VMs within the same region.
