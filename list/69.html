Question : 29 Topic: 2

You have an Azure Active Directory (Azure AD) tenant named contoso.onmicrosoft.com.

The User administrator role is assigned to a user named Admin1.
An external partner has a Microsoft account that uses the [email protected] sign in.
Admin1 attempts to invite the external partner to sign in to the Azure AD tenant and receives the following error message: `Unable to invite user [email protected] `" Generic authorization exception.`
You need to ensure that Admin1 can invite the external partner to sign in to the Azure AD tenant.
What should you do?

  • CorrectA..From the Users settings blade, modify the External collaboration settings.
  • B..From the Custom domain names blade, add a custom domain.
  • C..From the Organizational relationships blade, add an identity provider.
  • D..From the Roles and administrators blade, assign the Security administrator role to Admin1.
Suggested Answer:

Question 2-29

Correct Answer: A..From the Users settings blade, modify the External collaboration settings.
Explanation: The error message indicates a problem with inviting an external user. In Azure AD, external collaboration settings control whether users with certain roles (like User administrator) can invite external users. To resolve this, you must go to the Users settings blade and adjust the External collaboration settings to allow invitations by user administrators.
-----------------
A..From the Users settings blade, modify the External collaboration settings.: Correct. This is where you configure who can invite external users and resolve the authorization issue.
B..From the Custom domain names blade, add a custom domain.: Incorrect. This is used to add and verify custom domains, not related to external user invitations.
C..From the Organizational relationships blade, add an identity provider.: Incorrect. This is used for configuring B2B or B2C identity federation, not for fixing invitation permissions.
D..From the Roles and administrators blade, assign the Security administrator role to Admin1.: Incorrect. The Security administrator role is unrelated to user invitations. Admin1 already has the User administrator role, which is sufficient if external collaboration settings are properly configured.

Example Process: Allowing a User Administrator to Invite External Users in Azure AD

Step 1: Sign in to Azure Portal
Go to https://portal.azure.com and sign in with an account that has the Global Administrator role.
Step 2: Navigate to Azure Active Directory
In the left-hand menu, select Azure Active Directory.
Step 3: Open User Settings
Under Users, click on User settings.
Step 4: Modify External Collaboration Settings
In the External users section, click on Manage external collaboration settings.
Step 5: Enable Invitations by User Administrators
Find the setting labeled "Admins and users in the guest inviter role can invite" and ensure it is set to Yes.
Also, confirm that "Members can invite" is set to No (optional, for tighter control), and "Guest invite settings" allow invitations from administrators.
Step 6: Save Changes
Click Save to apply the updated settings.
Step 7: Retry the Invitation
Now, the user with the User administrator role (e.g., Admin1) should be able to invite the external partner without encountering the authorization error.
-----------------
Note: These settings ensure that only authorized roles can invite external users, and they must be explicitly enabled to avoid errors like the one described in the question.
