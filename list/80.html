Question : 40 Topic: 2

You have an Azure DNS zone named adatum.com.

You need to delegate a subdomain named research.adatum.com to a different DNS server in Azure.
What should you do?

  • CorrectA..Create an NS record named research in the adatum.com zone.
  • B..Create a PTR record named research in the adatum.com zone.
  • C..Modify the SOA record of adatum.com.
  • D..Create an A record named *.research in the adatum.com zone.
Suggested Answer:

Question 2-40

Correct Answer: A. Create an NS record named research in the adatum.com zone – To delegate a subdomain like research.adatum.com to another DNS server, you must create a name server (NS) record in the parent zone (adatum.com) that points to the DNS servers responsible for the subdomain.
-----------------
Explanation of Other Options:
B. Create a PTR record named research in the adatum.com zone – Incorrect. PTR records are used for reverse DNS lookups, not for delegating subdomains.
C. Modify the SOA record of adatum.com – Incorrect. The SOA (Start of Authority) record defines authoritative information about the zone itself, not for delegating subdomains.
D. Create an A record named *.research in the adatum.com zone – Incorrect. An A record maps a domain to an IP address and a wildcard A record does not delegate authority to another DNS server.

Additional:

An NS (Name Server) record specifies which authoritative name servers are responsible for a DNS zone. You can include multiple NS records in your zone file to provide redundancy and improve the reliability of your DNS service.
