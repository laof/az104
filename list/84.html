Question : 44 Topic: 2

HOTSPOT -

You have an Azure subscription that contains a storage account named storage1. The subscription is linked to an Azure Active Directory (Azure AD) tenant named contoso.com that syncs to an on-premises Active Directory domain.
The domain contains the security principals shown in the following table.
In Azure AD, you create a user named User2.
The storage1 account contains a file share named share1 and has the following configurations.
For each of the following statements, select Yes if the statement is true. Otherwise, select No.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Question 2-44

Correct Answers:
You can assign the Storage File Data SMB Share Contributor role to User1 for share1: Yes – User1 is a synced on-premises AD user, and since the storage account uses AD DS authentication, User1 can be assigned roles for accessing Azure Files via SMB.
You can assign the Storage File Data SMB Share Reader role to Computer1 for share1: No – Computer accounts cannot be assigned Azure RBAC roles for accessing Azure Files. Only users, groups, and service principals are supported.
You can assign the Storage File Data SMB Share Elevated Contributor role to User2 for share1: No – User2 is a cloud-only Azure AD user and not part of the on-premises AD domain. Since the storage account is configured for AD DS authentication, only domain-joined users can access the file share.
-----------------
Explanation of Other Options:
User1 is valid because they are synced from on-prem AD and can authenticate via AD DS.
Computer1 is not valid because Azure RBAC does not support assigning roles to computer accounts.
User2 is not valid because they are not part of the on-prem AD domain required for AD DS authentication.
