Question : 50 Topic: 2

You have three offices and an Azure subscription that contains an Azure Active Directory (Azure AD) tenant.

You need to grant user management permissions to a local administrator in each office.
What should you use?

  • A..Azure AD roles
  • CorrectB..administrative units
  • C..access packages in Azure AD entitlement management
  • D..Azure roles
Suggested Answer:

Correct Answer:

B.. administrative units
-----------------

Explanation of Correct Option:

Administrative units in Azure AD allow you to delegate administrative permissions to specific users over a subset of users or groups, such as those in a particular office or region. This is ideal for scenarios where you want local administrators to manage users only within their own office.
-----------------

Explanation of Incorrect Options:

A.. Azure AD roles — Incorrect because these roles apply at the tenant level and do not provide scoping to specific groups or offices.
C.. access packages in Azure AD entitlement management — Incorrect because access packages are used for managing access to resources, not for delegating user management permissions.
D.. Azure roles — Incorrect because Azure roles are used for managing Azure resources, not for managing users in Azure AD.

Steps to Configure Administrative Units in Azure AD:

1. Sign in to the Azure portal as a Global Administrator or Privileged Role Administrator.
2. In the left-hand menu, select Azure Active Directory.
3. Under Manage, select Administrative units.
4. Click + Add to create a new administrative unit.
5. Enter a name and optional description for the AU, then click Review + create and Create.
6. After creation, open the AU and select Users or Groups to add members that the AU will manage.
7. Go to Roles and administrators within the AU to assign scoped roles (e.g., User Administrator) to local admins.
8. Click + Add assignment, choose the role, and select the user to assign.
9. The assigned user now has delegated permissions only over the users/groups in that AU.
