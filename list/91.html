You have an Azure Directory (Azure AD) tenant named Adatum and an Azure Subscription named Subscription1. Adatum contains a group named Developers.
Subscription1 contains a resource group named Dev.
You need to provide the Developers group with the ability to create Azure logic apps in the Dev resource group.
Solution: On Dev, you assign the Logic App Contributor role to the Developers group.
Does this meet the goal?

  • CorrectA..Yes
  • B..No
Suggested Answer:

Correct Answer:

A.. Yes
-----------------

Explanation of Correct Option:

The Logic App Contributor role allows users to create, edit, and manage logic apps. Assigning this role to the Developers group at the Dev resource group scope gives them the necessary permissions to create logic apps within that resource group. Therefore, this solution meets the goal.
-----------------

Explanation of Incorrect Option:

B.. No — Incorrect because the Logic App Contributor role is exactly what is needed to allow creation and management of logic apps. Denying this would ignore the correct use of role-based access control in Azure.

Logic App Contributor Role - Key Permissions:

✔️ Microsoft.Logic/* — Full access to manage Logic Apps, including:
  ✔️ Create logic app workflows
  ✔️ Edit and update workflows
  ✔️ Delete logic apps
  ✔️ Manage triggers and actions
  ✔️ View and manage run history and diagnostics
❌ Microsoft.Authorization/* — Not included, so:
  ❌ Cannot assign roles or manage access control (RBAC)
-----------------

Clarification:

The Logic App Contributor role can create logic apps.
