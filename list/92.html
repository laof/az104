HOTSPOT -

You have an Azure Load Balancer named LB1.
You assign a user named User1 the roles shown in the following exhibit.
Use the drop-down menus to select the answer choice that completes each statement based on the information presented in the graphic.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Correct Answers:

User1 can assign access to other users for LB1.
User1 can delete a virtual machine from the resource group.
-----------------

Explanation of Correct Options:

User1 can assign access to other users for LB1 — This is correct because User1 has the User Access Administrator role scoped to LB1, which allows managing role assignments (i.e., granting access) at that resource level.
User1 can delete a virtual machine from the resource group — This is correct because User1 has the Virtual Machine Contributor role at the resource group level, which includes permissions to manage (including delete) virtual machines.
-----------------

Explanation of Incorrect Options:

delete LB1 — Incorrect because neither of the assigned roles grants permission to delete a load balancer.
create a NAT rule for LB1 — Incorrect because User1 does not have a role like Network Contributor or Load Balancer Contributor that would allow modifying load balancer configurations.
modify the load balancing rules in the resource group — Incorrect because the Virtual Machine Contributor role does not include permissions to manage load balancers.
deploy an Azure Kubernetes Service (AKS) cluster to the resource group — Incorrect because deploying AKS requires broader permissions than those granted by the Virtual Machine Contributor role.
