Question : 53 Topic: 2

You have an Azure subscription named Subscription1 that contains a virtual network named VNet1. VNet1 is in a resource group named RG1.

Subscription1 has a user named User1. User1 has the following roles:
✑ Reader
✑ Security Admin
✑ Security Reader
You need to ensure that User1 can assign the Reader role for VNet1 to other users.
What should you do?

  • A..Remove User1 from the Security Reader role for Subscription1. Assign User1 the Contributor role for RG1.
  • CorrectB..Assign User1 the Owner role for VNet1.
  • C..Assign User1 the Contributor role for VNet1.
  • D..Assign User1 the Network Contributor role for VNet1.
Suggested Answer:

Correct Answer:

B.. Assign User1 the Owner role for VNet1
-----------------

Explanation of Correct Option:

The Owner role includes all permissions of the Contributor role plus the ability to manage role assignments (i.e., assign roles to other users). To allow User1 to assign the Reader role for VNet1 to others, they must have Microsoft.Authorization/roleAssignments/write permission, which is included in the Owner role.
-----------------

Explanation of Incorrect Options:

A.. Remove User1 from the Security Reader role for Subscription1. Assign User1 the Contributor role for RG1 — Incorrect because the Contributor role does not allow managing role assignments.
C.. Assign User1 the Contributor role for VNet1 — Incorrect because Contributor can manage resources but cannot assign roles.
D.. Assign User1 the Network Contributor role for VNet1 — Incorrect because this role allows managing network resources but not role assignments.

Additional:

Owner = Grants full access to manage all resources, including the ability to assign roles in Azure RBAC.
Contributor = Grants full access to manage all resources, but does NOT allow you to assign roles in Azure RBAC. (you cannot add users or change their rights)
User Access Administrator = Lets you manage user access to Azure resources.
Reader = View all resources, but does not allow you to make any changes.
Security Admin = View and update permissions for Security Center. Same permissions as the Security Reader role and can also update the security policy and dismiss alerts and recommendations.
Network Contributor = Lets you manage networks, but not access to them. (so you can add VNET, subnet, etc)
