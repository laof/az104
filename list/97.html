HOTSPOT -

You have an Azure subscription that contains the hierarchy shown in the following exhibit.
You create an Azure Policy definition named Policy1.
To which Azure resources can you assign Policy1 and which Azure resources can you specify as exclusions from Policy1? To answer, select the appropriate options in the answer area.
NOTE: Each correct selection is worth one point.
Hot Area:

Suggested Answer:

Correct Answer:

You can assign Policy1 to: Tenant Root Group, ManagementGroup1, Subscription1, RG1, and VM1
You can exclude Policy1 from: ManagementGroup1, Subscription1, RG1, and VM1
Explanation: Azure Policy can be assigned at any scope level: Tenant Root Group, Management Group, Subscription, Resource Group, or Resource. However, you cannot exclude the scope where the policy is assigned. So if the policy is assigned at the Tenant Root Group, you can exclude any child scope (like management groups, subscriptions, RGs, or resources), but not the root itself.
-----------------

Other Options:

- Subscription1 and RG1 only
This is too limited; policies can be assigned at higher scopes like management groups and tenant root.
- ManagementGroup1 and Subscription1 only
Excludes RG1 and VM1, which are valid scopes for policy assignment.
- Tenant Root Group, ManagementGroup1, and Subscription1 only
Excludes RG1 and VM1, which are also valid scopes.
